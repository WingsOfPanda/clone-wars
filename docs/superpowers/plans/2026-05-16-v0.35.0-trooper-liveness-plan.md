# v0.35.0 Trooper-Liveness Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stop false-positive `*S=timeout` writes in `cw_consult_wait` for slow troopers (esp. opencode/DeepSeek V4 Pro) via per-provider timeout multiplier (Layer A) + mtime-based liveness probe (Layer B), without sacrificing the safety net for genuinely wedged troopers.

**Architecture:** Two complementary layers, both opt-in via configuration with sensible defaults. Layer A multiplies the base per-kind timeout by a per-provider factor (opencode = 2.5×). Layer B inserts an mtime-based liveness check before declaring timeout — extends the deadline by `LIVENESS_GRACE_S` if outbox was touched within `LIVENESS_PROBE_S`, capped by `MAX_DEADLINE_FACTOR × baseline`. Mirrors `bin/deep-research-monitor.sh`'s `PROBE_S`/`STUCK_S` pattern.

**Tech Stack:** Pure bash 4.2+ + tmux 3.0+ Claude Code plugin. No Node/Python runtime. `tests/run.sh` runs all `test_*.sh` via bash; `tests/lib/assert.sh` provides `pass`/`assert_eq`/`assert_contains`/`assert_file_exists`/`assert_exit`. Static-wiring locks (`test_v0_*.sh`) version-stamped with skip-guards.

**Spec:** `docs/superpowers/specs/2026-05-16-v0.35.0-trooper-liveness-design.md` (committed `273321e`)

**Already-merged baseline:** v0.34.0 main `602177e`. Branch `feat/v0.35.0-trooper-liveness` already exists with the spec commit.

```bash
# cw_contract_timeout_multiplier <provider>
# Print the timeout_multiplier field for <provider>. Default "1.0" when
# the field is absent or malformed (non-positive, non-numeric).
# Used by cw_consult_wait (lib/consult-wait.sh) to scale per-kind base
# timeouts for slow providers like opencode (DeepSeek V4 Pro).
cw_contract_timeout_multiplier() {
  local provider="$1" v
  v=$(_cw_contract_field "$provider" timeout_multiplier)
  if [[ -n "$v" ]] && [[ "$v" =~ ^[0-9]+(\.[0-9]+)?$ ]] \
     && awk -v x="$v" 'BEGIN { exit !(x > 0) }'; then
    printf '%s\n' "$v"
  else
    printf '1.0\n'
  fi
}
```

Locate the `while :;` loop at line 69. Replace lines 69-71:


With the probe-aware version:

```bash
  # v0.35.0 Layer B: mtime liveness probe before declaring timeout.
  local probe_s="${CW_CONSULT_LIVENESS_PROBE_S:-120}"
  local grace_s="${CW_CONSULT_LIVENESS_GRACE_S:-180}"
  local cap_factor="${CW_CONSULT_MAX_DEADLINE_FACTOR:-2}"
  local hard_deadline=$(( $(date +%s) + timeout * cap_factor ))

  while :; do
    local now=$(date +%s)
    local remaining=$(( poll_deadline - now ))
    if (( remaining <= 0 )); then
      # Liveness probe: check outbox.jsonl mtime.
      local mtime delta
      mtime=$(stat -c '%Y' "$outbox" 2>/dev/null \
              || stat -f '%m' "$outbox" 2>/dev/null \
              || echo 0)
      delta=$(( now - mtime ))
      if (( probe_s > 0 )) && (( mtime > 0 )) \
         && (( delta <= probe_s )) && (( now < hard_deadline )); then
        poll_deadline=$(( now + grace_s ))
        (( poll_deadline > hard_deadline )) && poll_deadline=$hard_deadline
        log_info "[$kind-wait] $commander alive (outbox mtime ${delta}s ago); +${grace_s}s grace, hard cap in $(( hard_deadline - now ))s"
        continue
      fi
      matched=""; new_offset="$OFFSET"; event=""; break
    fi
```

Note the structural changes:
1. Knob initialization (`probe_s`, `grace_s`, `cap_factor`, `hard_deadline`) moved OUTSIDE the while loop (so they're computed once).
2. `now` computed once per iteration via `local now=...`.
3. `(( probe_s > 0 ))` guard makes `CW_CONSULT_LIVENESS_PROBE_S=0` the escape hatch.
4. Chained `stat -c '%Y' || stat -f '%m'` handles GNU stat (Linux) + BSD stat (macOS).

The rest of the while loop body (cw_outbox_wait_since call + tail/grep/EXP_ID guard) stays unchanged.
