## Engineer prelude (read once before starting)

Clone Wars is a Claude Code plugin orchestrating multi-model TUIs (`codex`, `claude`, `opencode`, `gemini`) as tmux panes for cross-verified investigation. The pieces relevant to this patch:

- **`bin/consult-drilldown.sh`**: dispatches a "drill deeper" round to 1 or 2 troopers in parallel during `/clone-wars:consult` Step 13. Current arg shape is 6/7/8/9 positional args:
  ```
  <topic> <section-title> <dd-dir> <focus> <commander1> <model1> [<commander2> <model2>] [<subproject>]
  ```
  At lines 77-79 it tries to read `$TOPIC_DIR/_consult/synthesis.md`, which was deleted in v0.12 — **this is the P0 break**. Step 13 aborts with `synthesis.md not found` whenever a user accepts the drill prompt. The fix: make the directive (caller) own deriving the design-doc path and pass it as a new positional arg at position 5.

- **`commands/consult.md` Step 13** (lines ~957-1054): the only caller. Has 4 invocation examples + prose. All need the new arg.

- **`lib/consult.sh:1185`** has the helper `cw_consult_design_doc_canonical_path <art_dir> <slug>` that returns `<art_dir>/design-doc/<UTC-today>-<slug>-design.md`. It matches what `bin/consult-walk-assemble.sh:36` writes byte-for-byte within a same-UTC-day session (the normal drill cadence). Use it from the directive to derive the design-doc path.

- **`config/prompt-templates/consult/drilldown.md`**: 4-line template loaded by `cw_consult_load_prompt`. Sent verbatim to the trooper. Line 4 still says "Read the synthesis you produced" — leak.

- **`lib/consult-prompts.sh:56`**: helper signature comment for `cw_consult_design_doc_drilldown_prompt` still names arg 2 as `<synthesis-path>`. The local var `syn` and the template `{{SYN}}` substitution token are intentionally NOT renamed (pure-internal, would force coordinated edits for zero gain — see spec Out-of-scope).

- **`tests/lib/assert.sh`** provides `assert_eq`, `assert_contains`, `assert_exit`, `assert_file_exists`, `pass`. All sourced via `source lib/assert.sh` (tests `cd "$(dirname "$0")"` first).

- **`tests/test_consult_drilldown_args.sh`** is the existing arg-validation test. **It currently has 9 cases that all assume the OLD 6/7/8/9-arg shape and assert "synthesis.md not found".** This test must be updated in the same task that changes the script signature, otherwise `bash tests/run.sh` goes red between Tasks 2 and 3.

- **Other drilldown-touching tests** are already compatible: `test_consult_step84_drilldown_wiring.sh` only greps for Step 13 heading + drilldown.sh invocation (still true post-edit); `test_consult_design_doc_drilldown_prompt.sh` calls the helper directly with arg 2 = a synthesis-named path but the helper's parameter name + behavior are unchanged; `test_consult_drilldown_collision_counter.sh` greps unrelated source lines.
