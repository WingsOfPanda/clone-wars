# v0.36.0 `_run/` Pointer Migration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Eliminate the cross-session pointer race where parallel `/clone-wars:*` invocations in different repos overwrite each other's `/tmp/cw-*` files by giving each directive invocation a fresh project-local mktemp dir under `$state_root/_run/<command>.XXXXXX/`. Closes the v0.36.0 dogfood symptom where one trooper reported "The design doc changed on disk while I was validating it."

**Architecture:** New `cw_run_dir <command>` + `cw_run_dir_last` helpers in `lib/state.sh`. First Bash block of each directive calls `cw_run_dir <command>` (mktemp + writeback to `_run/.last`). Subsequent Bash blocks call `cw_run_dir_last` (reads `.last`). All `/tmp/cw-*` pointer reads/writes convert mechanically to `$RUN_DIR/<key>.txt`. 24h stale sweep happens inside `cw_run_dir`; cleanup-on-success not required. Mirrors v0.31.0's `_args/` mktemp pattern.

**Tech Stack:** Pure bash 4.2+ + tmux 3.0+ Claude Code plugin. `tests/run.sh` runs all `test_*.sh`; `tests/lib/assert.sh` provides `pass`/`assert_eq`/`assert_contains`/`assert_file_exists`/`assert_exit`. Static-wiring locks (`test_v0_*.sh`) version-stamped with skip-guards. Reuses existing `cw_atomic_write` helper (`lib/state.sh:101`) for `.last` writeback.

**Spec:** `docs/superpowers/specs/2026-05-16-v0.36.0-run-dir-pointers-design.md` (committed `7ceae58`)

**Scope correction from spec:** `commands/deploy.md` actually has **~11 `/tmp/cw-*` sites** (`/tmp/cw-deploy-args-path.txt`, `/tmp/cw-init-err`, `/tmp/cw-stashpop-err`), not "0–1" as the spec table estimated. T5 covers them as a real migration task; T6 corrects the spec table.

```bash
# cw_run_dir <command>
# Resolve a per-invocation directive scratch dir under $state_root/_run/.
# - Sweeps any sibling _run/<*>/ subdirs whose mtime is >24h old (idempotent;
#   crash-safe; concurrent live invocations untouched since they have fresh
#   mtime). Sweep window overridable via CW_RUN_SWEEP_S.
# - Creates _run/.gitignore = '*' on first call (mirrors $state_root/.gitignore
#   from v0.31.0).
# - Returns absolute path of a fresh mktemp -d _run/<command>.XXXXXX/ via stdout.
# - Writeback the same path to _run/.last (atomic) so subsequent Bash blocks
#   within the same directive can read it without needing /tmp.
cw_run_dir() {
  local command="${1:-}" root run_root run_dir
  [[ -n "$command" ]] || { echo "cw_run_dir: missing <command> arg" >&2; return 2; }
  root=$(cw_state_root)
  cw_state_ensure
  run_root="$root/_run"
  mkdir -p "$run_root"
  [[ -f "$run_root/.gitignore" ]] || printf '*\n' > "$run_root/.gitignore"

  # Sweep stale subdirs.
  local sweep_s="${CW_RUN_SWEEP_S:-86400}"
  local now d mtime
  now=$(date +%s)
  for d in "$run_root"/*/; do
    [[ -d "$d" ]] || continue
    mtime=$(stat -c '%Y' "$d" 2>/dev/null \
            || stat -f '%m' "$d" 2>/dev/null \
            || echo 0)
    if (( mtime > 0 )) && (( now - mtime > sweep_s )); then
      rm -rf "$d"
    fi
  done

  run_dir=$(mktemp -d -p "$run_root" "$command.XXXXXX") \
    || { echo "cw_run_dir: mktemp failed under $run_root" >&2; return 1; }
  printf '%s' "$run_dir" | cw_atomic_write "$run_root/.last" \
    || { echo "cw_run_dir: .last writeback failed" >&2; return 1; }
  printf '%s\n' "$run_dir"
}

# cw_run_dir_last
# Read the most-recently-mktemp'd run dir for THIS project's $state_root.
# Used by directive Bash blocks 2..N to discover the run dir without /tmp.
# Errors if .last is missing (would mean the directive's first Bash block
# didn't call cw_run_dir).
cw_run_dir_last() {
  local root run_root last
  root=$(cw_state_root)
  run_root="$root/_run"
  last="$run_root/.last"
  [[ -f "$last" ]] \
    || { echo "cw_run_dir_last: $last missing — first Bash block must call cw_run_dir" >&2; return 1; }
  cat "$last"
}
```
