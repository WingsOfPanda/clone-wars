## Why

The 2026-05-21 overnight `/clone-wars:deep-research` (topic *train-a-real-alpha*) shipped a working +1581 chained-Elo winner but left a 77 GB archive on disk ‚Äî 207 mid-stream `checkpoint-update-NNN.pt` files in rex/exp-007 alone (97 MB each, ~19 GB for one experiment). The artifact tree was also split across two sibling directories (the `_deep-research/` artifact root vs. the `<cmdr>-<provider>-<sid>/` pane dirs), so a post-hoc analyst has to know to read both. And the global `CW_DEEP_RESEARCH_EXPERIMENT_TIMEOUT_OVERRIDE` env var forced operator-wide timeout decisions when experiments needed different ceilings.

This release is conductor-side hygiene: nothing in the trooper-side IPC contract changes, so no `config/prompt-templates/deep-research/experiment.md` edits and no `commands/deep-research*.md` directive changes. Just finalize.sh + lib/deep-research.sh + experiment-send.sh.

Trooper-side protocol changes (`result.json` schema enforcement, shared `_deep-research/lib/`, knob-consistency audit, K_corroboration semantics) are deferred to v0.53.0's own spec/plan.

## Architecture

`bin/deep-research-finalize.sh` (currently 74 lines: per-trooper phase normalization ‚Üí active-marker removal ‚Üí session-summary atomic write) gains three post-cleanup steps that run AFTER phase normalization and BEFORE the session-summary atomic write:

1. **Prune** intermediate `.pt` files per experiment (default-on, `--keep-intermediate` opts out).
2. **Co-locate** each trooper's pane outbox + inbox into the artifact tree via relative symlinks (always on).
3. **Warn** when any experiment exceeds a size threshold (default 2 GB post-pruning, `CW_DEEP_RESEARCH_SIZE_WARN_GB` overrides).

Each step is a new helper in `lib/deep-research.sh`. `cw_deep_research_render_summary` is extended to read an optional `<art-dir>/warnings.txt` and emit a `## Warnings` section.

Separately, `bin/deep-research-experiment-send.sh` gains a `--timeout N` flag with precedence **CLI > env var > default**.

```
finalize.sh execution order (post-v0.52.0):
1. Per-trooper reconcile + phase normalization     (unchanged)
2. Remove active-<sid>.txt                          (unchanged)
3. NEW: cw_deep_research_prune_intermediate_checkpoints
4. NEW: cw_deep_research_link_pane_artifacts
5. NEW: cw_deep_research_compute_size_warnings
6. cw_deep_research_render_summary | cw_atomic_write   (unchanged, now reads warnings.txt)
```

## Item #19 ‚Äî Checkpoint pruning

### Helper

```bash
cw_deep_research_prune_intermediate_checkpoints <art-dir>
```

Iterates `<art-dir>/troopers/*/experiments/exp-*/`. For each experiment dir:

- Read `result.json:checkpoint_path` via existing `cw_deep_research_json_field`.
- If null / missing / empty string ‚Üí skip the dir entirely (forensics preserved; the operator can prune manually if they want).
- If non-null:
  - Resolve relative to the experiment dir; reject if the resolved path escapes the dir (`realpath --relative-to` test) ‚Äî log_warn and skip.
  - `find . -maxdepth 1 -name '*.pt' ! -wholename "$kept" -print -delete`
- Emit one log line per dir: `pruned: <cmdr>/<exp-id> kept=<filename> removed=<N> freed=<human-size>`.

`du -sB1` before/after to compute freed bytes; print with `numfmt --to=iec --suffix=B`.

Honors `CW_DEEP_RESEARCH_KEEP_INTERMEDIATE=1` (set non-empty ‚Üí helper is a no-op that prints `keep_intermediate=1; pruning skipped`).

### Finalize wiring

`bin/deep-research-finalize.sh` adds:

```bash
# Lane #19: prune intermediate checkpoints (unless opted out)
KEEP_INTERMEDIATE="${CW_DEEP_RESEARCH_KEEP_INTERMEDIATE:-}"
case "${1:-}" in
  --keep-intermediate)
    KEEP_INTERMEDIATE=1
    shift
    ;;
esac
CW_DEEP_RESEARCH_KEEP_INTERMEDIATE="$KEEP_INTERMEDIATE" \
  cw_deep_research_prune_intermediate_checkpoints "$ART"
```

Flag must be the **first** arg if present (keeps the existing `<topic>` positional in place).

|---|---|---|
| 1 | exp dir with `result.json:checkpoint_path=./final.pt` + `final.pt` + `mid-001.pt`, `mid-002.pt` | `final.pt` survives; `mid-001.pt`, `mid-002.pt` removed; log line emitted |
| 2 | exp dir with no `result.json` | All `.pt` survive; helper skips silently |
| 3 | exp dir with `result.json:checkpoint_path=null` | All `.pt` survive |
| 4 | exp dir with `result.json:checkpoint_path=../../escape.pt` | Helper skips with log_warn; no `.pt` removed |
| 5 | `CW_DEEP_RESEARCH_KEEP_INTERMEDIATE=1` | All `.pt` survive across all dirs; helper logs `keep_intermediate=1; pruning skipped` |

## Item #20 ‚Äî Outbox / inbox co-location

### Helper

```bash
cw_deep_research_link_pane_artifacts <art-dir> <topic-dir>
```

For each commander in `<art-dir>/troopers.txt`:

- Resolve pane dir: `<topic-dir>/<cmdr>-codex/` (deep-research is codex-fixed per v0.27.0; no provider lookup needed in v0.52.0 ‚Äî when v0.53.0 generalizes the roster, this becomes a `troopers.txt` v2 schema read).
- For each of `outbox.jsonl` and `inbox.md`:
  - If `<pane-dir>/<file>` does not exist ‚Üí log_warn `pane artifact missing: <cmdr>/<file>` and continue.
  - Compute relative path from `<art-dir>/troopers/<cmdr>/<file>` to `<pane-dir>/<file>`.
  - `ln -sfn <relative-path> <art-dir>/troopers/<cmdr>/<file>` (idempotent symlink replace).

Always runs (no flag ‚Äî symlinks are cheap and pure-addition).

### Finalize wiring

```bash
cw_deep_research_link_pane_artifacts "$ART" "$TD"
```

`$TD` is the `cw_topic_state_dir "$TOPIC"` result that finalize.sh already computes.

| 2 | Run helper twice on same tree | Still 6 symlinks (idempotent), no errors |
| 3 | 1 trooper's pane dir missing outbox.jsonl | log_warn, no symlink for that file, other 5 symlinks created |

## Item #24 ‚Äî Size warnings

### Helper

```bash
cw_deep_research_compute_size_warnings <art-dir>
```

Threshold: `CW_DEEP_RESEARCH_SIZE_WARN_GB` (default `2`). For each `<art-dir>/troopers/*/experiments/exp-*/`:

- `du -sB1` (bytes) of the dir.
- If `>= threshold * 1073741824` ‚Üí append one TSV line to `<art-dir>/warnings.txt`:
  `size_warn<TAB><cmdr>/<exp-id><TAB><size_gb_with_1_decimal><TAB><file_count>`

Truncate `warnings.txt` at the start of the helper (so re-runs are idempotent). If no warnings fire, the file is left empty (or absent ‚Äî the render helper treats both the same).

### Render helper change

`cw_deep_research_render_summary` (in `lib/deep-research.sh`) currently emits Status / Scoreboard / Halt / Current direction / Recent events sections. Add a new `## Warnings` section between Halt and Current direction:

- Read `<art-dir>/warnings.txt` if present and non-empty.
- Each TSV line renders as:
  `- size_warn: rex/exp-007 19.0 GB (213 files)`
- If file is missing or empty ‚Üí omit the section entirely.

### Finalize wiring

```bash
cw_deep_research_compute_size_warnings "$ART"
# (existing) render_summary will pick warnings.txt up
```

Runs AFTER pruning (step 3), so warnings reflect post-prune sizes.

### Tests ‚Äî `tests/test_deep_research_size_warnings.sh`

| Case | Setup | Expected |
|---|---|---|
| 1 | One exp dir with a 3 GB fixture file (`truncate -s 3G`) | `warnings.txt` has 1 line; rendered summary has `## Warnings` section with that experiment |
| 2 | All exp dirs < 2 GB | `warnings.txt` is empty or absent; rendered summary has no `## Warnings` section |
| 3 | `CW_DEEP_RESEARCH_SIZE_WARN_GB=10` + 3 GB exp | Threshold respected; no warnings.txt entry |

## Item #26 ‚Äî `--timeout` flag on experiment-send

### Arg parser change

`bin/deep-research-experiment-send.sh` flag loop currently handles `--inputs`, `--context-file`, `--smoke-test`. Add `--timeout`:

```bash
TIMEOUT_FLAG=""   # init at top alongside INPUTS, CONTEXT_FILE, SMOKE_TEST

# in the parse loop:
  --timeout)
    TIMEOUT_FLAG="$2"
    [[ "$TIMEOUT_FLAG" =~ ^[1-9][0-9]*$ ]] \
      || { echo "error: --timeout must be a positive integer (seconds); got '$TIMEOUT_FLAG'" >&2; exit 2; }
    shift 2
    ;;
```

### Resolution change

Line 137 changes from:

```bash
TIME_BUDGET_S="${CW_DEEP_RESEARCH_EXPERIMENT_TIMEOUT_OVERRIDE:-$(cw_consult_timeout experiment)}"
```

to:

```bash
TIME_BUDGET_S="${TIMEOUT_FLAG:-${CW_DEEP_RESEARCH_EXPERIMENT_TIMEOUT_OVERRIDE:-$(cw_consult_timeout experiment)}}"
```

Precedence: CLI flag > env var > `cw_consult_timeout experiment` default.

### Tests ‚Äî `tests/test_deep_research_experiment_send_timeout.sh`

|---|---|---|

| 2 | `--timeout 900` | `900` |
| 3 | env `CW_DEEP_RESEARCH_EXPERIMENT_TIMEOUT_OVERRIDE=600` + `--timeout 900` | `900` (CLI wins) |
| 4 | `--timeout abc` | Exit code 2, usage error to stderr |

Uses the `CW_DEEP_RESEARCH_DRY_RUN=1` codepath the script already supports for tests (skips pane nudge + actual codex spawn).
