# v0.34.0 — Long-Session Tooling + Workflow Patterns Design

**Goal:** Close 6 dogfood findings that improve `/clone-wars:deep-research` ergonomics during long, evolving sessions — relax brief-length guidance, pre-flight check input file readability, ship `fresh-trooper` + `refine` scripts, allow custom topic slugs, and document the two workflow patterns that worked in the CRF session.

**Architecture:** Pure additive bundle on top of v0.33.0's schema work. Two new bin scripts (`deep-research-fresh-trooper.sh`, `deep-research-refine.sh`). Two flag additions (`experiment-send.sh --inputs`, `deep-research-init.sh --slug`). One optional per-experiment file (`task_context.md`). Directive prose updates for #11 (brief length), #19 (approach diversity), #20 (asymmetric-framing convergence).

**Tech Stack:** Bash 4.2+, tmux 3.0+. No new dependencies. Reuses existing `bin/teardown.sh --pairs` for fresh-trooper's pane teardown and `bin/spawn.sh` for respawn.

**Branch:** `feat/v0.34.0-long-session-tooling`.

**Baseline:** v0.33.0 (will be at main when this ships); plugin.json reports `0.33.0` going into v0.34.0.

## Design decisions

Four load-bearing choices made during brainstorming (2026-05-15):

**D1 — Fresh-trooper semantics (#13):** pane respawn + state preserved. `bin/deep-research-fresh-trooper.sh <topic> <commander>` invokes `bin/teardown.sh --pairs <topic> <commander>` (kills the tmux pane gracefully via the existing 9s banner), then re-invokes `bin/spawn.sh <commander> codex <topic>` (codex CLI starts a fresh session in the new pane). `state.txt` is preserved — `exp_counter`, `phase`, `current_exp_id`, etc. — so the next dispatch picks up where the old trooper left off. Experiment history under `troopers/<cmdr>/experiments/` is untouched. The new trooper sees no inbox until the next dispatch. Yoda's next experiment brief should reference the prior context implicitly (it's already in `session-summary.md` and `scoreboard.md`). (Alternative considered: codex `/clear` inside existing pane. Rejected — codex CLI's session-reset semantics aren't standardized.)

**D2 — Refine delivery (#14):** numbered refine files + tmux nudge. `bin/deep-research-refine.sh <topic> <commander> <exp-id> <refinement-text>` writes `troopers/<commander>/experiments/<exp-id>/refine-1.md` (numbers increment if refine-1.md exists: refine-2.md, refine-3.md, ...). Then sends a one-line tmux nudge via existing send-keys: `REFINE: read refine-1.md before continuing`. Trooper reads, acknowledges (no protocol event required — implicit in next outbox emission), adjusts. Multiple refines accumulate as an audit trail. No inbox.md mutation; doesn't conflict with bug #4's inbox-stacking concern. (Alternative considered: pane interrupt + send-keys. Rejected — breaks experiment wall-clock timing.)

**D3 — File-access probe (#12):** explicit `--inputs=<path1>,<path2>,...` flag on `experiment-send.sh`. Before writing inbox, the script tests `[[ -r $path ]]` for each. Any unreadable → rc=2 with the specific path listed (`pre-flight: cannot read /home/liupan/crf/file.html`). Yoda passes the flag when the brief references file inputs; for tasks with no file inputs (most ML benchmarks), the flag is omitted and probe is a no-op. (Alternative considered: auto-parse brief for paths. Rejected — false-positive + false-negative brittleness.)

**D4 — Brief length + task_context.md (#11):** two-part. (a) Directive prose: drop the "1-2 sentence max" wording; replace with "Briefs may be 1-2 sentences for simple tasks or 1-2 paragraphs for complex ones. Anything past ~300 tokens should go in a `task_context.md` file per-experiment." (b) `experiment-send.sh` gains `--context-file=<path>` flag. If passed, the file content is interpolated into the rendered experiment.md via a new `{{TASK_CONTEXT}}` placeholder (which is empty when the flag is omitted). The trooper reads it as part of the dispatch context. Per-experiment placement (`<branch_dir>/task_context.md`) keeps it scoped to the experiment, not session-wide. (Alternative considered: just relax wording, no file. Rejected — long briefs in inbox.md pollute the visible flow.)

## Per-script changes

### `bin/deep-research-fresh-trooper.sh` (NEW, ~50 LoC)

```
Usage: bin/deep-research-fresh-trooper.sh <topic> <commander>

Exit codes:
  0 = ok
  1 = trooper state or pane missing (refuse — call spawn.sh directly)
  2 = usage error / invalid topic
```

Flow:
1. Source `lib/log.sh` + `lib/state.sh` + `lib/consult.sh`.
2. Validate args, auto-prefix topic.
3. Resolve `ART_DIR`; refuse rc=1 if `troopers/<commander>/state.txt` is absent.
4. Read current phase from state.txt; refuse rc=1 if `phase=working` (can't reset a trooper mid-experiment; tell user to run abort or wait).
5. Invoke `bin/teardown.sh --pairs <topic> <commander>` (graceful 9s banner). Wait for completion.
6. Invoke `bin/spawn.sh <commander> codex <topic>` (re-spawns in new pane, same commander+topic).
7. Update state.txt: keep `exp_counter` + `current_exp_id` (should be empty if phase != working); reset `phase=idle`, `last_event=fresh-trooper-respawn`, `probe_sent_ts=`.
8. Print confirmation with the new pane_id.

The script does NOT touch `troopers/<commander>/experiments/*` — full history preserved.

### `bin/deep-research-refine.sh` (NEW, ~40 LoC)

```
Usage: bin/deep-research-refine.sh <topic> <commander> <exp-id> <refinement-text>

Exit codes:
  0 = ok
  1 = trooper or experiment dir missing
  2 = usage error / invalid topic
```

Flow:
1. Source helpers + validate args (topic auto-prefix, commander regex, exp-id `^exp-[0-9]+$` regex).
2. Resolve `BRANCH_DIR = $ART_DIR/troopers/<cmdr>/experiments/<exp-id>`; refuse rc=1 if absent.
3. Find the next refine slot: `n=1; while [[ -f "$BRANCH_DIR/refine-$n.md" ]]; do n=$((n+1)); done`.
4. Write the refinement text to `$BRANCH_DIR/refine-$n.md`.
5. Read pane_id from `<topic-dir>/<cmdr>-codex/pane.json` (same shape experiment-send reads).
6. Send a one-line tmux nudge: `bin/send.sh <cmdr> <topic> "REFINE: read $BRANCH_DIR/refine-$n.md before continuing"`. This invokes the v0.33.0 send.sh which will print the mid-experiment warning (expected — refines are explicitly for mid-experiment).
7. Print confirmation.

Refines do not modify state.txt (the trooper is still phase=working on the same exp-id).

### `bin/deep-research-experiment-send.sh` — `--inputs` + `--context-file` flags

**Flag parsing additions** (before existing positional args parse):

```bash
INPUTS=""
CONTEXT_FILE=""
# Parse --inputs / --context-file flags
while [[ $# -gt 0 ]]; do
  case "$1" in
    --inputs=*)       INPUTS="${1#*=}"; shift ;;
    --context-file=*) CONTEXT_FILE="${1#*=}"; shift ;;
    --) shift; break ;;
    -*) break ;;  # unrecognized flag — drop to positional parsing
    *) break ;;
  esac
done
```

(The exact placement requires care — experiment-send.sh today is strictly positional. The flag parsing happens BEFORE the 5-positional-arg check.)

**Pre-flight file-access probe** (after the topic validation block):

```bash
if [[ -n "$INPUTS" ]]; then
  IFS=',' read -ra INPUT_PATHS <<< "$INPUTS"
  for p in "${INPUT_PATHS[@]}"; do
    [[ -r "$p" ]] || { log_error "pre-flight: cannot read input path '$p'"; exit 2; }
  done
fi
```

**Task context interpolation** (before the awk render block):

```bash
TASK_CONTEXT_VAL=""
if [[ -n "$CONTEXT_FILE" ]]; then
  [[ -r "$CONTEXT_FILE" ]] \
    || { log_error "cannot read --context-file: $CONTEXT_FILE"; exit 2; }
  TASK_CONTEXT_VAL=$(<"$CONTEXT_FILE")
fi
```

Add `{{TASK_CONTEXT}}` to the awk replacement set (empty string if no file passed).

**Template change in `config/prompt-templates/deep-research/experiment.md`:**

Insert a new section after "Approach brief:" line (around current line 21):

```
{{TASK_CONTEXT}}
```

The `awk` render emits the file content verbatim when CONTEXT_FILE is set, or an empty line when it's not. The trooper reads it inline with the rest of the prompt.

### `bin/deep-research-init.sh` — `--slug` flag

**Flag addition:**

```bash
SLUG_OVERRIDE=""
# (in the existing while-getopts loop)
    --slug)     SLUG_OVERRIDE="$2"; shift 2 ;;
    --slug=*)   SLUG_OVERRIDE="${1#*=}"; shift ;;
```

**Slug derivation block (lines 56-62 today):**

```bash
if [[ -n "$SLUG_OVERRIDE" ]]; then
  # Validate the override: lowercase + a-z0-9- + 1-18 chars
  [[ "$SLUG_OVERRIDE" =~ ^[a-z][a-z0-9-]{0,17}$ ]] \
    || { log_error "--slug must match ^[a-z][a-z0-9-]{0,17}\$; got '$SLUG_OVERRIDE'"; exit 2; }
  SLUG_BASE="$SLUG_OVERRIDE"
else
  SLUG_BASE=$(printf '%s' "$TOPIC" \
    | tr '[:upper:]' '[:lower:]' \
    | tr -c 'a-z0-9-' '-' \
    | sed 's/--*/-/g; s/^-//; s/-$//' \
    | cut -c1-18 \
    | sed 's/-$//')
fi
```

Validation is stricter than auto-derivation: requires lowercase start + 1-18 chars total. Keeps the 32-char `deep-research-` + slug invariant.

### Directive prose updates

**`commands/deep-research.md` Phase 4 — relax brief-length wording:**

Find the existing "1-2 sentence" guidance. Replace with:

```
**Briefs may be 1-2 sentences for simple tasks or 1-2 paragraphs for
complex ones.** Anything past ~300 tokens should go in a per-experiment
`task_context.md` file (pass via `bin/deep-research-experiment-send.sh
--context-file=<path>`). Long inline briefs make session-summary.md hard
to scan.
```

**`commands/deep-research.md` Phase 1 — approach diversity guidance (#19):**

After the existing roster-size guidance, append a new sub-section:

```
**Approach diversity — prefer different pipelines over identical ones.**
3 troopers running the same pipeline produce 3 identical results — no
triangulation signal. 3 troopers running DIFFERENT pipelines (e.g.
single-pass VLM / page-typed routing / OCR+VLM hybrid) provide real
cross-validation. When you assign initial approaches in Phase 3, prefer
diverse strategies that probe the problem from different angles. Note
the diversity rationale in the first round's session-summary.md
`## Current direction` section.
```

**`commands/deep-research.md` Phase 5 — asymmetric-framing convergence (#20):**

After the existing dispatch guidance, append:

```
**Asymmetric-framing convergence — leverage observed quality gaps.** If
one trooper consistently produces lower-quality outputs across multiple
experiments (visible in scoreboard or consensus.md `## Contested`),
explicitly frame the next round to defer: "Trooper X has been more
accurate on this task; for this experiment, use their values as a
baseline unless you have strong contrary evidence." This converges the
laggard faster than uniform direction. Document the asymmetry in
`session-summary.md` `## Recent decisions`.
```
