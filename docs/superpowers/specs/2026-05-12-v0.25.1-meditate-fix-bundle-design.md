# v0.25.1 — `/clone-wars:meditate` fix bundle design

**Status:** approved 2026-05-12. Implementation plan to follow.

**Goal:** Patch-release closing 4 bugs surfaced by the v0.25.0 strict-dogfood pass on `/clone-wars:meditate "explore SOTA autoresearch pipeline like in https://github.com/karpathy/autoresearch"`. No new features; pure fix + simplification.

**Architecture (one paragraph):** Three of the four bugs are mechanical corrections — directive prose typos in `commands/meditate.md` Step 2 (preflight call shape + spawn arg order) and a one-line extension to `lib/consult.sh:cw_consult_topic_validate` (mirroring the v0.25.0 `cw_consult_art_dir` prefix-aware change). The fourth bug is a user-validated design refinement: Yoda no longer runs its own `Skill(literature-review, ...)` (which duplicated Cody-as-claude's web search), but Yoda's keyword classifier *is preserved* — its result (`_meditate/lit-track.txt` content) is now passed to each trooper via a new `{{LIT_GUIDANCE}}` placeholder in their research prompt, telling them how much effort to spend on academic lit retrieval. Conductor = orchestrator + synthesizer only; troopers = the only retrievers.

**Tech Stack:** pure bash 4.2+. Reuses every v0.25.0 file; deletes 1 lib helper (`cw_meditate_parse_lit_flag`) + 1 unit-test file (`test_meditate_parse_lit_flag.sh`); modifies 1 prompt template (adds `{{LIT_GUIDANCE}}`); modifies 1 bin script (`bin/meditate-research-send.sh` reads lit-track + renders guidance block); preserves 12-task list shape.

**Spec ancestor:** `docs/superpowers/specs/2026-05-11-v0.25.0-meditate-command-design.md` (v0.25.0 approved 2026-05-11).

## 1. Bugs closed

### Bug 1 — `commands/meditate.md` Step 2 preflight call shape

Current (broken; from v0.25.0 directive prose):

```
"$CLAUDE_PLUGIN_ROOT/bin/preflight-layout.sh" --art-dir _meditate "$ART_DIR"
```

`bin/preflight-layout.sh`'s actual signature is `[flags] <topic> <N>` — `--art-dir` expects an absolute path (not a relative `_meditate`), and `<N>` is missing. The dogfood produced `Usage: ...` and exited 1.

Fix (relying on the v0.25.0 prefix-aware `cw_consult_art_dir` to route `meditate-*` → `_meditate/`):

```
"$CLAUDE_PLUGIN_ROOT/bin/preflight-layout.sh" "$MEDITATE_TOPIC" "$N"
```

### Bug 2 — `commands/meditate.md` Step 2 spawn arg order

Current (broken):

```
"$CLAUDE_PLUGIN_ROOT/bin/spawn.sh" \
  --target-pane <pane_id_from_preflight> \
  --preflight-art-dir "$ART_DIR" \
  <commander> <provider> "$MEDITATE_TOPIC"
```

`bin/spawn.sh` declares positionals at line 70 with `COMMANDER="$1"; MODEL="$2"; TOPIC="$3"; shift 3` *before* the flag-parse loop. With flags-first, the dogfood captured `COMMANDER=--target-pane`, `MODEL=<pane-id>`, `TOPIC=--preflight-art-dir`. Resolution failed at `model '%1951' has no entry in contracts.yaml`.

Fix (positionals before flags, per usage doc at `bin/spawn.sh:46`):

```
"$CLAUDE_PLUGIN_ROOT/bin/spawn.sh" \
  <commander> <provider> "$MEDITATE_TOPIC" \
  --target-pane <pane_id_from_preflight> \
  --preflight-art-dir "$ART_DIR"
```

### Bug 3 — `lib/consult.sh:cw_consult_topic_validate` prefix-awareness

Current (broken):

```bash
cw_consult_topic_validate() {
  local topic="$1"
  [[ -n "$topic" ]] || return 1
  [[ "$topic" == consult-* ]] || return 1
  [[ "$topic" =~ ^[A-Za-z0-9_.-]+$ ]] || return 1
  [[ "$topic" != .* && "$topic" != -* ]] || return 1
  [[ "$topic" != *..* ]] || return 1
}
```

The `consult-*` prefix check rejects every `meditate-*` topic. `cw_consult_wait research` and `cw_consult_wait adversary` (which both internally call `cw_consult_assert_topic` → this validator) fail with `[FAIL] invalid topic: meditate-explore-sota-autores`. Dogfood worked around by manual outbox polling.

Fix (mirror the v0.25.0 `cw_consult_art_dir` extension):

```bash
cw_consult_topic_validate() {
  local topic="$1"
  [[ -n "$topic" ]] || return 1
  [[ "$topic" == consult-* || "$topic" == meditate-* ]] || return 1
  [[ "$topic" =~ ^[A-Za-z0-9_.-]+$ ]] || return 1
  [[ "$topic" != .* && "$topic" != -* ]] || return 1
  [[ "$topic" != *..* ]] || return 1
}
```

After this fix, `bin/consult-research-wait.sh meditate-<topic> ...` and `bin/meditate-adversary-wait.sh ...` work end-to-end with no shim or fallback poll.

### Bug 4 — Drop conductor-side literature-review (preserve trooper-side; pass classifier result to troopers)

**Observation from dogfood.** With trooper roster {rex/codex, cody/claude, wolffe/opencode}, Cmdr Cody (claude) already runs WebSearch + WebFetch as part of its research, retrieving the exact same SOTA papers that the conductor-side `Skill(literature-review, ...)` call retrieves. The conductor-side `_meditate/literature-review.md` and Cody's `findings-cody.md` were ~80% overlapping on the dogfood topic. **Yoda-as-claude doing the same lit-review-skill work as Cody-as-claude is pure duplication.**

**User rule (2026-05-12):**

1. "only troopers doing is enough, conductor wait to receive their findings and put together, not need to do its own research too"
2. "we still need yoda to auto decide whether trooper need to do lit review. just yoda itself won't do lit review."

**Final scope.** Yoda still classifies the topic and decides whether literature emphasis is warranted. That decision is **passed to the troopers** so each trooper knows how much effort to spend on academic lit search. Yoda itself does NOT run any retrieval skill — it only orchestrates and synthesizes.

**Removed in v0.25.1:**

- `commands/meditate.md` Step 3 `Skill(skill: "literature-review", ...)` parallel call — removed (the entire Skill invocation; the trooper dispatch calls in Step 3 remain)
- `commands/meditate.md` Step 0 `cw_meditate_parse_lit_flag` invocation + `LIT_STATE` variable + flag-parse block — removed
- `commands/meditate.md` Step 1 `case "$LIT_STATE" in force-on/force-off/auto` branching — collapsed to direct `cw_meditate_classify_topic` call (no flag overrides)
- `commands/meditate.md` frontmatter `argument-hint: [--lit | --no-lit] <topic>` → `argument-hint: <topic>`
- `commands/meditate.md` "When to use" section's prose about `--lit` / `--no-lit` flags
- `lib/meditate.sh:cw_meditate_parse_lit_flag` — deleted
- `tests/test_meditate_parse_lit_flag.sh` — deleted (entire file)
- `bin/meditate-synth-preliminary.sh` reference to `literature-review.md` optional-input warning — removed (file no longer written by Yoda)
- `tests/test_v0_25_0_static_wiring.sh` invariant 3: drop `cw_meditate_parse_lit_flag` from the required-functions loop; `cw_meditate_classify_topic` + `cw_meditate_art_dir` remain required

**Kept (preserved unchanged):**

- `commands/meditate.md` Step 1 (literature auto-detect classifier — Yoda runs `cw_meditate_classify_topic` against the topic; writes `_meditate/lit-track.txt` with `ON|OFF` + reason). This decision is now consumed by *troopers*, not by Yoda's own lit-review skill.
- `lib/meditate.sh:cw_meditate_classify_topic` (Yoda's keyword classifier — still used)
- `lib/meditate.sh:cw_meditate_lit_keywords` (Yoda's keyword list — still used)
- `lib/meditate.sh:cw_meditate_art_dir` (used by all bin scripts)
- `tests/test_meditate_lit_keywords.sh` (the classifier still works; the test still passes)

- `config/prompt-templates/meditate/landscape-skeleton.md` (unchanged)
- `bin/meditate-synth-final.sh`, `bin/meditate-teardown.sh`, `bin/meditate-adversary-send.sh`, `bin/meditate-adversary-wait.sh` (all unchanged)
- `tests/test_meditate_init.sh`, `tests/test_meditate_confidence_gate.sh`, `tests/test_meditate_e2e.sh`

**New / modified for trooper-hint propagation:**

- `config/prompt-templates/meditate/research.md` — **modified.** New `{{LIT_GUIDANCE}}` placeholder replaces the static `## SOTA evidence` paragraph. The send-script renders one of two blocks based on `lit-track.txt` content:
  - `ON` → "The topic is academic / SOTA-shaped. Prioritize peer-reviewed papers (arXiv, conference proceedings) over blog posts or vendor docs. List 3+ recent papers, projects, or benchmarks with citations including authors, year, venue, URL/DOI where available."
  - `OFF` → "The topic is not academic-shaped. Brief SOTA-evidence section is fine — list 1-2 anchor sources or write 'Not applicable' with a one-line reason."
- `bin/meditate-research-send.sh` — **modified.** Before calling `cw_consult_load_prompt`, read `$ART_DIR/lit-track.txt`, branch on `ON|OFF`, and pass the resolved `LIT_GUIDANCE` block to the template loader as a third placeholder. Falls back to the `OFF` block if `lit-track.txt` is missing (defensive — Step 1 should always write it).

The conductor does not run `Skill(literature-review, ...)` at any point. The conductor's lit-track classification serves as a *hint to troopers* about where to spend their retrieval effort. Each trooper still does its own retrieval using its TUI's native search tools — that retrieval is now appropriately weighted.

## 7. Open design questions

None. The brainstorming pass closed on:

- **Conductor lit-retrieval:** dropped entirely. Yoda runs no `Skill(literature-review, ...)`.
- **Yoda's classifier:** preserved. Step 1 still classifies the topic via `cw_meditate_classify_topic` and writes `_meditate/lit-track.txt`.
- **Trooper-side propagation:** lit-track state is now consumed by `bin/meditate-research-send.sh` and rendered into each trooper's research prompt as `{{LIT_GUIDANCE}}`, telling the trooper whether to emphasize academic papers.
- **Flag overrides:** `--lit` / `--no-lit` dropped. Yoda's classifier is the only decider; user influences via topic phrasing.
- **Static-wiring test:** extend invariant 3 in-place (drop the deleted parse-flag helper from the EXPECTED list). No new v0.25.1 file.
