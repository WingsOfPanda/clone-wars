# v0.25.1 — `/clone-wars:meditate` fix bundle design

**Status:** approved 2026-05-12. Implementation plan to follow.

**Goal:** Patch-release closing 4 bugs surfaced by the v0.25.0 strict-dogfood pass on `/clone-wars:meditate "explore SOTA autoresearch pipeline like in https://github.com/karpathy/autoresearch"`. No new features; pure fix + simplification.

**Architecture (one paragraph):** Three of the four bugs are mechanical corrections — directive prose typos in `commands/meditate.md` Step 2 (preflight call shape + spawn arg order) and a one-line extension to `lib/consult.sh:cw_consult_topic_validate` (mirroring the v0.25.0 `cw_consult_art_dir` prefix-aware change). The fourth bug is a user-validated design simplification: remove conductor-side literature-review duplication. The trooper-side research prompt template is unchanged — troopers (rex/cody/wolffe) continue to retrieve and cite SOTA papers as part of their own research. The change removes Yoda's *redundant* parallel `Skill(literature-review, ...)` call (when Yoda-is-claude runs alongside Cody-as-claude doing the same web search, they produce ~80% overlapping output).

**Tech Stack:** pure bash 4.2+. Reuses every v0.25.0 file; deletes 2 lib helpers + 2 unit-test files + 1 prompt-template field; renames Step numbering in 1 directive.

**Spec ancestor:** `docs/superpowers/specs/2026-05-11-v0.25.0-meditate-command-design.md` (v0.25.0 approved 2026-05-11).

## 1. Bugs closed

### Bug 1 — `commands/meditate.md` Step 2 preflight call shape

Current (broken; from v0.25.0 directive prose):

```
"$CLAUDE_PLUGIN_ROOT/bin/preflight-layout.sh" --art-dir _meditate "$ART_DIR"
```

`bin/preflight-layout.sh`'s actual signature is `[flags] <topic> <N>` — `--art-dir` expects an absolute path (not a relative `_meditate`), and `<N>` is missing. The dogfood produced `Usage: ...` and exited 1.

Fix (relying on the v0.25.0 prefix-aware `cw_consult_art_dir` to route `meditate-*` → `_meditate/`):

```
"$CLAUDE_PLUGIN_ROOT/bin/preflight-layout.sh" "$MEDITATE_TOPIC" "$N"
```

### Bug 2 — `commands/meditate.md` Step 2 spawn arg order

Current (broken):

```
"$CLAUDE_PLUGIN_ROOT/bin/spawn.sh" \
  --target-pane <pane_id_from_preflight> \
  --preflight-art-dir "$ART_DIR" \
  <commander> <provider> "$MEDITATE_TOPIC"
```

`bin/spawn.sh` declares positionals at line 70 with `COMMANDER="$1"; MODEL="$2"; TOPIC="$3"; shift 3` *before* the flag-parse loop. With flags-first, the dogfood captured `COMMANDER=--target-pane`, `MODEL=<pane-id>`, `TOPIC=--preflight-art-dir`. Resolution failed at `model '%1951' has no entry in contracts.yaml`.

Fix (positionals before flags, per usage doc at `bin/spawn.sh:46`):

```
"$CLAUDE_PLUGIN_ROOT/bin/spawn.sh" \
  <commander> <provider> "$MEDITATE_TOPIC" \
  --target-pane <pane_id_from_preflight> \
  --preflight-art-dir "$ART_DIR"
```

### Bug 3 — `lib/consult.sh:cw_consult_topic_validate` prefix-awareness

Current (broken):

```bash
cw_consult_topic_validate() {
  local topic="$1"
  [[ -n "$topic" ]] || return 1
  [[ "$topic" == consult-* ]] || return 1
  [[ "$topic" =~ ^[A-Za-z0-9_.-]+$ ]] || return 1
  [[ "$topic" != .* && "$topic" != -* ]] || return 1
  [[ "$topic" != *..* ]] || return 1
}
```

The `consult-*` prefix check rejects every `meditate-*` topic. `cw_consult_wait research` and `cw_consult_wait adversary` (which both internally call `cw_consult_assert_topic` → this validator) fail with `[FAIL] invalid topic: meditate-explore-sota-autores`. Dogfood worked around by manual outbox polling.

Fix (mirror the v0.25.0 `cw_consult_art_dir` extension):

```bash
cw_consult_topic_validate() {
  local topic="$1"
  [[ -n "$topic" ]] || return 1
  [[ "$topic" == consult-* || "$topic" == meditate-* ]] || return 1
  [[ "$topic" =~ ^[A-Za-z0-9_.-]+$ ]] || return 1
  [[ "$topic" != .* && "$topic" != -* ]] || return 1
  [[ "$topic" != *..* ]] || return 1
}
```

After this fix, `bin/consult-research-wait.sh meditate-<topic> ...` and `bin/meditate-adversary-wait.sh ...` work end-to-end with no shim or fallback poll.

### Bug 4 — Drop conductor-side literature-review duplication

**Observation from dogfood.** With trooper roster {rex/codex, cody/claude, wolffe/opencode}, Cmdr Cody (claude) already runs WebSearch + WebFetch as part of its research, retrieving the exact same SOTA papers that the conductor-side `Skill(literature-review, ...)` call retrieves. The conductor-side `_meditate/literature-review.md` and Cody's `findings-cody.md` were ~80% overlapping on the dogfood topic. The conductor pass is pure duplication.

**User rule (2026-05-12):** "only troopers doing is enough, conductor wait to receive their findings and put together, not need to do its own research too, which would duplicate with claude code trooper's research"

**Important scope clarification (per user 2026-05-12):** *Trooper-side* literature review is **preserved**. The fix removes only the *conductor's* parallel lit-review track. Every trooper's research prompt (`config/prompt-templates/meditate/research.md`) is unchanged — it still asks for `## SOTA evidence`, `## Independent Discovery`, citation-required `## Approaches`, and explicitly tells the trooper to use WebSearch / WebFetch / equivalent TUI tools to find authoritative sources.
