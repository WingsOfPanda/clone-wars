## v0.48 — Halt + Scoreboard Rendering Bundle

### Helpers (extracted first, callers migrated second)

**`cw_deep_research_halt_flag_read <halt-flag-path>`** — added to `lib/deep-research.sh`.

- Reads `halt.flag` and emits a normalized k=v block on stdout, one field per line, preserving newlines in multi-line values via `\n` literal escaping in output.
- Detects format on first non-blank line:
  - Contains `=` and first key is `halted_by` → structured (current format)
  - Otherwise → prose (legacy pre-v0.43 format). Emits only:
    ```
    format=prose
    reason=<full file contents, newlines preserved>
    ```
- Missing or empty file → emits `format=missing`.
- Closes #1 (the `tr -d '\n'` bug), #2 (prose fallback), #3 (separates parse from phase-normalize).
- Unit tests (5 cases): structured halt, prose halt, missing halt, empty halt, halt with multi-line reason field.

**`cw_deep_research_scoreboard_render_row <metric_value> <runtime_s> <metric_name> <status> <approach>`** — added to `lib/deep-research.sh`.

- Formats one tab-separated scoreboard row with `printf '%.4f'` for `metric_value` and `printf '%.2fs'` for `runtime_s`.
- `metric_value` of `(running)` or empty passes through unformatted.
- Other fields pass through verbatim.
- Closes #6 (float width) + #7 (runtime drift). Raw values still preserved in `result.json` for K-corroboration arithmetic.
- Unit tests (4 cases): long float metric, integer runtime, missing metric → `(running)`, NaN guard.

### Migrations

#### Finding #1+#2+#3 — halt rendering

**Files:** `bin/deep-research-finalize.sh`, `lib/deep-research.sh` (consumer of halt.flag in `cw_deep_research_render_summary`).

- `bin/deep-research-finalize.sh:32` replaces `cat "$ART/halt.flag" | tr -d '\n'` with a call to `cw_deep_research_halt_flag_read`, formats output as a fenced k=v block in the session-summary `## Halt` section.
- `cw_deep_research_render_summary` halt rendering:
  - For `format=structured` halts → render as fenced k=v block (newlines preserved, no concatenation).
  - For `format=prose` halts → render as `- Reason: <text>` block.
  - For `format=missing` → omit the section entirely.
- Phase-normalization (idle → complete for non-current troopers) splits into its own pre-render step that runs unconditionally, decoupled from halt.flag parse outcome. Closes #3 — even if halt.flag is unparseable, troopers reach `phase=complete`.

#### Finding #5 — scoreboard tie-breaker

**File:** `bin/deep-research-score.sh:91` (the `sort -k1,1 -rn`).

Replace with multi-key sort:
```bash
sort -t$'\t' -k1,1rn -k4,4n -k2,2n
```
Field order: metric (desc), runtime (asc), exp_id (asc). Tab-delimited fields are emitted upstream of this sort in score.sh.

Unit test: 3-way tie at metric=0.945, different runtimes — assert deterministic winner.

#### Finding #6+#7 — scoreboard formatting

**File:** `bin/deep-research-score.sh` (the scoreboard row emission loop).

Replace inline row-printing with calls to `cw_deep_research_scoreboard_render_row`. All raw values (including unrounded `metric_value` and `runtime_s`) remain in `result.json`; only the rendered scoreboard.md changes.

#### Finding #13 — schema_version line

**File:** `bin/deep-research-score.sh` (scoreboard.md generation).

First line of scoreboard.md becomes:
```
<!-- scoreboard schema_version=2 -->
```

Cross-run parsers can dispatch on this line. Version 1 = pre-v0.48 (no comment); version 2 = v0.48+ (this format).

## v0.49 — State-File Hygiene Cleanup

### Per-finding fixes

#### Finding #8 — archive timestamp drift

**File:** `bin/deep-research-teardown.sh`.

Symptom: paired archive paths drift by up to 8s because `cw_state_archive` and the artifact-dir `mv` each call `date -u +…` independently.

Fix: compute timestamp once near the top of teardown.sh, pass as positional arg to both consumers:

```bash
STAMP=$(date -u +%Y%m%dT%H%M%SZ)
cw_state_archive ... "$STAMP"
mv "$ART_DIR" "$ARCHIVE_BASE/.../$base-$STAMP/_deep-research"
```

Requires `cw_state_archive` to accept an optional STAMP positional arg; if absent, falls back to computing its own (back-compat for other callers).

#### Finding #9 — lane_abandon_reason fragility

**Files:** `lib/deep-research.sh` (writer `cw_deep_research_trooper_state_write` + reader `cw_deep_research_trooper_state_field`).

Two-sided fix:

- **Writer:** escape literal `=` as `\=` and embedded newlines as literal `\n` (two characters) in any value before writing to state.txt. Reverse on read.
- **Reader:** split on first `=` only — `local key="${line%%=*}"; local val="${line#*=}"` — instead of awk -F= behavior that splits on every `=`.

Test fixture (`tests/test_deep_research_state_field_escape.sh`): write a 200-char reason value containing `=` and embedded newline; read back, assert equality after unescape.

#### Finding #10 — stale probe_sent_ts

**File:** `bin/deep-research-monitor.sh`.

Symptom: `probe_sent_ts` written when a probe fires, never cleared. Resume diagnostics see stale timestamps.

Fix: when the monitor processes any non-probe event (`progress`, `done`, `error`, `heartbeat`), clear `probe_sent_ts=` to empty in the same state.txt write that updates `last_event`/`last_event_ts`.

Test (`tests/test_deep_research_probe_clear.sh`): seed state with `probe_sent_ts=2026-05-20T10:00:00Z`, fire a `done` event, assert next state has `probe_sent_ts=` empty.

#### Finding #12 — plateau_window field-name collision (doc-only rename)

**File:** `commands/deep-research.md` — the literal halt.flag example block (around L920-930, under "Optional keys (Yoda's target-met / corroboration halt SHOULD include)").

Rename the *measured* field from `plateau_window=<N>` to `plateau_observed_n=<N>`. The *configured* `plateau_window` field in metric.md (read by `cw_deep_research_check_completion` at L490) stays untouched — that's the authoritative config; the rename only affects what Yoda is *instructed to write* into runtime halt.flag artifacts.

No code change; directive prose only. Lock invariant guards the rename. Acknowledged limitation: the lock verifies the *instruction* in the directive, not the actual halt.flag artifact in any given run (those are Yoda-runtime free-form). This is sufficient because future runs follow the updated directive; legacy archives are immutable and stay readable via the `format=prose` fallback in `cw_deep_research_halt_flag_read`.

## Risk surface

### v0.48

- **`cw_deep_research_render_summary` halt section format changes.** Consumers: anyone reading session-summary.md, including Yoda's resume-handler 3.b in `commands/deep-research-resume.md`. Mitigation: lock invariant #4 prevents regression to `tr -d`; unit test for halt_flag_read covers all three format paths.
- **scoreboard.md schema_version line is a new first line.** Existing parsers may treat HTML comments as content. Mitigation: the `<!--` prefix is valid Markdown that renders as nothing; cross-run scoreboard comparison tooling (if any exists) gets the explicit version flag it needs.

### v0.49

- **state.txt format change for reasons containing `=`.** Writer and reader change land in the same commit. Old archives read by new code may have unescaped `=` — but the reader's first-`=` split handles them correctly (it's strictly more permissive than awk `-F=`). Lock invariant #3 prevents drift.
- **`cw_state_archive` STAMP arg is optional.** Other callers (e.g. `bin/teardown.sh`, `bin/consult-teardown.sh`) continue calling without it and pay the date computation themselves. Back-compat.

## What this does NOT do

- No prompt-template changes (#4 dropped).
- No time-budget UX changes (#11 verified working).
- No new helper for trooper state.txt parsing beyond first-`=` split (existing parser stays single-purpose).
- No backfill of legacy archives (immutable by design).
