  `cw_consult_design_doc_self_review`; the wait scripts inline

  v0.12 split + 2 months after v0.17 silent-ignore. 30 LoC of directive

  flag that already does nothing.
- **Sentinel-block strip helper** (`cw_consult_strip_block` +
  `TARGETS_BLOCK_START`/`END` plumbing) exists to preserve byte-equal
  template rendering for per-sub-project blocks that v0.14 hub-mode
  removal already deleted. Helper is now empty-string-passthrough every
  time it's called.
- **Three duplication clusters** â€” `consult-research-wait.sh` â†”
  `consult-verify-wait.sh` (~80% identical, 100 LoC dup),
  `lib/contracts.sh` awk parser repeated 5Ã— across getters (60 LoC dup),
  `bin/teardown.sh` `--pairs` branch vs 2-arg branch (25 LoC dup), and
  preflight orphan cleanup duplicated between `consult-teardown.sh` and
  `deploy-teardown.sh` (20 LoC dup).
- **Three pattern inconsistencies** â€” manual `tmp+mv` in `deploy.md`
  Step 5b/8/1 (4 sites) where `cw_atomic_write` is already widely used;
  `deploy.md` Step 5b re-implements the DAG-parse regex inline instead
  of calling `cw_deploy_dag_parse_line`; `consult.md` Step 9 prose
  expands one idea into 14 lines; `consult.md` patterns block has rich

  writes 3 files via 3 separate pipelines from the same data; `bin/spawn.sh`

None of these are bugs (everything works). All of them are friction:
larger files to read, more places to misalign, dead code that misleads
future-Claude grep results.

```bash
cw_preflight_kill_orphans() {
  local pfp="$1"; shift
  declare -A live; for c in "$@"; do live["$c"]=1; done
  [[ -f "$pfp" ]] || return 0
  while IFS=$'\t' read -r cmdr pane; do
    [[ -n "$cmdr" && -n "$pane" ]] || continue
    [[ "${live[$cmdr]:-0}" == "1" ]] && continue
    log_info "killing preflight orphan pane $pane (commander=$cmdr)"
    tmux kill-pane -t "$pane" 2>/dev/null || log_warn "kill-pane $pane failed"
  done < "$pfp"
  rm -f "$pfp"
}
```

### Cluster 4 â€” `lib/contracts.sh` awk parser dedup (P0-7)

```bash
_cw_contract_field() {
  local provider="$1" field="$2" path
  path=$(cw_contracts_path)
  [[ -f "$path" ]] || return 1
  awk -v p="$provider" -v f="$field" '
    /^[A-Za-z][A-Za-z0-9_-]*:[[:space:]]*$/ {
      key = $0; sub(/:[[:space:]]*$/, "", key)
      in_block = (key == p); next

      v = $0; sub("^  "f":[[:space:]]*", "", v)
      gsub(/^[ \t]+|[ \t\r]+$/, "", v)
      print v; exit
    }
  ' "$path"
}
```

## Out of scope

- **Not splitting `lib/consult.sh` further.** Per P2-1 the call graph
  is one-way; splitting for splitting's sake isn't a real win. After
  Cluster 1 the file is ~870 LoC, acceptable.
- **Not collapsing single-repo `bin/spawn.sh` `.last_pane` branch
  into preflight.** Per P1-4, real spec contract â€” v0.20.0 spec
  deliberately kept single-repo "byte-equal v0.19.0". v0.24.0 honors
  that.
- **Not deduping `cw_consult_topic_dir` / `cw_deploy_topic_dir`** into
  `cw_topic_state_dir` (P1-1). The named alternatives improve
  readability at callsites â€” pure pass-throughs are cheap, alternative

- **Not deduping `cw_cmdr_rank` and `cw_consult_provider_to_commander`
  (P1-2)** â€” they answer different questions, sharing a YAML schema
  would be a bigger refactor.
- **Not deduping `bin/*.sh` preamble** (P1-3) â€” bootstrap paradox
  makes it a wash.
- **Not upgrading `commands/deploy.md` intervention patterns to match
  `commands/consult.md`** â€” opposite direction was chosen (strip
  consult, keep deploy terse). Per user.
- **No new commands.** No new env vars beyond what already exists.
  No new test framework. No new config format.
