# v0.32.0 — Monitor Surface + Dispatch UX Design

**Goal:** Close 7 findings from the v0.29.0 CRF-extraction dogfood feedback at `/home/liupan/crf/CLONE_WARS_DEEP_RESEARCH_FEEDBACK.md` — the highest-impact quality-of-life pain in `/clone-wars:deep-research` with zero blast outside that surface.

**Architecture:** Pure deep-research maintenance bundle. Monitor becomes phase-aware and cursor-persistent; `deep-research-init.sh` gains two CLI flags that bypass Phase 1/2 AskUserQuestions; new abort script collapses the manual TaskStop + teardown.sh + deep-research-teardown.sh sequence into one bash call.

**Tech stack:** Bash 4.2+, tmux 3.0+, existing `lib/` helpers. No new dependencies.

**Branch:** `feat/v0.32.0-monitor-surface`. Single PR.

**Baseline:** v0.31.0 at main `6889212`; plugin.json reports `0.31.0`.

---

## Motivation

The user ran `/clone-wars:deep-research` intensively for 15 rounds (~24 hours) on a CRF extraction task on clone-wars v0.29.0. The framework's substrate (pane allocation, tmux integration, spawn.sh, codex CLI integration, archive layout) held up cleanly. The polish layer had 21 documented frictions. This release closes the noisiest 7:

| # | Finding | Impact |
|---|---|---|
| 1 | Monitor fires stale/stuck on idle troopers — ~50+ noise notifications observed per session | high |
| 2 | Monitor missed `done` events 3× in the session (mechanism unverified) | medium |
| 3 | Monitor replay on restart — every prior outbox event re-emits as fresh | high |
| 7 | Topic validation rejects `rex` (one accidental typo) instead of auto-prefixing | low (annoying) |
| 8 | `PROBE_S=300` / `STUCK_S=600` defaults too aggressive for codex xhigh (routine 15-25min experiments) | high |
| 16 | No one-shot graceful teardown — user had to TaskStop each Monitor by hand, then run teardown.sh + deep-research-teardown.sh | medium |
| 23 | UNCONDITIONAL Phase 1/2 prompts feel heavy-handed when the user already explicitly answered in their original message | medium (UX) |

v0.33.0 (schema rework + consensus primitive) and v0.34.0+ (long-session tooling + workflow templates) are queued in project memory and explicitly out of scope.

## Design decisions

Six load-bearing choices made during brainstorming (2026-05-15):

**D1 — Monitor phase-aware policy:** stale/stuck events fire only when `phase=working`. Once Yoda probes a stale trooper (`phase=stale`) or the trooper asks a question (`phase=blocked`), Monitor goes silent. Yoda owns all escalation past initial detection. (Alternative considered: allow `phase=stale` to escalate to stuck. Rejected — risks re-introducing the noise loop; Yoda judgment is more reliable than threshold ladders.)

**D2 — Cursor persistence strategy:** persist across runs. On Monitor startup, if `liveness-cursor.txt` is non-empty AND its value ≤ current outbox size, honor it. Otherwise init to current outbox byte-count (skip everything prior to first scan). (Alternative considered: explicit `--resume` flag. Rejected — fewer moving parts; persistence is already implicit in the file's existence.)

**D3 — Missed-done safety net:** periodic full line-count rescan every `CW_DEEP_RESEARCH_RESCAN_EVERY_S=30` seconds. The rescan re-reads the entire outbox by line, tracks emitted lines via side cursor `liveness-rescan-emitted.txt`, and forwards any unseen `done`/`error`/`question` events. Catches both stale-mtime and parse-failure cases. (Alternative considered: switch byte-cursor to line-cursor. Rejected — addresses one cause, not both.)

**D4 — UNCONDITIONAL exception mechanism:** explicit `--time-budget` and `--metric` flags on `deep-research-init.sh`. State-file presence becomes the bypass; flags are the documented mechanism that produces a pre-populated state file. No NLP, no answer-caching, no Yoda lexical judgment. UNCONDITIONAL stamp now reads "UNCONDITIONAL when state file absent; skip when present." (Alternative considered: Yoda-judgment confirmation prompt. Rejected — false-positive lexical matches risk re-litigating the v0.28.2 always-ask invariant.)

**D5 — Abort script semantics:** `bin/deep-research-abort.sh` writes `halt.flag` → invokes `deep-research-finalize.sh` (renders landscape doc from current scoreboard) → invokes `deep-research-teardown.sh` (`--pairs`-mode batched teardown + archive). Monitor TaskStop is deferred to the next Yoda turn because TaskStop is a Claude tool. The resume handler's Step 2 already calls TaskStop on `monitor-tasks.txt` entries when `halt.flag` is detected. (Alternative considered: hard kill without finalize. Rejected — losing the synthesis pass loses the session's research value.)

**D6 — Topic auto-prefix:** in `deep-research-experiment-send.sh`, prepend `deep-research-` when missing. So `rex` becomes `deep-research-rex`. Validation against `cw_consult_topic_validate` still runs on the prefixed result. (Alternative considered: relaxed any-valid-slug check. Rejected — risks collisions with consult/meditate topics in the same state dir.)

## Per-file changes

### `bin/deep-research-monitor.sh` (75 → ~115 LoC)

Phase-aware liveness, cursor persistence, periodic rescan, default bumps.

**Source addition** (after current `source "$PLUGIN_ROOT/lib/log.sh"`):
```bash
source "$PLUGIN_ROOT/lib/state.sh"
source "$PLUGIN_ROOT/lib/deep-research.sh"
```

**Default bumps** (replace lines 25-26):
```bash
PROBE_S="${CW_DEEP_RESEARCH_PROBE_S:-900}"
STUCK_S="${CW_DEEP_RESEARCH_STUCK_S:-1800}"
RESCAN_EVERY_S="${CW_DEEP_RESEARCH_RESCAN_EVERY_S:-30}"
```

**Cursor restore** (replace lines 30-34):
```bash
CURSOR_FILE="$ART_DIR/troopers/$COMMANDER/liveness-cursor.txt"
RESCAN_CURSOR_FILE="$ART_DIR/troopers/$COMMANDER/liveness-rescan-emitted.txt"
mkdir -p "$(dirname "$CURSOR_FILE")"
touch "$RESCAN_CURSOR_FILE"

OFFSET=0
if [[ -s "$CURSOR_FILE" ]]; then
  prev=$(cat "$CURSOR_FILE")
  cur_size=$(wc -c < "$OUTBOX" 2>/dev/null || echo 0)
  if [[ "$prev" =~ ^[0-9]+$ ]] && (( prev <= cur_size )); then
    OFFSET=$prev
  else
    OFFSET=$cur_size
  fi
else
  OFFSET=$(wc -c < "$OUTBOX" 2>/dev/null || echo 0)
fi
printf '%d' "$OFFSET" > "$CURSOR_FILE"
LAST_RESCAN=0
```

**Phase check** (before mtime-liveness block, after byte-tail emit block):
```bash
phase=$(cw_deep_research_trooper_state_field "$ART_DIR" "$COMMANDER" phase 2>/dev/null || echo "")
if [[ "$phase" == "working" ]]; then
  # Existing mtime liveness check runs here
  ...
fi
```

**Periodic rescan block** (inserted at end of main loop, before `sleep 2`):
```bash
now=$(date +%s)
if (( now - LAST_RESCAN >= RESCAN_EVERY_S )); then
  if [[ -f "$OUTBOX" ]]; then
    line_num=0
    while IFS= read -r line; do
      line_num=$((line_num + 1))
      ev=$(printf '%s' "$line" | sed -n 's/.*"event":"\([^"]*\)".*/\1/p')
      case "$ev" in
        done|error|question)
          if ! grep -qE "^${line_num}\t${ev}\$" "$RESCAN_CURSOR_FILE"; then
            summary=$(printf '%s' "$line" | sed -n 's/.*"summary":"\([^"]*\)".*/\1/p')
            emit "$ev" "$summary (rescan)"
            printf '%d\t%s\n' "$line_num" "$ev" >> "$RESCAN_CURSOR_FILE"
          fi
          ;;
      esac
    done < "$OUTBOX"
  fi
  LAST_RESCAN=$now
fi
```

**Byte-cursor writeback** (after each successful tail-and-emit pass): `printf '%d' "$OFFSET" > "$CURSOR_FILE"`.

**Files touched:** `bin/deep-research-monitor.sh` only.

### `bin/deep-research-init.sh` — `--time-budget` + `--metric` flags

**`--time-budget=<value>`** — accepts `none`, `Nh`, `Ns` (raw seconds). Parsing:
```bash
case "$VAL" in
  none) BUDGET=none ;;
  *h)   BUDGET=$(( ${VAL%h} * 3600 )) ;;
  *s|[0-9]*) BUDGET=${VAL%s} ; [[ "$BUDGET" =~ ^[0-9]+$ ]] || { log_error "..."; exit 2; } ;;
  *) log_error "invalid --time-budget value: $VAL"; exit 2 ;;
esac
echo "$BUDGET" > "$ART_DIR/time-budget.txt"
date -u +%Y-%m-%dT%H:%M:%SZ > "$ART_DIR/session-start.txt"
```

**`--metric=<k1=v1,k2=v2,...>`** — comma-separated KV pairs. Required: `primary_metric`, `direction`. Optional: `min_acceptable`, `target`, `K_corroboration`, `hard_constraints`, `notes`. Parse, validate required keys, pipe to `cw_deep_research_format_metric_block`:
```bash
{ for kv in ${METRIC//,/ }; do printf '%s\n' "$kv"; done } \
  | cw_deep_research_format_metric_block > "$ART_DIR/metric.md"
```

The default `K_corroboration=1`, `plateau_window=5`, `plateau_threshold=0.01` apply when not specified (already handled by the helper).

**Flag parsing**: standard while-getopts pattern; positional `<topic>` remains the trailing arg. `--seed-from` already exists and is preserved unchanged.

### `bin/deep-research-abort.sh` (NEW, ~50 LoC)

```
Usage: bin/deep-research-abort.sh <topic> [<reason>]
Exit codes: 0=ok, 1=no active session, 2=usage error
```

Flow:
1. Source `lib/log.sh` + `lib/state.sh` + `lib/consult.sh` + `lib/deep-research.sh`.
2. Validate `$#` ∈ {1, 2}. Auto-prefix topic (same as #7 semantics).
3. Resolve `ART_DIR = $(cw_state_root)/state/$(cw_repo_hash)/$TOPIC/_deep-research`. Refuse rc=1 if absent.
4. `printf 'user-aborted via bin/deep-research-abort.sh at %s reason=%s\n' "$(date -u ...)" "${REASON:-unspecified}" > "$ART_DIR/halt.flag"`.
5. `"$PLUGIN_ROOT/bin/deep-research-finalize.sh" "$TOPIC"` — renders final landscape doc.
6. `"$PLUGIN_ROOT/bin/deep-research-teardown.sh" "$TOPIC"` — `--pairs`-mode batched teardown + archive.
7. Print TaskStop deferral hint with task IDs. The script reads `monitor-tasks.txt` **before** invoking teardown (step 6) so the task IDs are captured in memory even if teardown moves the file into archive.

**Slash-command surface:** no new directive in `commands/` for v0.32.0. Users invoke via Bash tool or shell. Promote to a directive in v0.33.0 only if dogfood proves the bash form clunky.

**Documentation caveats:**
- There may be 0-3 stale events sitting in the chat after abort, until the user types anything that triggers the resume handler's TaskStop pass.
- `deep-research-teardown.sh` kills trooper panes only; the user's conductor session pane (where `/clone-wars:deep-research` was invoked) is unaffected. Abort is safe to run from within the deep-research session.
