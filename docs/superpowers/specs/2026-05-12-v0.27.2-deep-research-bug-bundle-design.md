## 1. Problem

The v0.27.0 strict-dogfood run surfaced three latent issues plus one missing enhancement:

1. **BUG #4 — `experiment-send.sh` sed substitution corrupts multi-line briefs.** `bin/deep-research-experiment-send.sh:102` uses `sed -e "s|{{APPROACH_BRIEF}}|...|g"`; sed's `s` command terminates at the first unescaped newline in the replacement string. Multi-line `$APPROACH_BRIEF` (the realistic case for follow-up experiments — every dispatch in the session except exp-001/exp-002 hit this) produces a **0-byte `prompt.md`**. The sanity check at line 110 silently passes because grep finds no placeholders in an empty file. The trooper receives an empty inbox and spins on it.

2. **BUG #5 — troopers stall post-training before emitting `done`.** Observed in 2 of the 4 dispatches that completed normally (exp-001 + exp-004, both rex). After writing `result.json`, troopers paused 3–4 min to "verify" / "summarize" / "explore" before emitting `{event:"done"}`. The advisor's wait shim polls correctly during this gap; total wall time wasted: ~7 min across the session. Recovery prompts that included explicit "TERMINAL STEP — emit done IMMEDIATELY" framing got instant done events for exp-003/exp-005/exp-006.

3. **BUG #6 — wait shim's byte-offset is brittle against stray events.** When BUG #4 produced an empty inbox, keeli emitted a phantom `{event:"done","summary":"no substantive task in inbox"}`. The freshly-spawned exp-003 wait shim (with the BUG-#4 pre-stamped byte offset) caught the phantom done as if it were exp-003's done, returning rc=0 immediately. The actual exp-003 ack came 3 minutes later. The wait shim matches on `{event:"done"...}` but never verifies the summary references the expected `$EXP_ID`.

4. **P2 — Hardware information is implicit, not explicit.** The prompt template doesn't surface hardware info to the trooper. Troopers self-detect CUDA via `torch.cuda.is_available()` (correct behavior) but the conductor never makes the hardware contract explicit. For ML topics, this is friction: the advisor should know the L20 is available and tell the trooper so they can knowingly opt into AMP / `torch.compile` / larger batch. Also avoids silent CPU-training on no-GPU machines.

Plus one trivial doc-only fix:

5. **P3 — `cw_consult_timeout experiment` default 1800s is a SAFETY CAP, not a target.** The current comment is unclear about that distinction.

### 3.4 P2 — Hardware probe (both-mode)

**New lib helpers** in `lib/deep-research.sh`:

```bash
# cw_deep_research_hardware_probe <out-path>
# Writes either GPU rows or no-gpu marker. Atomic tmp+mv. rc=0 always.
cw_deep_research_hardware_probe() {
  local out="${1:-}"
  [[ -n "$out" ]] || { echo "cw_deep_research_hardware_probe: out-path required" >&2; return 2; }
  if command -v nvidia-smi >/dev/null 2>&1; then
    {
      printf 'detected_at\t%s\n' "$(date -u +%Y-%m-%dT%H:%M:%SZ)"
      nvidia-smi \
        --query-gpu=name,memory.total,memory.free,driver_version \
        --format=csv,noheader,nounits 2>/dev/null \
        | awk -F', ' '{ printf "gpu\t%s\t%s\t%s\t%s\n", $1, $2, $3, $4 }'
    } > "$out.tmp" && mv "$out.tmp" "$out"
  else
    printf 'detected_at\t%s\nno-gpu\n' "$(date -u +%Y-%m-%dT%H:%M:%SZ)" > "$out.tmp" \
      && mv "$out.tmp" "$out"
  fi
}

# cw_deep_research_hardware_diff_alert <baseline-path> <current-path>
# Emits ONE "ALERT:" line to stdout when memory.free dropped >50% on any
# GPU row. Otherwise no output. rc=0 always.
cw_deep_research_hardware_diff_alert() {
  local baseline="${1:-}" current="${2:-}"
  [[ -f "$baseline" && -f "$current" ]] || return 0
  awk -F'\t' -v base="$baseline" '
    BEGIN {
      while ((getline line < base) > 0) {
        split(line, f, "\t")
        if (f[1] == "gpu") { base_free[f[2]] = f[4] }
      }
      close(base)
    }
    $1 == "gpu" {
      name=$2; cur_free=$4
      b = base_free[name]
      if (b > 0 && cur_free < b * 0.5) {
        drop = int((1 - cur_free / b) * 100)
        printf "ALERT: gpu \047%s\047 memory.free %s -> %s MiB (-%d%%)\n", name, b, cur_free, drop
      }
    }
  ' "$current"
}
```

## 6. Out of scope (deferred to v0.28+)

- **Mid-session GPU saturation handling.** Diff alert tells the trooper memory dropped, but doesn't automatically reduce batch size. v0.28 could add a "compute pressure" field to the trooper prompt that triggers conservative defaults.

- **Resume after conductor crash.** If Claude Code dies mid-loop, the troopers survive but the conductor's task list is lost. v0.28 could persist conductor state to `_deep-research/conductor-state.json`.
