# v0.41.0 — Simplification Sweep

**Date:** 2026-05-16
**Branch target:** `feat/v0.41.0-simplification-sweep` (forked from `main` @ a6bb5a0)
**Type:** Refactor-only release. Zero behavioral change except where explicitly noted in Lane A.

## Motivation

A code-simplifier sweep over `bin/`, `lib/`, `hooks/`, and `tests/` after v0.40.0
landed surfaced 10 duplication candidates. Six are real, low-risk, and yield a
measurable LOC reduction with no behavioral change (or a single, explicit,
desirable one). Four are deferred — they were either misdiagnosed (silent
divergence that is actually intentional layered hardening), or the churn cost
exceeds the payoff at this point in the release cycle.

This spec locks the accepted subset, names the rejected items and why, and
groups the accepted work into commit-boundary lanes so the PR diff stays easy
to review one lane at a time.

## Scope — what's in

Six lanes, ~30 lines removed net, zero new tests required.

### Lane A — deep-research topic helpers

Seven `bin/deep-research-*.sh` scripts open-code a topic prefix check plus a
follow-on `cw_consult_topic_validate` call. The shape splits into two
*intentional* behavioral camps:

**Auto-prefix camp (4 sites)** — accept either bare slug or prefixed form,
normalize the variable, then validate. Used by user-invoked entry points
(`/clone-wars:deep-research-abort mnist` should work).

- `bin/deep-research-abort.sh:34`
- `bin/deep-research-refine.sh:25`
- `bin/deep-research-fresh-trooper.sh:25`
- `bin/deep-research-experiment-send.sh:50`

**Hard-error camp (3 sites)** — internal helpers; reject any bare slug. These
are invoked from other bin scripts that have already normalized.

- `bin/deep-research-teardown.sh:20`
- `bin/deep-research-consensus.sh:34`
- `bin/deep-research-score.sh:21`

**Add two helpers in `lib/deep-research.sh`** (mirroring the
`cw_consult_assert_*` shape at `lib/consult.sh:399-404`):

```bash
# cw_deep_research_normalize_topic <topic-var-name>
#   Auto-prefix bare slug with 'deep-research-' if missing, then validate.
#   Mutates the named variable. Exits 2 on validation failure.
cw_deep_research_normalize_topic() {
  local _var="$1"
  local _val="${!_var}"
  [[ "$_val" == deep-research-* ]] || _val="deep-research-$_val"
  cw_consult_topic_validate "$_val" || { log_error "invalid topic: $_val"; exit 2; }
  printf -v "$_var" '%s' "$_val"
}

# cw_deep_research_assert_topic <topic>
#   Require explicit 'deep-research-' prefix. Exits 2 on invalid topic.
cw_deep_research_assert_topic() {
  [[ "$1" == deep-research-* ]] \
    || { log_error "topic must start with 'deep-research-': $1"; exit 2; }
  cw_consult_topic_validate "$1" || { log_error "invalid topic: $1"; exit 2; }
}
```

The split is the **explicit, desirable behavioral change** for this lane —
*before* this refactor the two camps were two ad-hoc patterns; *after* they
become two named helpers. The behavior at each call site is preserved.

**Caller migration:**

- Auto-prefix sites collapse three lines to one (`cw_deep_research_normalize_topic TOPIC`).
- Hard-error sites collapse four lines to one (`cw_deep_research_assert_topic "$TOPIC"`).

**Estimated delta:** ~ -10 LOC.

### Lane B — meditate topic helper

Five `bin/meditate-*.sh` scripts repeat the identical hard-error block:

- `bin/meditate-research-send.sh:25`
- `bin/meditate-synth-final.sh:22`
- `bin/meditate-adversary-send.sh:22`
- `bin/meditate-teardown.sh:19`
- `bin/meditate-synth-preliminary.sh:26`

All five are byte-identical:

```bash
[[ "$TOPIC" == meditate-* ]] || { log_error "topic must start with 'meditate-': $TOPIC"; exit 2; }
```

**Add `cw_meditate_assert_topic` in `lib/meditate.sh`** (clean mirror of
`cw_consult_assert_topic`):

```bash
cw_meditate_assert_topic() {
  [[ "$1" == meditate-* ]] \
    || { log_error "topic must start with 'meditate-': $1"; exit 2; }
  cw_consult_topic_validate "$1" || { log_error "invalid topic: $1"; exit 2; }
```

Note we *add* the `cw_consult_topic_validate` step that the open-coded form
omits. This is a strict-monotonic tightening (rejects fewer inputs than it
accepts now, never the reverse), but worth flagging in the CHANGELOG.

**Estimated delta:** ~ -5 LOC.

### Lane E — commander helper (narrow)

Of the 5 open-coded commander-regex sites, only ONE can safely route through
`cw_consult_assert_commander`:

- `bin/deploy-wave-wait.sh:28` — `[[ "$COMMANDER" =~ ^[a-z0-9_-]+$ ]] || { log_error ...; exit 2; }`

That regex is byte-identical to the helper at `lib/consult.sh:402-404`.

The remaining four are **intentionally distinct hardening** and stay open-coded:

- `bin/spawn.sh:148` — adds `length ≤ 32` cap because the commander becomes
  both a tmux pane name and a state-dir path component. Migrating loses the cap.
- `bin/deep-research-{refine,fresh-trooper,experiment-send}.sh` — use the
  stricter `^[a-z][a-z0-9-]*$` (leading letter required, no underscores) to
  match the convention in `config/commanders.yaml`. The helper is more
  permissive and would accept commander names that the rest of deep-research
  would reject downstream.

**Migration:** one-line swap in `deploy-wave-wait.sh`. No new helper.

**Estimated delta:** ~ -1 LOC.

### Lane F — metric.md awk collapse

`lib/deep-research.sh:402-408` spawns 6 awk processes to read 7 fields from
`metric.md`. The pattern:

```bash
min_op=$(awk -F':\\*\\* ' '/^\*\*min_acceptable:/{print $2}' "$m" | awk '{print $1}')
min_val=$(awk -F':\\*\\* ' '/^\*\*min_acceptable:/{print $2}' "$m" | awk '{$1=""; sub(/^ /, ""); print}')
# ... same shape for target:
K_req=$(awk -F':\\*\\* ' '/^\*\*K_corroboration:/{print $2}' "$m" | tr -d ' ')
plateau_window=$(awk -F':\\*\\* ' '/^\*\*plateau_window:/{print $2}' "$m" | tr -d ' ')
plateau_threshold=$(awk -F':\\*\\* ' '/^\*\*plateau_threshold:/{print $2}' "$m" | tr -d ' ')
```

**Collapse to a single awk pass** that reads the file once and emits all 7
shell-eval-ready assignments:

```bash
eval "$(awk -F':\\*\\* ' '
  /^\*\*min_acceptable:/   { split($2, a, " "); printf "min_op=%s; min_val=%s\n", a[1], substr($2, length(a[1])+2) }
  /^\*\*target:/           { split($2, a, " "); printf "tgt_op=%s; tgt_val=%s\n", a[1], substr($2, length(a[1])+2) }
  /^\*\*K_corroboration:/  { gsub(/ /,"",$2); printf "K_req=%s\n", $2 }
  /^\*\*plateau_window:/   { gsub(/ /,"",$2); printf "plateau_window=%s\n", $2 }
  /^\*\*plateau_threshold:/{ gsub(/ /,"",$2); printf "plateau_threshold=%s\n", $2 }
' "$m")"
```

The `eval` is safe because `$m` comes from a controlled source (the topic's
own metric.md, written by the directive); values are space-stripped or
known-quoted. If preferred, a `read`-loop variant is equally short.

**Estimated delta:** ~ -5 LOC, ~6 fewer subshells per `check_completion` call
(once per turn — perf is incidental, clarity is the point).

## Scope — what's out (and why)

For each deferred item, the *why* matters because we want future sweeps to
not waste time re-proposing the same idea.

### Out: Item 5 — JSON event extraction helper

`bin/list.sh:66` and `bin/deploy-turn-send.sh:39` both use sed pipelines to
extract a JSON field, but:

- They extract DIFFERENT fields (`event` vs `state`).
- They read DIFFERENT files (outbox.jsonl vs status.json).
- `list.sh` anchors at line start (`^{"event":"`) for defense against
  progress-note text shadowing; `deploy-turn-send.sh` does not.

The shared shape is too thin. A `cw_extract_json_field` would either replicate
both modes (anchored, unanchored) at the caller, or pick one and silently
change the other's behavior. Two callers, no plurality.

### Out: Item 7 — consult-walk-assemble duplicate pipeline

`bin/consult-walk-assemble.sh:51` and `:55` both run
`grep -v '^#' | awk -F'\t' 'NF{print $1}'`, but they sit in distinct
`case`-branches and feed different post-processors. Lifting to a pre-case
array adds dead work in the path that takes neither branch, and makes each
case-branch less self-contained. The duplication is 2 visible lines —
within readable tolerance.

### Out: Item 8 — test scaffold extraction

232 test files; the simplifier estimated ~80 affected. CLAUDE.md explicitly
cautions against drive-by refactoring at this scale ("don't drive-by-refactor
adjacent code"). The mechanical change is safe but the PR review burden is
enormous. If wanted, this is its own dedicated sweep, not a piggyback on
v0.41.0.

### Out: Item 10 — PLUGIN_ROOT bootstrap × 46

Each `bin/` script then sources a different lib subset, so a shared
bootstrap script saves at most one line per file (the `PLUGIN_ROOT=` line)
and adds an indirection layer. The simplifier itself flagged this as low
payoff; defer indefinitely.

### Out: Watchlist — all five

- **consult.sh 906-LOC split.** Five callers `source lib/consult.sh`
  expecting all symbols. Splitting requires a coordinated migration not
  justified by current pain.
- **Teardown unification.** TSV column counts genuinely differ (consult 2-col,
  deploy 3-col, deep-research 1-col). Unifying requires unifying the TSV
  format first.
- **`set -- "${NEW_ARGS[@]:-}"` edge case.** Worth a separate investigation
  if it ever causes a real bug; cosmetic until then.
- **Shared arg-parser helper.** Bash arg parsers tend to be either too magic
  or barely shorter than the open-coded version.
- **`cw_consult_eligible_providers` inline.** One-line wrapper; either
  direction is fine; status quo is readable.

### Out: Item 4 — spawn.sh + 3 deep-research regex sites (kept open-coded)

Already covered in Lane E above. Calling out explicitly so future sweeps see
the rationale: **the divergent regexes are layered intentional hardening,
not accidental drift.**

behavioral change (the one exception — Lane B adds a `cw_consult_topic_validate`

the test.
