# v0.25.0 — `/clone-wars:meditate` command design

**Status:** approved 2026-05-11. Implementation plan to follow.

**Goal:** Add a new user-facing slash command, `/clone-wars:meditate`, for deep
multi-aspect exploration of hard topics — SOTA architecture surveys, deep
reference research, multi-angle thinking on contested problems. Produces a
tradeoff-explicit landscape doc with a directional Conclusion, intended as a
hand-off seed for `/clone-wars:consult` (which then formalizes the findings
into a buildable design spec).

**Architecture (one paragraph):** /clone-wars:meditate reuses the v0.24.0
consult spawn/dispatch/wait infrastructure for the research phase, drops
consult's verify round entirely, adds a literature-review parallel track
that auto-detects on ML/SOTA keywords, runs preliminary synthesis on
Yoda, gates a confidence check before adversary dispatch, runs an
adversarial-review round across all N troopers in parallel against the
preliminary synthesis (re-using the same panes via `tmux respawn-pane`),
and writes a final landscape doc with a tradeoff matrix + adversary
critiques + directional Conclusion.

**Tech stack:** pure bash 4.2+, tmux 3.0+, file-based IPC. New helpers in
`lib/meditate.sh`; six new `bin/meditate-*.sh` scripts; three new prompt
templates under `config/prompt-templates/meditate/`. Reuses
`lib/consult-wait.sh`, `lib/tmux.sh`, `lib/state.sh`, `lib/contracts.sh`,
`lib/commanders.sh`, `bin/spawn.sh`, `bin/preflight-layout.sh`,
`bin/teardown.sh --pairs`.

---

## 1. Why this exists

`/clone-wars:consult` produces a *deploy-audit-passing 6-section design doc*
— it forces a buildable artifact. That output shape is wrong for:

- "Explore SOTA continuous-batching schedulers"
- "Think about transformer-vs-mamba trade-offs from multiple aspects"
- "Deep reference research on logical replication patterns"

These topics call for *understanding without commitment* — multiple aspects
surfaced, contested findings preserved, blind spots actively probed. Forcing
them through consult collapses interesting tension into a single recommended
approach prematurely.

`/forcevision` (personal command, not in this plugin) covers some of this
ground with dual Claude+Codex research and cross-evaluation, but:

- It targets 2 models (Claude+Codex via companion) — meditate uses the
  plugin's 2–3 trooper roster (tmux panes the user can watch live).
- It synthesizes to a recommendation — meditate intentionally produces a
  tradeoff matrix with a directional (not prescriptive) Conclusion.
- It uses cross-evaluation (each side grades the other's research) —
  meditate uses adversarial-review (post-synthesis challenge of the
  combined landscape).

`/clone-wars:meditate` fills the gap: clone-wars-native, N-trooper, adversary-
gated, tradeoff-explicit, consult-feeding.

**Intended workflow:**

```
/clone-wars:meditate <topic>           → landscape-<date>-<slug>.md
                                              ↓ user reviews
/clone-wars:consult <follow-up topic>  → 6-section design doc
                                              ↓
/clone-wars:deploy <design doc>        → trooper implementation
```

## 2. User-facing surface

### 2.1 Command

`/clone-wars:meditate <topic>` — research a topic across all active troopers,
optionally enrich with peer-reviewed literature, synthesize a tradeoff-
explicit landscape, then adversarially challenge it before writing the final
doc with Conclusion.

### 2.2 Flags

| Flag | Meaning | Default |
|---|---|---|
| `--lit` | Force literature-review track ON | (auto-detected) |
| `--no-lit` | Force literature-review track OFF | (auto-detected) |

No `--use-force`, no `--no-adversary`, no fast-path flag. Meditate is
invoked deliberately for hard topics; Yoda-solo defeats the point.

### 2.3 Trigger phrases (for natural-language routing)

The command should be invoked when the user says any of:

- "explore SOTA …"
- "deep think about …"
- "research from multiple aspects …"
- "find new architectures for …"
- "deep reference research on …"
- "think through tradeoffs of …"
- "survey the landscape of …"
- "meditate on …"

Phrases that route to `/clone-wars:consult` instead (because they require
a buildable spec):

- "design …"
- "compare X vs Y for decision"
- "should we adopt …"

The line is fuzzy on purpose — the meditate → consult chain is the
intended path when both apply.

## 3. Pipeline

10 phases (numbered 0–10 to leave room between confidence-gate sub-phases).

### Phase 0 — args-file + init + roster load

Same shape as consult's Step 0. Differences:

- Resolves `--lit` / `--no-lit` flags (new `cw_meditate_parse_lit_flag` in
  `lib/meditate.sh`). Strips flag tokens from `$ARG_RAW` before topic
  validation.
- Invokes `bin/meditate-init.sh "$ARG_RAW"` which:
  - Mirrors `bin/consult-init.sh` for slug/topic-dir/state setup
  - Writes `_meditate/topic.txt` (with date prefix and slug)
  - Writes `_meditate/troopers.txt` from `providers-active.txt` (falls
    back to `providers-available.txt` exactly like consult-init does)
  - Returns the canonical topic string on stdout
- Computes `$REPO_HASH`, `$TOPIC_DIR`, loads `TROOPERS[]` array

### Phase 1 — Literature auto-detect

New routing block in `commands/meditate.md` Step 1 (no separate bin
script — Yoda reads the topic and decides).

**Decision logic:**

1. If `--lit` was passed → ON
2. If `--no-lit` was passed → OFF
3. Otherwise auto-detect via keyword scan. ON if any of these tokens
   appear case-insensitively in topic:
   - ML/method: `loss`, `embedding`, `network`, `model`, `architecture`,
     `training`, `optimizer`, `scheduler`, `transformer`, `mamba`,
     `attention`, `regularization`, `augmentation`, `fine-tune`
   - SOTA: `SOTA`, `state-of-the-art`, `benchmark`, `paper`, `arxiv`
   - Methods: `algorithm`, `inference`, `quantization`, `distillation`,
     `pruning`
4. Write decision to `_meditate/lit-track.txt` (`ON` or `OFF` + reason).

The exact keyword list lives in `lib/meditate.sh`'s
`cw_meditate_lit_keywords` constant so it can be tuned over time without
touching the directive.

### Phase 2 — Parallel spawn (consult-style with rollback)

Reuse v0.24.0's spawn pipeline byte-equal:

- `bin/preflight-layout.sh --art-dir _meditate "$TOPIC_DIR"` allocates
  N panes off Yoda's pane.
- N parallel `bin/spawn.sh --target-pane <id> --preflight-art-dir
  "$TOPIC_DIR/_meditate" <cmdr> <provider> "$CONSULT_TOPIC"` calls.
- Same auto-retry-once rollback semantics as consult Step 3
  (codex cold-start absorption).

The only new param is `--art-dir _meditate` to `preflight-layout.sh` —
this flag already exists from v0.20.0 deploy work (`bin/preflight-layout.sh
--art-dir <subdir>`). Reused verbatim.

### Phase 3 — Research dispatch + optional literature track

N parallel Bash tool calls:

- `bin/meditate-research-send.sh "$CONSULT_TOPIC" <cmdr> <provider>` ×N

If literature track ON, an (N+1)th parallel call:

- `Skill(skill: "literature-review", args: "<topic>... Save to:
  $TOPIC_DIR/_meditate/literature-review.md")`

The meditate research prompt template (`config/prompt-templates/meditate/
research.md`) differs from consult's research.md in three ways:

1. Asks for SOTA framing: "list current SOTA approaches by name and link
   to evidence."
2. Adds "Independent Discovery" section requirement: every trooper must
   list ≥3 files/URLs they opened beyond what Yoda's identity-prompt
   suggested (anti-correlated-blind-spots guard from forcevision §2a-post).
3. Drops the "must produce a recommendation" framing — research findings
   are observations, not advocacy.

### Phase 4 — Research wait

Reuse `cw_consult_wait research <topic> <cmdr> <provider>` from
`lib/consult-wait.sh` byte-equal. No code change.

Optional: poll for `literature-review.md` existence if lit track ON (skip
silently if literature-review skill fails — log warn, continue).

### Phase 5 — Preliminary synthesis (Yoda)

New `bin/meditate-synth-preliminary.sh "$CONSULT_TOPIC"` produces
`_meditate/landscape-draft.md` with these sections (NO Conclusion yet):

```markdown
## Topic
<verbatim user topic>

## Approaches
1. <Approach A> — <one-line summary from synthesis>
2. <Approach B> — ...
3. ...

## Tradeoff matrix
| Priority    | Best fit | Reason                        |
|-------------|----------|-------------------------------|
| <criterion> | <approach> | <why, with citation>        |
| ...         | ...      | ...                           |

## Findings by trooper
### Rex (codex) findings
<digest of rex's findings.md>

### Cody (claude) findings
<digest of cody's findings.md>

### Wolffe (opencode) findings
<digest of wolffe's findings.md>

## Open questions
- ...
- ...

## Citations
- ...
```

The bin script is a thin wrapper — Yoda does the actual synthesis with
the Edit/Write tools, the script just sets paths and validates that
inputs (findings.md ×N + optional literature-review.md) exist.

### Phase 5.5 — Confidence gate

Yoda evaluates 5 signals against `landscape-draft.md` + findings:

| # | Signal | How to check |
|---|---|---|
| S1 | ≥N-1 troopers converged on same top approach | Grep top-approach name across `findings-*.md` |
| S2 | Every claim in top finding has ≥2 citations across troopers | Cross-grep cited files/URLs in findings-*.md |
| S3 | Zero CONTESTED markers in preliminary synth | Grep `landscape-draft.md` for `CONTESTED` |
| S4 | Every approach in matrix has file/URL/paper backing | Grep `Reason` column for `://` or `/` path or paper-id pattern |
| S5 | At least one trooper acknowledged uncertainty | Grep findings-*.md for "uncertain", "unclear", "depends on", "could not determine" |

**All 5 must hold** to offer skip. If any fails → adversary runs
unconditionally (no prompt, save the round trip).

If all 5 hold, fire one `AskUserQuestion`:

```
question: "Yoda is confident in the preliminary findings. Skip adversary
           and write Conclusion now?"
options:
  1. (recommended) "Run adversary (default — safer)"
  2. "Skip adversary, write Conclusion now"
```

Default is **run adversary** — user must actively opt out. Matches the
rule "if user disagrees for jump, it still goes to adversary."

Record the gate outcome in `_meditate/adversary-skip.txt`:

```
timestamp: <ISO-8601>
signals_passed: S1,S2,S3,S4,S5
user_decision: skip|continue
```

If decision == skip → jump to Phase 8. If continue → Phase 6.

### Phase 6 — Adversary dispatch

New `bin/meditate-adversary-send.sh "$CONSULT_TOPIC" <cmdr> <provider>`
×N parallel Bash calls. Each:

1. Reads `_meditate/landscape-draft.md`
2. Builds an inbox payload from `config/prompt-templates/meditate/
   adversary.md` template
3. Writes inbox.md via `cw_inbox_write`
4. Nudges the existing pane (reuses spawn-time pane via `cw_pane_send`
   — no respawn needed; troopers are still running their TUIs)
5. Records adversary-OFFSET in `_meditate/adversary-<cmdr>.txt`

The adversary prompt template:

```
You are now playing adversary against a synthesized landscape doc that
was built from your earlier research findings. Your job is to break
confidence in the synthesis — not to validate it.

Default to skepticism. Look for:
- Approaches that were missed or wrongly excluded
- Tradeoff matrix rows where the "Best fit" assignment is wrong
- Citations that don't actually support the claim
- Convergent findings that may share a correlated blind spot
- Frames the synthesis adopted that exclude valid alternative frames

Report only material findings. A finding should answer:
1. What is the weakness in the synthesis?
2. Why is that synthesis claim vulnerable?
3. What concrete change to the landscape doc would reduce the risk?

Write your output to: {{OUT_PATH}}
End with: END_OF_INSTRUCTION

The synthesis to challenge:
<inline copy of landscape-draft.md>
```

The output path is `_meditate/adversary-<cmdr>.md`. Each trooper writes
independently.

### Phase 7 — Adversary wait

Add `adversary` as a third kind to `cw_consult_wait` in
`lib/consult-wait.sh`:

```bash
case "$kind" in
  research) ... ;;
  verify)   ... ;;
  adversary)
    state_key="AS"
    timeout_env_var="CW_MEDITATE_ADVERSARY_TIMEOUT_OVERRIDE"
    timeout_key="adversary"
    handler_phase="adversary"
    ;;
  ...
esac
```

`cw_consult_timeout adversary` returns 600s (10 min) by default —
adversary work is shorter than research because the trooper is reviewing
a single document, not exploring a codebase.

The done-event handler checks `[[ -s "$adversary_file" ]]` analogously
to verify's done handler.

(Naming note: the lib file stays `lib/consult-wait.sh` — the function is
already kind-parameterized and used by both commands. Renaming it to
`lib/wait.sh` is a v0.25 nice-to-have but not required for this release.)

### Phase 8 — Final synthesis (Yoda)

New `bin/meditate-synth-final.sh "$CONSULT_TOPIC"` produces the canonical
output `_meditate/landscape-<YYYY-MM-DD>-<slug>.md`. Sections:

```markdown
## Topic
<verbatim user topic>

## Approaches
<same as draft>

## Tradeoff matrix
<same as draft, possibly revised by adversary critiques>

## Adversary critiques
- **Rex (codex):** <one paragraph summary from adversary-rex.md>
- **Cody (claude):** ...
- **Wolffe (opencode):** ...

(If adversary skipped:)
> _Adversary phase skipped after Yoda's confidence gate passed (signals:
> S1,S2,S3,S4,S5) and user accepted skip. Findings are single-pass —
> no post-synthesis challenge was performed._

## Open questions
<merged from draft + new questions surfaced by adversary>

## Conclusion (for /clone-wars:consult hand-off)
<Yoda's directional take. NOT a buildable recommendation.>

Body should:
- State the strongest approach with explicit caveats
- List adversary-surfaced weaknesses the design phase must address
- Suggest a concrete /clone-wars:consult topic to formalize the next step

Example body:
"Approach A looks strongest for codebases prioritizing latency over cost,
given <reasons>. Adversary critiques flag <X> which the design phase
should address head-on. Suggested consult invocation:

    /clone-wars:consult Design <X> using approach A, with
    mitigations for <adversary-flagged-issue>

If your priorities shift toward cost, see the matrix row for cost-
sensitivity (Approach C)."

## Citations
- paper://...
- file://...
- url://...
```

The Conclusion is the **only** place Yoda gives a directional take. The
rest of the doc is descriptive. The Conclusion is also where the
adversary-skip note lives if applicable.

### Phase 9 — Teardown panes + archive

Reuse `bin/teardown.sh --pairs <topic> <cmdr1> <cmdr2> ...` byte-equal
(the v0.20.5 N-pane teardown with single graceful banner). Wrap in
`bin/meditate-teardown.sh` to also clean up `_meditate/preflight-panes.txt`
orphans via `cw_preflight_kill_orphans` (the v0.24.0 lib helper).

Archive: state dir moved from `state/<repo-hash>/<topic>/_meditate/` to
`archive/<repo-hash>/<topic>/_meditate-<timestamp>/`.

### Phase 10 — Present final doc + suggested next step

Yoda prints to the user:

```
Meditation complete. Landscape doc:
  <full path to landscape-<date>-<slug>.md>

Suggested next step:
  /clone-wars:consult <topic from Conclusion's suggested-invocation>

Or hand-edit the topic if you want to investigate a different angle.
```

No automatic invocation. User reviews the landscape doc, decides whether
to proceed to consult, and invokes it explicitly.

## 5. Confidence gate details

This is the only genuinely new mechanism not present in either consult
or forcevision. Document it precisely.

### 5.1 Why the gate exists

Adversary phase costs ~5–8 min (one round-trip × N troopers in parallel,
gated by codex's slower wall-clock). For topics where preliminary
synthesis converged strongly with good citations, that time is mostly
re-confirmation — adversary attacks a synthesis everyone already agrees
on. The gate lets the user skip in clear-convergence cases without
losing the option to run adversary.

### 5.2 Why all 5 signals must hold (not "3 of 5")

Strong convergence is a conjunction, not a disjunction:

- S1 alone (everyone agreed on top approach) is suspicious without
  citation backing — could be correlated blind spots.
- S2 alone (good citations) is necessary but not sufficient — could
  still be one perspective dominating.
- S3 alone (no CONTESTED markers) could mean Yoda just didn't surface
  contestation, not that none exists.
- S4 alone (matrix is well-cited) doesn't preclude missing approaches.
- S5 alone (uncertainty acknowledged) is necessary because the alternative
  ("100% agreement everywhere") is a known signal of correlated
  blind spots, NOT genuine confidence.

Requiring all 5 means: independent troopers converged with their own
evidence, AND at least one acknowledged the limits of their own analysis,
AND the synthesis surfaced no live disagreements, AND every matrix entry
is grounded. That's actually-confident.

### 5.3 Why the default option is "Run adversary" even when the gate offers skip

User-facing safety: the gate firing means "Yoda thinks skip is OK." But
Yoda is judging its own synthesis — the very thing adversary exists to
challenge. So the default option in the AskUserQuestion stays "run
adversary." User must actively pick "skip" to take the shortcut.

This matches the rule the user explicitly stated: "if user disagree for
jump, it still goes to adversary."

### 5.4 What the audit log captures

`_meditate/adversary-skip.txt`:

```
timestamp: 2026-05-11T20:34:11Z
signals_passed: S1=true S2=true S3=true S4=true S5=true
user_decision: skip
```

Or if some signal failed (no prompt fires; signals still logged for
later analysis):

```
timestamp: 2026-05-11T20:34:11Z
signals_passed: S1=true S2=false S3=true S4=true S5=true
user_decision: not-offered
```

The final landscape doc grep-reads this and stamps the Conclusion section
appropriately.

## 6. Prompt template details

### 6.1 `config/prompt-templates/meditate/research.md`

Mostly mirrors `config/prompt-templates/consult/research.md` v0.24.0 but:

- Drops the "produce a recommendation" framing
- Adds "Independent Discovery" section requirement (≥3 files/URLs/papers
  opened beyond Yoda's identity-prompt suggestions)
- Asks for SOTA framing where applicable
- Asks for inline citations using `[name](url-or-path)` markdown links
- Requires `findings-<cmdr>.md` at `{{WRITE_TO}}` placeholder

Placeholders: `{{TOPIC}}`, `{{WRITE_TO}}` (matches consult's research.md).

### 6.2 `config/prompt-templates/meditate/adversary.md`

Built on the `codex:adversarial-review` pattern (skepticism, break-
confidence, material findings only). Adapted to landscape-doc target
instead of code-change target:

- Attack surface: synthesis blind spots, missing approaches, miscited
  matrix rows, correlated agreement
- Output: structured markdown findings with file/URL backing
- No JSON contract (codex-adversarial-review uses JSON; meditate uses
  markdown for downstream Yoda merge)

Placeholders: `{{LANDSCAPE_DRAFT}}` (inline copy of the draft),
`{{OUT_PATH}}` (`_meditate/adversary-<cmdr>.md`).

### 6.3 `config/prompt-templates/meditate/landscape-skeleton.md`

Static template Yoda reads at Phase 5 / Phase 8 to know the section
structure. NOT a prompt — it's a structural reference for the bin scripts
to validate output shape (grep for required section headings).

Sections (locked):

```
## Topic
## Approaches
## Tradeoff matrix
## Findings by trooper        (Phase 5 only — dropped from Phase 8)
## Adversary critiques        (Phase 8 only — absent from Phase 5)
## Open questions
## Conclusion                 (Phase 8 only — absent from Phase 5)
## Citations
```

`bin/meditate-synth-preliminary.sh` validates draft has the Phase-5
section set. `bin/meditate-synth-final.sh` validates final has the
Phase-8 section set. Either failing → exit 1 with the missing section
name on stderr.

## 9. Out of scope (deliberate)

- **Multi-round adversary** — adversary runs once, Yoda merges once. No
  back-and-forth between Yoda and troopers. Add later if needed.
- **Drilldown** — no Step-13-style drill-into-section iteration. The
  meditate → consult hand-off is the iteration mechanism.
- **Multi-repo meditate** — single-context only (matches v0.14.0 consult
  policy). Multi-repo SOTA exploration would require a different design
  doc.
- **Fast-path (Yoda solo)** — explicitly rejected. Meditate is for hard
  topics; Yoda-solo is `/forcevision`'s job.
- **`--no-adversary` flag** — explicitly rejected. User can decline the
  adversary skip but cannot disable adversary upfront. Forces the
  default path to be the rigorous one.
- **Auto-invoking `/clone-wars:consult` at Phase 10** — user reviews,
  user invokes. No silent chaining.
- **Renaming `lib/consult-wait.sh` to `lib/wait.sh`** — meditate adds an
  `adversary` kind to the existing function; renaming is a nice-to-have
  but breaks backwards compat for no functional benefit. Defer to v0.26
  if it ever matters.
