# v0.20.2 Stale-String Fixes Design

**Date:** 2026-05-09
**Plugin version bump:** 0.20.1 → 0.20.2
**Branch:** `fix/v0.20.2-stale-string-fixes`
**Source:** plugin-validator pass on v0.20.1 (commit f1ebd94) + secondary sweep
**Scope:** stale-string sweep — removes drift left over from v0.12 (synthesis.md removal) and v0.17 (/spec deletion), plus one P0 runtime break in `bin/consult-drilldown.sh`

## Problem

Plugin-validator's review of v0.20.1 surfaced three stale-string defects, and a secondary sweep during brainstorming surfaced two more in the same class. All five trace to the same root cause: when synthesis.md was removed in v0.12 and `/spec` was removed in v0.17, the cleanup missed several files.

1. **P0 — drill-deeper is broken on every consult since v0.12.** `bin/consult-drilldown.sh:78-79` reads `_consult/synthesis.md`, which was removed in v0.12 (sibling `bin/consult-synthesize.sh:16` literally comments "Does NOT write the legacy _consult/synthesis.md (removed in v0.12)"). Step 13 of `commands/consult.md` aborts with `synthesis.md not found` whenever a user accepts the drill prompt.
2. **P0 — Pattern numbering hole.** `commands/consult.md` has `### Pattern 1` (line 1100), `### Pattern 3` (line 1124), `### Pattern 4` (line 1151), with no Pattern 2. Git history (`git log -S 'Pattern 2' -- commands/consult.md`) returns zero hits — Pattern 2 was never authored, so this is a long-standing typo, not lost content.
3. **P1 — `/spec` references in `lib/consult.sh` (lines 460, 484, 1184).** `/spec` was deleted in v0.17. Lines 460 and 484 are user-visible: their strings are emitted into design-doc seed prose by `cw_consult_synthesize`, so they leak into users' final committed design docs as "Master Yoda fills this in during the /spec walk". Line 1184 is a code comment.
4. **P1 — drill prompt template still says "synthesis".** `config/prompt-templates/consult/drilldown.md:4` reads `Read the synthesis you produced: {{SYN}}` — sent verbatim to every trooper on every drill round. The trooper now actually reads a design-doc, not a synthesis.
5. **P2 — helper-signature comment uses stale name.** `lib/consult-prompts.sh:56` documents `cw_consult_design_doc_drilldown_prompt`'s second arg as `<synthesis-path>` though the value flowing through it is the design-doc path.

All five confirmed by direct grep on the live tree.

## Goal

After v0.20.2 ships:

- A user invoking `/clone-wars:consult` and accepting the Step 13 "Drill deeper" prompt successfully spawns a drill round (no `synthesis.md not found` abort).
- The directive at `commands/consult.md` Step 13 owns deriving and passing the design-doc path to `bin/consult-drilldown.sh` — script no longer reaches into `$TOPIC_DIR/_consult/` to find a source doc.
- Pattern numbering in `commands/consult.md` is contiguous: 1, 2, 3.
- No `/spec ` substring (with trailing space, to skip `/specs/` directory-path matches) appears anywhere in `lib/consult.sh`.
- Drill prompts sent to troopers no longer say "synthesis."
- `bash tests/run.sh` stays green; a new static-wiring test locks in all six fixes.
- `.claude-plugin/plugin.json` and `.claude-plugin/marketplace.json` show `0.20.2`.
- Single-repo and multi-repo `/clone-wars:deploy`, all of `/clone-wars:medic`, and the rest of `/clone-wars:consult` are byte-equal to v0.20.1.

## Architecture

Five surgical edits across four files (plus version bump + new test). Each fix is local and reversible by inversion. The drilldown contract change touches three files together — script, directive, and helper-signature comment — but all three are coordinated under a single concept.

Bottom-up apply order minimizes test churn:

1. Drill template prose update (Fix 4) — pure string, safe anytime
2. Helper signature comment update (Fix 5) — pure comment, safe anytime
3. Script signature change (Fix 1a) — adds new positional arg at position 5
4. Directive caller update (Fix 1b) — passes the new arg
5. Pattern renumber (Fix 2) — pure markdown heading swap
6. `/spec` purge in lib/consult.sh (Fix 3) — pure string swap
7. Version bump (Fix 6)
8. Static-wiring test (Fix 7) — locks all of the above

## Components

### Fix 1a — `bin/consult-drilldown.sh`: add `<design-doc-path>` at position 5

The script's actual current signature has 6/7/8/9 positional args (validator misquoted this in their report; verified by reading the script):

```
Usage: bin/consult-drilldown.sh <topic> <section-title> <dd-dir> <focus> \
    <commander1> <model1> [<commander2> <model2>] [<subproject>]
```

New signature inserts `<design-doc-path>` at position 5, between `<focus>` and `<commander1>`. This keeps trooper args clustered at the end and the optional `[<subproject>]` last:

```
Usage: bin/consult-drilldown.sh <topic> <section-title> <dd-dir> <focus> \
    <design-doc-path> <commander1> <model1> [<commander2> <model2>] [<subproject>]
```

Concrete edits to `bin/consult-drilldown.sh`:

- Lines 4-22 (usage block + examples): rewrite all four example invocations to include the new arg in position 5.
- Line 43 (`usage()` echo): update.
- Line 51 (arg-count guard): `[[ $# -eq 6 || $# -eq 7 || $# -eq 8 || $# -eq 9 ]]` → `[[ $# -eq 7 || $# -eq 8 || $# -eq 9 || $# -eq 10 ]]`.
- Lines 53-67 (positional unpack): insert `DESIGN_DOC="$5"` and renumber the rest:
  ```bash
  TOPIC="$1"
  TITLE="$2"
  DD_DIR="$3"
  FOCUS="$4"
  DESIGN_DOC="$5"
  COMMANDER1="$6"
  MODEL1="$7"
  COMMANDER2=""
  MODEL2=""
  SUBPROJECT=""
  case "$#" in
    7) ;;
    8) SUBPROJECT="$8" ;;
    9) COMMANDER2="$8"; MODEL2="$9" ;;
    10) COMMANDER2="$8"; MODEL2="$9"; SUBPROJECT="${10}" ;;
  esac
  ```
- Lines 77-79 (`SYNTHESIS=...synthesis.md` block): replace with:
  ```bash
  [[ -f "$DESIGN_DOC" ]] || { log_error "design-doc not found: $DESIGN_DOC"; exit 2; }
  ```
  Drop the `TOPIC_DIR=...` line entirely (no longer needed since the path is now an arg).
- Line 97 (`cw_consult_design_doc_drilldown_prompt` call): replace `"$SYNTHESIS"` with `"$DESIGN_DOC"`.

### Fix 1b — `commands/consult.md` Step 13: derive + pass design-doc path

Step 13's two parallel `Bash` tool calls (the K=2 + K=1 fan-out for N=3 roster) both invoke `consult-drilldown.sh`. Both need the new arg.

Add a shared computation block immediately before Step 13's drill block (or at the top of Step 13):

```bash
SLUG="${CONSULT_TOPIC#consult-}"
DESIGN_DOC=$(cw_consult_design_doc_canonical_path "$TOPIC_DIR/_consult" "$SLUG")
[[ -f "$DESIGN_DOC" ]] || { log_error "no design-doc at $DESIGN_DOC; cannot drill"; exit 1; }
```

The `cw_consult_design_doc_canonical_path` helper (lib/consult.sh:1185) returns `<art_dir>/design-doc/<TODAY-UTC>-<slug>-design.md`. This matches the path written by `bin/consult-walk-assemble.sh:36` byte-for-byte within the same UTC day, which is the normal drill cadence (drill happens immediately after assemble in Step 12).

Then the existing drill calls gain `"$DESIGN_DOC"` between `$DRILL_FOCUS` and the trooper roster:

```bash
"$CLAUDE_PLUGIN_ROOT/bin/consult-drilldown.sh" \
  "$CONSULT_TOPIC" "$DRILL_TOPIC" "$DRILL_DIR" "$DRILL_FOCUS" \
  "$DESIGN_DOC" \
  rex codex cody claude
```

All four call-shape examples in Step 13 (1-trooper, 2-trooper, 3-trooper K=2 / 3-trooper K=1, plus retry/different-trooper paths) need this new positional. Sweep with `grep -n 'consult-drilldown.sh' commands/consult.md` post-edit and verify every invocation includes `$DESIGN_DOC`.

### Fix 7 — Static-wiring test

**New file: `tests/test_v0_20_2_static_wiring.sh`**

Locks all surface invariants. Pure file-content asserts; no tmux / no spawning, so it runs in any CI environment:

```bash
#!/usr/bin/env bash
# v0.20.2 stale-string fixes — static wiring assertions.
set -euo pipefail
cd "$(dirname "$0")/.."
source tests/lib/assert.sh

# Fix 1a — consult-drilldown.sh drops synthesis.md, takes design-doc-path arg
drill_src=$(cat bin/consult-drilldown.sh)
[[ "$drill_src" != *'synthesis.md'* ]] || { echo "FAIL: drilldown still references synthesis.md"; exit 1; }
[[ "$drill_src" == *'DESIGN_DOC='* ]]   || { echo "FAIL: drilldown missing DESIGN_DOC unpack"; exit 1; }
[[ "$drill_src" == *'<design-doc-path>'* ]] || { echo "FAIL: drilldown usage missing <design-doc-path>"; exit 1; }

# Fix 1a — arg-count guard updated to 7|8|9|10
[[ "$drill_src" == *'$# -eq 7'* ]]  || { echo "FAIL: drilldown arg-count guard not bumped to 7+"; exit 1; }
[[ "$drill_src" == *'$# -eq 10'* ]] || { echo "FAIL: drilldown arg-count guard missing 10"; exit 1; }

# Fix 1b — directive uses canonical helper to compute the path + passes it
dir_src=$(cat commands/consult.md)
[[ "$dir_src" == *'cw_consult_design_doc_canonical_path'* ]] \
  || { echo "FAIL: directive missing canonical_path helper call"; exit 1; }
[[ "$dir_src" == *'$DESIGN_DOC'* ]] \
  || { echo "FAIL: directive doesn't pass \$DESIGN_DOC to drilldown"; exit 1; }

# Fix 2 — Pattern numbering contiguous 1,2,3 (no 4, no skipped 2)
patterns=$(grep -E '^### Pattern [0-9]+:' commands/consult.md | awk '{print $3}' | tr -d ':')
[[ "$patterns" == $'1\n2\n3' ]] \
  || { echo "FAIL: Pattern numbering not 1,2,3 (got: $(echo "$patterns" | tr '\n' ',' ))"; exit 1; }

# Fix 3 — /spec purged from lib/consult.sh (trailing space skips /specs/ paths)
hits=$(grep -c '/spec ' lib/consult.sh || true)
[[ "$hits" == "0" ]] || { echo "FAIL: lib/consult.sh has $hits /spec references"; exit 1; }

# Fix 4 — drill template no longer says "synthesis"
tmpl=$(cat config/prompt-templates/consult/drilldown.md)
[[ "$tmpl" != *'synthesis'* ]] || { echo "FAIL: drill template still says 'synthesis'"; exit 1; }
[[ "$tmpl" == *'design doc'* ]] || { echo "FAIL: drill template missing 'design doc'"; exit 1; }

# Fix 5 — helper signature comment renamed to <design-doc-path>
prompts_src=$(cat lib/consult-prompts.sh)
[[ "$prompts_src" != *'<synthesis-path>'* ]] \
  || { echo "FAIL: lib/consult-prompts.sh still has <synthesis-path>"; exit 1; }
[[ "$prompts_src" == *'<design-doc-path>'* ]] \
  || { echo "FAIL: lib/consult-prompts.sh missing <design-doc-path>"; exit 1; }

# Fix 6 — version bump in both manifests
grep -q '"version": "0.20.2"' .claude-plugin/plugin.json \
  || { echo "FAIL: .claude-plugin/plugin.json not at 0.20.2"; exit 1; }
grep -q '"version": "0.20.2"' .claude-plugin/marketplace.json \
  || { echo "FAIL: .claude-plugin/marketplace.json not at 0.20.2"; exit 1; }

```

## Risk assessment

- **Fix 1a (script signature)**: only contract change in this patch. Risk: forgotten caller passes the old 6/7/8/9-arg shape and gets the missing-arg error. Mitigated by `grep -rn 'consult-drilldown' bin/ commands/ lib/ tests/` before commit — only `commands/consult.md` Step 13 and the new test should appear.
- **Fix 1b (directive change)**: relies on `cw_consult_design_doc_canonical_path` returning the same path that `consult-walk-assemble.sh` wrote. Both use `$(date -u +%Y-%m-%d)`, byte-equal in a same-UTC-day session (the normal drill cadence). Edge case: a consult started just before UTC midnight that drills just after — design-doc was written under yesterday's date, canonical helper returns today's date, and Step 13 errors with `no design-doc at <today-path>`. This was already broken before v0.20.2 (drilldown.sh's synthesis.md check would fail with a different message), so not a regression. A glob-based fallback (`ls -t`) is intentionally not added — out of scope for a stale-string patch; the canonical helper is what the user explicitly chose ("cleaner contract").
- **Fix 2 (Pattern renumber)**: pure heading text change. Risk: a cross-reference elsewhere in the doc says "see Pattern 4". Validator's spot-check showed none; verify with `grep -n 'Pattern [234]' commands/consult.md` post-edit.
- **Fix 3 + 4 + 5 (string swaps)**: pure text edits, zero behavior. The seed-prose strings appear in users' future committed design docs but won't break their existing committed design docs from earlier versions.

Rollback is `git revert <merge-sha>` — every change is a localized text edit.
