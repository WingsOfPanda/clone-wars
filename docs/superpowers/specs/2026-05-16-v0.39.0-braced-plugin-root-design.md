# v0.39.0 — directive CLAUDE_PLUGIN_ROOT brace migration (design)

**Date:** 2026-05-16
**Type:** Bugfix bundle (closes v0.38.0 dogfood finding)
**Scope:** Single PR; pure markdown sed + new lint test + version bump

## Summary

Convert all 96 unbraced `$CLAUDE_PLUGIN_ROOT` references in `commands/*.md`
directives to the braced `${CLAUDE_PLUGIN_ROOT}` form so Claude Code's
slash-command renderer substitutes them into absolute install paths at
directive-render time. Add a permanent lint that prevents regression.

## Problem

While running `/clone-wars:medic` at v0.38.0 as a real dogfood pass, Step A's
bash block failed with `source: /lib/state.sh: No such file or directory`
when copy-pasted verbatim into the Bash tool. The directive's source is:

```bash
source "$CLAUDE_PLUGIN_ROOT/lib/state.sh"
```

Investigation revealed Claude Code's renderer **does** substitute
`${CLAUDE_PLUGIN_ROOT}` (braced) at directive-render time, replacing it
with the absolute install path before the directive is shown to the model.
It **does not** substitute `$CLAUDE_PLUGIN_ROOT` (unbraced) — that form
survives literal into the directive text. Whether bash works depends on
whether the model mentally expands it or copy-pastes verbatim. Verbatim
paste → bash sees `$CLAUDE_PLUGIN_ROOT` as an unset variable → expands to
empty string → command fails.

Census across `commands/*.md` in v0.38.0 source:

| File | UNBRACED (broken if copy-pasted) | BRACED (works always) |
|---|---:|---:|
| `medic.md` | 1 | 1 |
| `list.md` | 1 | 1 |
| `teardown.md` | 1 | 1 |
| `meditate.md` | 18 | 0 |
| `consult.md` | 34 | 0 |
| `deploy.md` | 41 | 0 |
| `deep-research.md` | 0 | 42 |
| `deep-research-resume.md` | 0 | 5 |
| **Total** | **96** | **51** |

v0.37.0's portable-paths work used the braced form correctly in the two
deep-research directives. The 96 unbraced sites are pre-v0.37 directives
that were never migrated.

## Goals

1. Zero unbraced `$CLAUDE_PLUGIN_ROOT` references in `commands/*.md`
2. Permanent lint (`tests/test_braced_plugin_root.sh`, no skip-guard) that
   fails on any future unbraced reference in `commands/`
3. Version-locked static-wiring test (`tests/test_v0_39_0_static_wiring.sh`)
   asserting 5 invariants
4. CLAUDE.md + CHANGELOG.md updated; v0.38.0 strict-dogfood row marked
   `[x] partial` per v0.28.0 precedent (first dogfood finding caught + fixed)

## Non-goals

- No version bump beyond v0.39.0; no follow-up releases planned
- No audit of `lib/`, `bin/`, `hooks/`, `config/` — bash scripts; `$VAR` is
  runtime expansion, not render-time substitution
- No changes to `commands/deep-research{,-resume}.md` (already 100% braced)
- No carve-out mechanism for "intentional literal `$CLAUDE_PLUGIN_ROOT`
  examples" — zero exist today; defer until a real use case appears
- No harness-side fix to support unbraced form (both forms are supported;
  we choose the one that works portably)
- No update to repo CLAUDE.md's "Execution discipline" section about
  `$CLAUDE_PLUGIN_ROOT` (the section is about background-work, not
  directive-substitution)

## Design

### The conversion

Mechanical sed replace across all 6 affected directives:

```bash
sed -i 's/\$CLAUDE_PLUGIN_ROOT/${CLAUDE_PLUGIN_ROOT}/g' \
  commands/consult.md commands/deploy.md commands/meditate.md \
  commands/medic.md commands/list.md commands/teardown.md
```

The regex is anchored to the bare token `$CLAUDE_PLUGIN_ROOT`. It matches
unbraced form only because braced form (`${CLAUDE_PLUGIN_ROOT}`) does not
contain `$CLAUDE_PLUGIN_ROOT` as a literal substring (the `{` interposes
between `$` and `C`). Idempotent under re-runs.

Post-sed verification:
- `grep -rc '\$CLAUDE_PLUGIN_ROOT' commands/` → 0
- `grep -rc '\${CLAUDE_PLUGIN_ROOT}' commands/` → 147 (51 existing + 96 new)
- `bash tests/run.sh` still green (no behavior change — pure render-time
  substitution gain)

### Permanent lint: `tests/test_braced_plugin_root.sh`

No version skip-guard. Runs on every release forever. Single bare grep:

```bash
#!/usr/bin/env bash
# Permanent invariant: every $CLAUDE_PLUGIN_ROOT in commands/*.md must be
# braced form (${CLAUDE_PLUGIN_ROOT}). The Claude Code slash-command renderer
# substitutes only the braced form into absolute install paths at
# directive-render time. Unbraced references survive literal into bash
# subshells where $CLAUDE_PLUGIN_ROOT is unset and expand to empty.
# See docs/superpowers/specs/2026-05-16-v0.39.0-braced-plugin-root-design.md
set -euo pipefail
cd "$(dirname "$0")/.."

[[ -d commands ]] || { echo "FAIL: commands/ not found at $(pwd)/commands" >&2; exit 1; }

set +e
hits=$(grep -rn '\$CLAUDE_PLUGIN_ROOT' commands/ 2>/dev/null)
rc=$?
set -e

# grep exit codes: 0=match found, 1=no match, 2+=error
if [[ $rc -gt 1 ]]; then
  echo "FAIL: grep error (rc=$rc) scanning commands/" >&2
  exit 1
fi
if [[ -n "$hits" ]]; then
  echo "FAIL: unbraced \$CLAUDE_PLUGIN_ROOT in commands/ (must be \${CLAUDE_PLUGIN_ROOT}):" >&2
  echo "$hits" >&2
  exit 1
fi
echo "PASS: all CLAUDE_PLUGIN_ROOT refs in commands/ are braced"
```

Explicit grep-error handling: exit-1 (no match) is the success path,
exit-2+ (error like missing directory) fails loudly instead of masking as
PASS. The `[[ -d commands ]]` guard catches the wrong-cwd case before grep
even runs.

### Version-locked static-wiring: `tests/test_v0_39_0_static_wiring.sh`

Version-guarded skip pattern (matches v0.38.0). 5 invariants:

1. `plugin.json` version == `0.39.0`
2. `tests/test_braced_plugin_root.sh` exists, is executable
3. Regression spot-check: `commands/medic.md` Step A contains
   `source "${CLAUDE_PLUGIN_ROOT}/lib/state.sh"` (braced)
4. Census assertion:
   `grep -rcF '${CLAUDE_PLUGIN_ROOT}' commands/ | awk -F: '{s+=$2} END {print s}'`
   returns ≥ 147 (current 51 + migrated 96; future migrations only add)
5. CLAUDE.md "Current focus" section names v0.39.0 as "Most recent merge"

### What this release does NOT do

- Does not change any runtime behavior — pure render-time substitution
  improvement
- Does not test "model pastes verbatim from directive into Bash" — no
  fresh-session simulator exists in the test harness; the lint enforces
  the invariant statically instead
- Does not modify `tests/run.sh` driver — new test files are auto-picked
  by the `test_*.sh` glob

## Risks

- **Sed false-positive on user-facing example text:** none today (census
  shows zero prose-context literal references); risk realizes only if a
  future directive author writes `Use $CLAUDE_PLUGIN_ROOT as a placeholder`
  in plain Markdown. Mitigation: when that day comes, add a
  comment-marker allowlist; for v0.39.0 the bare grep is correct
- **Static-wiring test census (invariant 4) drifts:** if a future PR
  removes braced refs intentionally, the `≥ 147` assertion fails. The fix
  is to update the invariant to the new count when that PR lands —
  expected and routine, no different from v0.38.0 census invariants
- **Marketplace/plugin.json read-before-edit collision:** late-stage PR
  writes to plugin.json hit this on every release; recovery is one
  Read+Edit per CLAUDE.md "Execution discipline" guidance
