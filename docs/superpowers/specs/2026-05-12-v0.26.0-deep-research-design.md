## 1. Problem

After meditate (deep landscape document) and consult (cross-verified design doc),
clone-wars has no command that **runs experiments**. The Karpathy/AIDE-pattern
autoresearch loop — agents that hypothesize approaches, implement them as
runnable code, benchmark, score, and select winners — is missing from the surface.

A user investigating "what's the best architecture for X under Y constraints"
currently has to:
1. Run `/clone-wars:meditate` to get a landscape (no execution).
2. Run `/clone-wars:consult` to get a design doc (no execution).
3. Write the experiments themselves and run them by hand.

Step 3 is where the work is. Deep-research collapses it.

## 2. Goal

A new slash command:

```
/clone-wars:deep-research <topic-with-explicit-metric>
                          [--max-rounds N]
                          [--branches-per-round K]
                          [--time-budget DURATION]
                          [--cost-warning USD]
                          [--allow-net]
                          [--seed-from PATH-TO-MEDITATE-LANDSCAPE]
```

That runs an AIDE-pattern multi-round tree search:

- **Conductor (Yoda / claude)** plans each round inline: hypothesizes branches,
  scores results, selects survivors, synthesizes the final landscape doc.
- **Troopers (codex only)** — K per round, each in its own pane and sandbox dir
  — execute one branch each in a single trooper turn: implement + run + emit
  `result.json`.
- Loop runs `--max-rounds` times, or until convergence (delta < 1% × 2 rounds),
  or until budget exhaustion.
- Final artifact: `_deep-research/deep-research-<date>-<slug>.md` with
  round-by-round table, winner branch + code path, and
  `Suggested next: /clone-wars:deploy <winner-code-path>` when applicable.

Single-machine only (the conductor host). Codex required (no fallback to
claude/opencode). v1 sandboxing is honor-system; the doc/README ship a clear
DANGER block.

## 3. Architecture

### 3.1 The big departure: roster split

Meditate's symmetric multi-provider roster does **not** apply.
Deep-research splits responsibilities:

| Phase           | Actor                          | Provider |
|-----------------|--------------------------------|----------|
| Hypothesize     | Conductor (Yoda)               | claude   |
| Allocate cmdrs  | Conductor (Yoda)               | claude   |
| Spawn           | Conductor (Yoda)               | claude   |
| Experiment      | K troopers per round, parallel | **codex only** |
| Score           | Conductor (`bin/.../score.sh`) | claude   |
| Select          | Conductor (Yoda)               | claude   |
| Teardown round  | Conductor (Yoda)               | claude   |
| Final synth     | Conductor (Yoda)               | claude   |

**Why codex-only for the experiment trooper:** codex is the only fleet provider
with the `--dangerously-bypass-approvals-and-sandbox` posture appropriate for
"write code, run code, measure result" loops with minimal friction. Claude's
permissions prompts and opencode's auth handshake make them poor fits for a
trooper that has to implement-and-run inside a single turn under wall-clock
pressure. Adding mixed-provider trooper support is a future decision (v0.27+).

**`bin/deep-research-init.sh` therefore refuses if codex is not in
`providers-available.txt`** with a clear error pointing the user to install
codex. The medic active-set (`providers-active.txt`) is **ignored** by
deep-research (it's a roster-selection mechanism for consult/meditate; deep-research
doesn't have a roster choice).

### 3.2 Per-round phases

For `round n in 1..MAX_ROUNDS`:

1. **Hypothesize (Yoda inline).** For round 1: Yoda researches the topic
   (WebSearch + Tavily paired per global rule; optionally seeded by
   `--seed-from`); decides on `K` distinct approaches; writes
   `_deep-research/round-<n>/branches.txt` (TSV:
   `branch_id\tcommander\tapproach_label\tapproach_brief`).
   For round n>1: Yoda reads `round-<n-1>/scoreboard.md` + (optionally)
   does additional research; selects top-M survivors as parents and
   proposes K children that vary the surviving approaches.
2. **Allocate commanders (Yoda + lib helper).**
   `cw_deep_research_allocate_commanders <round> <K>` returns K unique
   commander names from the codex-eligible commander pool (Captain rank +
   Trooper rank in `lib/commanders.sh`). Mod-rotates across rounds so
   commanders aren't reused round-over-round until the pool is exhausted.
   Pool exhaustion → error with clear remediation ("reduce K or max-rounds").
3. **Spawn (Yoda; K parallel Bash tool calls).** Each call:
   `bin/spawn.sh <commander> codex "deep-research-<slug>"`. Spawn opens
   trooper pane in the **branch sandbox dir** via `tmux split-window -c
   $BRANCH_DIR` so the trooper's cwd is already inside its sandbox. (The
   v0.20.3 multi-repo deploy precedent: cwd-on-spawn, never `cd` after.)
   Stage 1 retry-once on cold-start failure (codex bootstrap); Stage 2
   AskUserQuestion for partial-success / abort. Identical to consult v0.19+.
4. **Dispatch experiments (Yoda; K parallel Bash tool calls).** Each call:
   `bin/deep-research-experiment-send.sh <slug> <commander> <branch_id>`.
   Renders the experiment template with branch context + budget knobs;
   writes inbox.md; nudges via `bin/send.sh @file` (canonical write+nudge
   convention from v0.22.0).
5. **Wait (Yoda; one foreground call).**
   `bin/deep-research-experiment-wait.sh <slug> <commander1> [...]`
   blocks until each trooper emits `{"event":"done"}` to outbox or
   `bin/deep-research-experiment-wait.sh`'s wall-clock cap fires.
   Per-branch hard cap = `ceil(time_budget_s / total_planned_branches)`
   passed via `CW_DEEP_RESEARCH_EXPERIMENT_TIMEOUT_OVERRIDE` env var.
   Thin shim around shared `cw_consult_wait` from `lib/consult-wait.sh`
   (extended to recognize a new `experiment` kind, mirroring v0.25.0's
   `adversary` extension).
6. **Score (mechanical).** `bin/deep-research-score.sh <slug> <round>`
   parses each `_deep-research/round-<n>-<cmdr>-<branch_id>/result.json`
   and writes `_deep-research/round-<n>/scoreboard.md` — table sorted
   descending by metric value. Failed/timeout branches grouped at bottom
   with `metric_value: null`. Atomic write via existing `cw_atomic_write`
   helper.
7. **Select (Yoda inline).** Read scoreboard.md; pick top-M survivors
   (default `M = ceil(K/2)`, cap `M ≤ K-1`). If all branches failed →
   set `DIVERGED=1` and break loop early. Compute round-over-round delta
   (best metric this round vs. best metric previous round); if `< 1%` for
   two consecutive rounds → set `CONVERGED=1` and break loop early.
8. **Teardown round troopers.** `bin/teardown.sh --pairs <slug> cmdr1
   [cmdr2 ...]` (the v0.20.5 batched 9s-banner-once teardown; not the
   per-trooper loop). Branch sandbox dirs **stay** (they're under
   `_deep-research/round-<n>-<cmdr>-<branch>/`); only panes + inboxes
   teardown. Scoreboard remains.
9. (Loop) Continue with `n = n+1`.

After loop exit:

10. **Final synth (Yoda inline).** Yoda writes the final landscape doc
    `_deep-research/deep-research-<date>-<slug>.md` using the schema in
    §3.6. Uses Write tool atomic single-shot.
11. **Archive.** `bin/deep-research-teardown.sh <slug>` archives the entire
    state dir to `~/.clone-wars/archive/<repo-hash>/deep-research-<slug>-<ts>/`
    (parallels meditate teardown).

### 3.4 Result.json schema

Trooper writes this atomically to `<branch-dir>/result.json`:

```json
{
  "branch_id": "rex-b1",
  "approach_label": "AIDE tree search with depth=3",
  "metric_name": "accuracy",
  "metric_value": 0.987,
  "status": "ok",
  "runtime_s": 287,
  "log_paths": ["./stdout.log", "./stderr.log"],
  "notes": "Used PyTorch nn.Module with 84k params; converged epoch 47/50."
}
```

Field semantics:
- `branch_id` (string, required) — must equal `<commander>-<branch_id>` from
  branches.txt; deserialized into the score table.
- `approach_label` (string, required, ≤120 chars) — short label for the table.
- `metric_name` (string, required) — should match `metric.txt`'s metric.
- `metric_value` (number or null, required) — null iff status ≠ "ok".
- `status` (string, required) — one of `ok`, `fail`, `timeout`, `cost_blown`.
  `cost_blown` is honor-system: trooper self-reports if it observed costs
  approaching `cost-warning`.
- `runtime_s` (number, required) — wall-clock seconds for the run phase only
  (excluding implement phase, since they're commingled in single-turn).
- `log_paths` (array of strings, required) — relative paths from branch dir;
  Yoda's score.sh validates they exist (mechanical spot-check; doesn't verify
  metric came from the log).
- `notes` (string, optional, ≤500 chars) — free-form trooper observations.

Validation (`cw_deep_research_validate_result_json` in `lib/deep-research.sh`):
- Required fields present
- `status` is one of the four enums
- `metric_value` is null iff status ≠ "ok"
- `log_paths[*]` exist on disk
- Returns 0 on valid; non-zero with stderr error message on invalid

Invalid result.json → score.sh treats branch as `status: "fail"` with
`notes: "Invalid result.json: <reason>"`.

### 3.5 Sandboxing — v1 honor-system

Each branch gets its own dir under `_deep-research/round-<n>-<cmdr>-<branch_id>/`.
Trooper pane is opened with `tmux split-window -c $BRANCH_DIR` so cwd is
already-rooted there.

**No** containerization, **no** chroot, **no** bwrap/firejail. Codex runs with
`--dangerously-bypass-approvals-and-sandbox` (same posture as every other
clone-wars command).

**Trooper template tells the trooper:**
- Stay inside the current directory.
- Do not modify files outside the branch dir.
- Do not run system-level commands (apt, brew, sudo, etc.).
- If `--allow-net=false` (default): do not fetch external resources
  (no curl, no wget, no pip install of new packages, no Tavily/WebFetch).
- If `--allow-net=true`: net access is permitted; use it only as needed.

**v1 enforcement is honor-system.** Codex troopers historically respect
explicit boundaries when stated clearly. v1 is dogfood-grade; a stronger
sandboxing story is v0.27+ scope.

**README + final doc** ship a DANGER block:

> ⚠️ `/clone-wars:deep-research` runs codex troopers with
> `--dangerously-bypass-approvals-and-sandbox` and has them write + execute
> arbitrary code in your repo. v1 sandboxing is honor-system. Do **not** run
> on machines with sensitive credentials, production data, or shared state.
> Use a scratch worktree (`/clone-wars:deep-research` does **not** create one
> for you).

### 3.7 Budget knobs + preflight confirmation

Defaults (set at top of `bin/deep-research-init.sh`):

| Knob                        | Default | Notes |
|-----------------------------|---------|-------|
| `--max-rounds`              | 3       | Hard cap on loop iterations |
| `--branches-per-round`      | 4       | K (used for every round) |
| `--time-budget`             | 1h      | Total wall-clock; per-branch cap = budget/total_branches |
| `--cost-warning`            | 5       | USD; informational, surfaced to trooper |
| `--allow-net`               | false   | If false, trooper told not to fetch external |
| `--seed-from`               | (none)  | Absolute path to a meditate landscape doc |

After init, **before round 1 hypothesize**, fire ONE `AskUserQuestion` to
confirm spend:

> "About to run K branches/round × R rounds = T total branches; per-branch
> wall-clock cap T_BRANCH; total wall-clock budget BUDGET; informational cost
> ceiling $C. Continue?"
>
> Options:
> - **Continue** (default recommended)
> - **Adjust budget** (re-run with different flags)
> - **Cancel**

This is the **only** AskUserQuestion in the entire run after init. After
"Continue" the loop is autonomous until convergence/budget/error.

### 3.8 Composability

**Forward composability — `--seed-from` from meditate:**

`--seed-from <path-to-meditate-landscape>` reads the `## Approaches` section
from a meditate landscape doc and bootstraps round-1 hypotheses with one
branch per approach (capped by `--branches-per-round K`). Yoda still chooses
labels and briefs; the seed provides candidate approaches Yoda would otherwise
research itself.

Helper: `cw_deep_research_extract_approaches <landscape-path>` parses the
markdown, returns approaches as TSV (`label\tbrief`).

**Backward composability — final doc → `/clone-wars:deploy`:**

When a winner converges with measurable improvement, the final doc emits a
`Suggested next: /clone-wars:deploy <winner-code-path>` line. User is
responsible for crafting a deploy design doc from the winning branch — there's
no automatic /clone-wars:deploy invocation. (v0.27+ may automate this.)

### 4.3 lib/deep-research.sh API

Public functions (alphabetized):

```
cw_deep_research_allocate_commanders <round> <K>
    # Returns K unique codex-eligible commander names, mod-rotated per round.
    # Errors with non-zero rc if K > pool size.

cw_deep_research_extract_approaches <landscape-md-path>
    # Parses ## Approaches section from a meditate landscape doc.
    # Outputs TSV: label\tbrief, one per line.

cw_deep_research_extract_metric <topic-text>
    # Heuristic extraction of "metric" from topic text.
    # Looks for keywords: "accuracy", "latency", "throughput", "loss",
    # "f1", "precision", "recall", "cost", "memory", "params",
    # plus common phrases ("optimize X", "minimize Y", "maximize Z").
    # Returns extracted metric to stdout, or empty string if ambiguous.
    # Caller (init.sh) AskUserQuestion-prompts user when empty.

cw_deep_research_validate_result_json <path>
    # Returns 0 if path is valid result.json per §3.4 schema.
    # Non-zero with stderr error otherwise.

cw_deep_research_compute_per_branch_timeout <total-budget-s> <max-rounds> <K>
    # Returns ceil(total-budget-s / (max-rounds * K)) seconds.
    # Used as CW_DEEP_RESEARCH_EXPERIMENT_TIMEOUT_OVERRIDE for each round.

cw_deep_research_check_convergence <slug> <round>
    # Reads round-(N-1)/scoreboard.md and round-N/scoreboard.md.
    # Returns 0 if delta < 1% (signal convergence); non-zero otherwise.
    # Yoda directive checks return code + accumulates "consecutive
    # convergence rounds" counter inline.
```

### 4.4 Codex commander pool — eligibility

`lib/commanders.sh` already has `cw_cmdr_rank`:

| Rank       | Commanders                                                |
|------------|-----------------------------------------------------------|
| Captain    | rex, keeli, colt, trauma, blackout                        |
| Commander  | cody, bly, wolffe, fox, gree, ponds, bacara, neyo, doom, faie |
| Sergeant   | hunter                                                    |
| Lieutenant | havoc, thorn, thire, stone                                |
| Trooper    | (default — fallback for unmapped names)                   |

**Reserved by provider in existing code:**
- cody → claude (v0.20.5 reserved for plugin-dev)
- bly → opencode (legacy; v0.20.5 deprecated in favor of wolffe)
- wolffe → opencode (canonical mapping)

**Codex-eligible pool for deep-research** (codex maps to "rex" canonically;
other commanders are unreserved by provider in v0.25.1):

```
rex, keeli, colt, trauma, blackout, fox, gree, ponds, bacara,
neyo, doom, faie, hunter, havoc, thorn, thire, stone
```

That's 17 commanders. Mod-rotation across rounds means K=4 × R=3 = 12
allocations exhaust ~70% of the pool. K=4 × R=5 = 20 exceeds pool — error
with remediation message.

`cw_deep_research_allocate_commanders` implementation strategy:
1. Compute total slots needed cumulatively across rounds (rounds 1..N each
   reserve K commanders that don't repeat across rounds until pool is
   exhausted).
2. Round 1 takes the first K from the pool (deterministic for testability).
3. Round 2 takes the next K (positions K..2K-1).
4. … etc., wrapping at pool size with explicit error if the wrap would
   re-use a commander still alive in another round (defensive — but in v1
   teardown happens between rounds so re-use is technically safe; we still
   error to keep audit trails clean).

## 5. The directive — `commands/deep-research.md` outline

Following meditate.md's pattern (~600 lines). Numbered steps, each with task
list + bash blocks. Sketch:

- **Frontmatter** — `description`, `argument-hint: <topic-with-metric> [--max-rounds N] [--branches-per-round K] [--time-budget D] [--cost-warning USD] [--allow-net] [--seed-from PATH]`, `allowed-tools: AskUserQuestion, Bash, Edit, Read, Skill, Task, TaskCreate, TaskUpdate, Write` (Skill needed for WebSearch/Tavily during hypothesize phase).
- **Intro** — what the command does; explicit note that conductor plans + codex executes; explicit DANGER block (sandboxing).
- **Task list (TaskCreate × ~20)** — phases enumerated upfront; sub-rows for per-round phases get TaskCreate'd at round entry (parallels v0.23.1's per-trooper sub-rows in deploy).
- **Step 0 — Init** — args parse, `bin/deep-research-init.sh`, write topic.txt, metric extraction (lib helper + AskUserQuestion fallback), budget.txt, `--seed-from` validation if present.
- **Step 1 — Preflight confirmation** — single `AskUserQuestion` for spend confirmation (§3.7).
- **Step 2 — Round loop** — `for ((n=1; n<=MAX_ROUNDS; n++))`:
    - **2.a** Hypothesize (Yoda inline; Skill / WebSearch / Tavily; write branches.txt)
    - **2.b** Allocate commanders (lib helper)
    - **2.c** Spawn K codex troopers (parallel Bash tool calls; Stage-1 retry; Stage-2 partial-success AskUserQuestion)
    - **2.d** Dispatch experiments (parallel Bash tool calls)
    - **2.e** Wait (foreground)
    - **2.f** Score (`bin/deep-research-score.sh`)
    - **2.g** Select (Yoda inline; check convergence + divergence)

## 8. Release gate (strict-dogfood checklist)

Verify on a real machine before marking gate complete:

1. `/clone-wars:deep-research "optimize MNIST classifier accuracy under 100k
   params"` runs end-to-end with default budget; final landscape doc lands;
   winner branch's `code/` has runnable script + readable result.json.
2. Round-1 troopers' panes `tmux display-message -p '#{pane_current_path}'`
   matches `<state>/round-1-<cmdr>-<branch>/` (cwd-on-spawn working).
3. Round-2 hypotheses cite round-1 winners (Yoda's hypothesize phase respects
   scoreboard) — verifiable by reading round-2/branches.txt's approach_brief
   columns.
4. Per-branch `timeout` enforces wall-clock cap (deliberately blocking
   trooper to verify it gets killed at budget).
5. Failed branch (deliberately introducing a divide-by-zero in the trooper's
   code) reports `status: fail` with `metric_value: null`; not propagated
   to next round.
6. `--seed-from <meditate-archive>/landscape-<date>-<slug>.md` bootstraps
   round 1 from landscape's `## Approaches` section.
7. `--allow-net=false` (default): trooper instruction text contains "do not
   fetch external resources" (verifiable via inbox.md inspection).
8. `--allow-net=true`: trooper instruction text omits the prohibition + says
   "net access is permitted".
9. Convergence early-exit fires when 2 consecutive rounds improve < 1%.
10. Teardown of all per-round commander panes via batched `--pairs` 9s
    banner (one banner not N×).
11. Cost-warning surfaced in trooper identity prompt verbatim.
12. README + final doc both contain DANGER block.

## 9. Out of scope (deferred to v0.27+)

- Cross-trooper re-verification of metrics (Yoda trusts trooper-reported
  numbers; v0.27 may add a sibling-trooper "re-run and confirm" round).
- Multi-machine compute (only conductor host).
- Container/bwrap/firejail sandboxing (honor-system only in v1).
- Cost-aware branching ("skip this branch because too expensive").
- Hyperparameter sweeps within a branch (each branch is one config).
- Mixed-provider troopers (codex only in v1).
- Automatic /clone-wars:deploy invocation from winner (manual step in v1).
- Tree-search backtracking (v1 is forward-only round-over-round; AIDE itself
  has full backtracking via UCB1; we model the simpler wave version).
- LSP / MCP integration (deep-research troopers run code; they don't need
  external code-intelligence in v1).
