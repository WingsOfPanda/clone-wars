## Architecture

### Per-finding deltas

#### Commit 1 — spawn.sh MODE chained-`||` (`bin/spawn.sh:150`)

Today:
```bash
[[ -n "$MODE" ]] || MODE=$(cw_contract_default_mode "$MODEL") || MODE=full
```
If `cw_contract_default_mode` returns rc=0 with empty stdout, `MODE` stays
empty and the next call (`cw_contract_mode_args "$MODEL" ""`) errors out.

Fix:
```bash
[[ -n "$MODE" ]] || MODE=$(cw_contract_default_mode "$MODEL")
[[ -n "$MODE" ]] || MODE=full
```

#### Commit 2 — drilldown collision counter (`bin/consult-drilldown.sh:113`)

Today the loop strips `${base%-[0-9]*}` to remove a previously-appended
`-N` collision suffix before re-appending. Bash glob `-[0-9]*` matches
`-<digit><any-trailing-chars>`, so for `drilldown-foo-2025-arch` it strips
`-2025-arch` and yields `drilldown-foo`. The collision iteration then writes
`drilldown-foo-2.md`, dropping the section name.

Fix: anchor the strip to a digits-only trailing counter via regex match.
```bash
if [[ "$base" =~ ^(.+)-[0-9]+$ ]]; then
  base="${BASH_REMATCH[1]}"
fi
```

#### Commit 3 — deploy.md two source-defaulting blocks (`commands/deploy.md:25-46`)

The directive has two source-defaulting code blocks. The top-level block
(lines 25-46) uses `find "$STATE_ROOT" -path '*/_consult/...'` — unscoped,
which would surface design-docs from OTHER repos under the same
`$CLONE_WARS_HOME`. Step 0 sub-step 4 (lines 99-110) uses
`find "$STATE_ROOT/state/$REPO_HASH" -path '*/_consult/...'` — correctly
scoped to this repo.

Fix: delete the top-level "Source defaulting" section entirely. Keep Step 0
sub-step 4 as the single canonical block.

#### Commit 4 — preflight CMDR_TO_CWD silent skip (`bin/preflight-layout.sh:73-77`)

Today:
```bash
while IFS=$'\t' read -r cmdr cwd; do
  [[ -n "$cmdr" && -n "$cwd" ]] && CMDR_TO_CWD["$cmdr"]="$cwd"
done < "$CWD_FROM"
```

Malformed lines drop silently. The v0.20.3 sub-repo cwd bug was hidden by
exactly this kind of silence elsewhere — a `log_warn` here would have caught
similar issues at preflight time.

Fix:
```bash
local lineno=0
while IFS=$'\t' read -r cmdr cwd; do
  lineno=$((lineno + 1))
  if [[ -n "$cmdr" && -n "$cwd" ]]; then
    CMDR_TO_CWD["$cmdr"]="$cwd"
  else
    log_warn "preflight cwd-map: skipping malformed line at $CWD_FROM:$lineno"
  fi
done < "$CWD_FROM"
```

## Test strategy

**Per-commit green discipline.** Bottom-up apply order ensures
`bash tests/run.sh` is green between each commit:

- Commits 1-4 (Tier 1 bugs) edit isolated files with no shared state. New
  regression tests come AFTER (commit 12), so commits 1-4 just need
  existing tests to keep passing.
- Commits 5-6 (Tier 2 deletes) each remove a function + its test in the
  same commit; shim test EXPECTED list edited together.
- Commits 8-10 (Tier 3 consolidation) edit `commands/consult.md` only;
  the existing `test_consult_directive_v017_static_wiring.sh` already
  locks step-label structure (0-16) and will fail loudly if the dedup
  goes too far.
- Commit 11 (new static-wiring test) lands AFTER all changes are in
  place, so its asserts have something to lock.
- Commit 12 (regression tests) lands AFTER the static-wiring lock —
  these are NEW tests, not changes to existing structure.
- Commit 13 (version bump) is mechanical.

## Rollback plan

Per-commit reverts via `git revert <sha>` (NOT `git reset --hard` —
preserves history). After a revert: re-run `bash tests/run.sh`, confirm
green, then decide:
- If the reverted commit was Tier 1 (a bug fix): block the PR, fix-and-
  recommit before merging.
- If the reverted commit was Tier 2 or Tier 3: merge partial PR; note
  the gap in the v0.20.4 release entry. Each tier delivers value
  independently.
