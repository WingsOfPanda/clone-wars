## Goal

Close eight quality-of-life gaps in `/clone-wars:deep-research` surfaced
by the 2026-05-17 dogfood: final-state freshness on halt, shared-dir
hygiene on archive, optional pre-flight smoke-test for trooper code,
formal lane-abandonment, winner-artifact alias, directive guidance for
context-file usage, and structured halt-flag format. Zero behavioral
change for runs that don't trigger the new code paths (e.g., no smoke
test passed → no smoke-test runs).

## Architecture / contract

Each accepted item lands as one lane (one commit). Lanes are mostly
independent and additive — no shared file beyond `lib/deep-research.sh`
(touched by Lane C only). Suite stays green at every lane boundary.

The trooper-dispatch flow (gated by `phase=idle` filter at
`commands/deep-research-resume.md:95`) gains one new sentinel value
(`phase=abandoned`) and one new optional pre-flight stage (skill-side
smoke-test). Both are inert when not exercised. The archive flow
(`bin/deep-research-teardown.sh`) gains two side effects (orphan sweep
+ winner symlink). The halt flow (`bin/deep-research-finalize.sh`)
gains one unconditional render-summary call. The `halt.flag` write
format is standardized across `bin/deep-research-abort.sh`,
`commands/deep-research-resume.md:118`, and any future Yoda-halt site.

## Components

### Lane A — re-render session-summary on halt (Item 2)

**File:** `bin/deep-research-finalize.sh` (one new call inserted before
the existing fallback rebuild check at line 64).

Current behavior: `finalize.sh` re-renders `session-summary.md` ONLY
when the file is absent. Halt-time append of `## Halt` runs over
whatever stale contents exist. Last night this missed `colt/exp-007`'s
status transition (final summary said `colt: working exp-007` when
exp-007 had already cleared target and triggered the halt).

New behavior: render unconditionally just before the append. Reads
per-trooper state files via existing `cw_deep_research_render_summary`
helper; atomic-writes via `cw_atomic_write`. No new state files; no
new args. Zero impact when state is already fresh (idempotent write).

### Lane B — shared-dir orphan sweep + winner symlink (Items 3 + 8)

**File:** `bin/deep-research-teardown.sh` (two new steps before
existing `mv "$TOPIC_DIR" "$archive_dir"` at line 37).

**Orphan sweep (Item 3):** scoped to `$TOPIC_DIR/_deep-research/shared/`
only. Pattern: `find … -maxdepth 2 \( -name '*.tmp' -o -name '*.lock' \)
-delete`. The shared/ dir is the only known location for trooper-built
artifacts with atomic-write/flock orphans (last night: 20 MB
`train_shard.tmp` + 0-byte `train_shard.bin.lock`). The depth limit
prevents accidentally walking trooper experiment dirs.

**Winner symlink (Item 8):** when scoreboard.md has at least one ok-row,
read the top-1 row's `<cmdr>` + `<exp-id>`, create
`$TOPIC_DIR/_deep-research/winner -> troopers/<cmdr>/experiments/<exp-id>/code`
(relative symlink so it survives the archive `mv`). Skipped silently
when scoreboard.md is missing or has zero ok rows (early-halted /
aborted runs). The landscape doc's "Suggested next" command line
(`commands/deep-research.md:725`) is updated to use the
`_deep-research/winner/` path instead of computing the full per-experiment
absolute path.

### Lane C — skill-side `--smoke-test <script>` flag (Items 1 + 7)

**File:** `bin/deep-research-experiment-send.sh` (new flag parser,
new pre-flight stage). Optional helper in `lib/deep-research.sh` if
the smoke-test invocation grows beyond a one-liner.

**Flag shape:** `--smoke-test <abs-path-to-script>`. When passed, the
script is invoked as a subprocess after the existing `--inputs`
readability probe (line 65) and before any state mutation. Working
directory: the trooper's experiment branch dir
(`troopers/<cmdr>/experiments/<exp-id>/code/`) so the script can
reference the trooper's freshly-rendered code. Environment: inherits
the dispatch environment plus `CW_SMOKE_TEST=1` so the script can
distinguish smoke-test from real-run mode. Timeout: fixed 60 seconds
in v0.43.0 (hard kill via `timeout 60`); configurable timeout deferred.

**Failure semantics:** non-zero exit → dispatch aborts with rc=2. The
script's stderr is captured to `troopers/<cmdr>/experiments/<exp-id>/smoke-test.err`
and printed verbatim to the conductor. Trooper state stays `idle`
(no transition to `working`). Yoda's resume handler sees the failure
event, can decide whether to dispatch a fixed experiment.

**Usage patterns:** Yoda may pass any executable script — Python file,
bash one-liner, etc. Typical smoke tests:
- `python -m py_compile train_eval.py` (Item 1-style typo catch)
- `python -c 'import importlib; importlib.import_module("dataset")'` (import-chain check)
- `python -c 'import torch; m = TrainEval(); _ = m(torch.zeros(1, 60))'` (1-shape forward pass)
- A shared/ validator script written by Yoda once per topic.

Omission of `--smoke-test` is the default — no smoke test runs. This
preserves the current contract for callers that don't opt in.

### Lane D — formal `phase=abandoned` lane-abandon signal (Item 6)

**Files:** `commands/deep-research-resume.md` (new abandon-decision
block in Step 5); `lib/deep-research.sh` (new helper
`cw_deep_research_lane_abandon` if needed for atomic state.txt update);
`commands/deep-research.md` Phase 4.a status table (document the new
phase value).

**Decision rule (Yoda-side, in resume.md Step 5):** before dispatching
to a trooper at `phase=idle`, check whether the trooper's lane should
be abandoned. Criteria, ALL three must hold:

1. Trooper has ≥ 3 completed experiments (status=ok in scoreboard.md).
2. None of the trooper's last 3 experiments scored ≥ `min_acceptable`
   (from metric.md).
3. Trooper's best metric is ≥ 5 × `plateau_threshold` BELOW the
   current overall leader's metric. The multiplier (5) is fixed in
   v0.43.0; configurability deferred.

When all three hold, Yoda transitions the trooper to `phase=abandoned`
via `cw_deep_research_lane_abandon <art-dir> <cmdr> <reason>`. The
helper atomically updates `troopers/<cmdr>/state.txt` to set
`phase=abandoned`, `lane_abandon_reason=<reason>`,
`lane_abandon_ts=<iso>`. Step 5's existing `phase=idle` filter
naturally excludes abandoned troopers from future dispatch.

**Reversibility:** Yoda may re-enable a trooper later by manually
editing state.txt back to `phase=idle` (e.g., user countermand). Not
automated.

**Surface in session-summary:** the existing Status table row for the
trooper shows `phase=abandoned` with a 1-line reason. The
render_summary helper needs no changes — it already reads `phase` from
state.txt.

### Lane E — directive guidance (Item 9) + halt-flag format (Item 10)

**Files:** `commands/deep-research.md` (Phase 4.a context-file note +
new halt-flag format spec); `bin/deep-research-abort.sh` (format
unification); `commands/deep-research-resume.md:118` (Yoda halt write
standardization).

**Item 9 — directive note:** add a one-paragraph block to
`commands/deep-research.md` Phase 4.a explaining that per-experiment
context belongs in `prompt.md` via the `--context-file=<path>` flag,
NOT in a separately-written `<cmdr>/exp-NNN-context.md` file. The
skill already supports this path; the dogfood drift was Yoda inventing
files the skill never asked for.

**Item 10 — `halt.flag` format spec:** standardize on `key=value` lines,
one per line, matching the existing `bin/deep-research-abort.sh:50-53`
`reason=...` convention.

Required keys (every write site):

```
halted_by=user|yoda
halted_at=<ISO-8601-UTC>
reason=<short prose, single line>
```

Optional keys (Yoda-halt synthesis-time writes SHOULD include these):

```
target_met=yes|no
floor_met=yes|no
k_so_far=<N>
k_required=<N>
plateau=yes|no
plateau_window=<N>
final_leader=<cmdr>/<exp-id>
final_leader_metric=<value>
architectures_corroborated=<comma-separated-labels>
```

**Format spec lives in `commands/deep-research.md`** as a dedicated
sub-section so all three writers can cite the same source. The three
writers are:

- `bin/deep-research-abort.sh` (user-abort path) — already uses
  `reason=` semantics; expand to include `halted_by=user`, `halted_at`.
- `commands/deep-research-resume.md:118` (Yoda-halt path via Step 6) —
  current single-line `user-halted at <HH:MM:SSZ>` becomes structured
  multi-line write.
- Yoda's synthesis-time halt (when target is met and Yoda decides to
  stop) — currently free-form (last night's
  `yoda-halted at 16:15:34Z: target met...` matched no documented
  format). Now: must conform.

Readers (`bin/deep-research-finalize.sh:32` and the existence checks
at `commands/deep-research-resume.md:32,38,87`) parse via awk `-F=` —
trivial migration.

## Data flow

```
/clone-wars:deep-research <topic>
    │
    ├─ Phase 0: init (unchanged)
    ├─ Phase 1: metric.md (unchanged)
    ├─ Phase 2: preflight (unchanged)
    ├─ Phase 3: spawn roster (unchanged)
    │
    ├─ Phase 4.a: first dispatch
    │     • dispatch via deploy-research-experiment-send.sh
    │     • (optional) --smoke-test <script> runs script in trooper cwd          ← Lane C
    │     • on smoke-test fail: rc=2, smoke-test.err captured, no state change   ← Lane C
    │     • document context-file convention                                     ← Lane E
    │
    ├─ Phase 4.b: re-entry loop
    │     • Step 5: per-trooper dispatch decision
    │       • before dispatch: check abandon criteria                            ← Lane D
    │       • if abandon: cw_deep_research_lane_abandon → phase=abandoned        ← Lane D
    │       • existing phase=idle filter excludes abandoned                      ← Lane D
    │     • Step 6: halt-flag write (if Yoda decides to halt)
    │       • format: structured key=value lines                                 ← Lane E
    │
    ├─ Phase 5: synthesis (Yoda writes landscape doc)
    │     • Suggested next: uses _deep-research/winner/ path                     ← Lane B
    │
    ├─ Phase 6: teardown + archive
    │     • bin/deep-research-finalize.sh
    │       • cw_deep_research_render_summary (unconditional)                    ← Lane A
    │       • read halt.flag (structured format)                                 ← Lane E
    │       • append ## Halt to session-summary
    │     • bin/deep-research-teardown.sh
    │       • sweep shared/*.tmp + shared/*.lock                                 ← Lane B
    │       • create winner symlink (if scoreboard has ok rows)                  ← Lane B
    │       • mv $TOPIC_DIR → $archive_dir
    │
    └─ Phase 7: present (unchanged)
```

## Failure modes

| Condition | Behavior |
|---|---|
| Lane A: state.txt missing for some commander | `cw_deep_research_render_summary` already handles missing trooper rows; skip, render rest |
| Lane B: shared/ has no .tmp or .lock files | find -delete is a no-op |
| Lane B: scoreboard.md has zero ok rows | Skip symlink creation, log info |
| Lane B: scoreboard.md missing entirely | Skip symlink creation, log info |
| Lane C: --smoke-test script doesn't exist | rc=2 at flag-validation, before state mutation |
| Lane C: --smoke-test script hangs | timeout 60s → SIGKILL, rc=2, error log includes "timed out" |
| Lane C: --smoke-test script non-zero exit | rc=2, stderr captured, trooper stays idle |
| Lane D: scoreboard.md missing for abandon check | Yoda treats criterion 1 as not-met → dispatch proceeds |
| Lane D: state.txt write fails atomically | Yoda retries; abandon-decision is idempotent |
| Lane E: halt.flag format violation by a writer | Reader (awk -F=) skips malformed lines; halt still proceeds |
| Lane E: existing free-form halt.flag in archived runs | Readers tolerate (one-line prose still has no `=`; falls through gracefully) |

## Migration / backward-compat

- **Lane A (render on halt):** purely additive. Older sessions had stale
  summary at halt; new sessions don't. No data migration.
- **Lane B (sweep + symlink):** purely additive. Archived runs already
  on disk keep their orphans (we don't touch the archive after mv).
  No data migration.
- **Lane C (smoke-test flag):** opt-in. Omitting `--smoke-test`
  preserves byte-equal current behavior. No test migration.
- **Lane D (phase=abandoned):** new phase value. Existing state.txt
  files have no `phase=abandoned` — that's the default. Render_summary
  helper already prints whatever `phase=` says; no parsing change.
- **Lane E (Item 9 directive note + Item 10 halt-flag format):** Item 9
  is documentation. Item 10 standardizes a previously-undocumented
  format — readers awk-parse `key=value`, tolerating one-line prose
  from old archives. The 3 writers all migrate in this same lane.

## Success criteria

- A deep-research run that completes successfully produces a
  `_deep-research/winner/` symlink resolving to the winning trooper's
  code dir. The landscape doc's "Suggested next" command uses the
  shortened path.
- A run halted by Yoda has a `session-summary.md` whose Status table
  reflects the FINAL state of every trooper (no stale "working
  exp-NNN" rows).
- Re-archiving a topic dir leaves no `*.tmp` or `*.lock` files in
  `shared/`.
- A trooper dispatched with `--smoke-test <script>` that exits 0
  proceeds normally; the same trooper with a script that exits 1
  stays in `phase=idle` and produces `smoke-test.err`.
- A trooper whose last 3 experiments all scored below floor (and lag
  the leader by ≥ 5 × plateau_threshold) transitions to
  `phase=abandoned` and is not dispatched further.
- `halt.flag` from any of the 3 writers conforms to the
  `key=value`-lines format. Existing readers parse it without
  modification.
