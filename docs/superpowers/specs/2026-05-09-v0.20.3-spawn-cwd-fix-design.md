# v0.20.3 Sub-Repo Trooper Spawn cwd Discipline

**Date:** 2026-05-09
**Plugin version bump:** 0.20.2 → 0.20.3
**Branch:** `fix/v0.20.3-spawn-cwd-fix`
**Source:** user report on multi-repo `/clone-wars:deploy` plus brainstorming-phase verification
**Scope:** four coordinated edits so multi-repo deploy preflight + spawn lands every trooper TUI natively rooted in its sub-repo cwd, no `cd` command, no transient hub flash. Single-repo deploy and all of `/clone-wars:consult` byte-equal v0.20.2.

## Problem

The user wants `/clone-wars:deploy` to enforce the discipline that each sub-repo trooper's pane is born in the right sub-repo cwd, never inheriting the conductor's hub cwd and `cd`-ing afterward. Brainstorming-phase investigation surfaced three problems together — none has shipped a strict-dogfood pass yet, so all three are latent in v0.20.2:

1. **`cw_pane_respawn` (lib/tmux.sh:45-54) uses cd-then-exec hack.** When `<cwd>` is non-empty, it builds `cd '$cwd' && exec $launch` and passes that string to `tmux respawn-pane -k -t <pane> "$cmd"`. The pane respawns with a shell that cd's then execs. End-state cwd is correct, but:
   - The `cd` command is visible in the pane.
   - Single-quote in `<cwd>` breaks the cd: `cd '/path/with'apostrophe' && exec ...` is a bash parse error and the respawn-pane kills the pane silently.
   - Violates the spirit of CLAUDE.md's discipline: "never rely on the conductor's cwd inheriting; use `tmux split-window -c <abs-path>`."
2. **`preflight-layout.sh` sentinel banner inherits Yoda's cwd.** The `tmux split-window` at line 104 has no `-c <cwd>` flag, so each preflight pane is allocated in the conductor's cwd (the hub). The sentinel banner runs in hub cwd until `cw_pane_respawn` later cd's it. There IS a transient hub-cwd flash visible if the user attaches to the pane during the ~ms between sentinel-create and respawn.
3. **Schema mismatch makes multi-repo preflight unworkable today.** `bin/consult-init.sh` writes troopers.txt as 2-col `<provider>\t<commander>`. `bin/deploy-multi-init.sh` writes troopers.txt as 3-col `<commander>\t<cwd>\t<provider>`. `bin/preflight-layout.sh` (line 102) parses with `IFS=$'\t' read -r prov cmdr` — assumes the consult shape. Fed deploy's file, it reads col 1 as "provider" but it's actually commander, col 2 as "commander" but it's a path, and silently drops col 3. Result: pane labels render with paths-as-commander, and `preflight-panes.txt` is keyed by the path string. Step 3b of `commands/deploy.md` then fails to find `${PREFLIGHT_PANES[$cmdr]}` (the lookup expects commander as key) and `--target-pane` is empty. Multi-repo deploy preflight has therefore never worked end-to-end — the v0.20.0 / v0.20.1 / v0.20.2 dogfood-pass lines are all open in CLAUDE.md, consistent with this latency.

Brainstorming-phase empirical verification (recorded for the implementation engineer):
- `tmux respawn-pane -k -c "$dir" -t <pane> "$cmd"` — empirically confirmed to set `pane_current_path` correctly on tmux 3.4 (the current install).
- The current cd-then-exec form also leaves pane_current_path correct on the happy path.
- Apostrophe-in-cwd breaks the cd-then-exec form (kills the pane).
- `cw_pane_respawn` has exactly ONE caller: `bin/spawn.sh:181` (the preflight `--target-pane` path).

## Goal

After v0.20.3 ships:

- `/clone-wars:deploy` multi-repo: each sub-repo trooper pane is allocated already-rooted in its sub-repo cwd via `tmux split-window -c <sub-repo-cwd>`. The respawn into the trooper TUI uses native `tmux respawn-pane -c <sub-repo-cwd>`. No `cd` command appears anywhere in the spawn flow. `tmux display-message -p '#{pane_current_path}'` returns the sub-repo path immediately after preflight, before any respawn.
- `/clone-wars:deploy` single-repo: byte-equal v0.20.2 (this path uses `cw_pane_spawn_down/right` which already use native `tmux split-window -c`).
- `/clone-wars:consult`: byte-equal v0.20.2 (preflight invocation does not pass `--cwd-from`, so cwd handling stays inherit-Yoda).
- `bash tests/run.sh` adds zero new failures vs baseline at b0bf1a3.
- `.claude-plugin/{plugin,marketplace}.json` show 0.20.3.

## Architecture

Four coordinated layers, applied bottom-up so each step leaves the tree green:

1. **Layer 3 first — `cw_pane_respawn` native -c flag** (lib/tmux.sh). Fixes the cd-then-exec hack. Helper change is independent — works whether or not preflight ever calls it with `<cwd>` set. Single-repo deploy continues to call `spawn.sh --cwd $TARGET_CWD` which threads through and lands here, also benefiting.
2. **Layer 1 — `cmdr-cwd-map.txt` writer** (bin/deploy-multi-init.sh). Writes the new 2-col file alongside the existing 3-col troopers.txt. Pure additive; no existing reader sees a change. Tested in isolation.
3. **Layer 2 — `--cwd-from` flag** (bin/preflight-layout.sh). Optional flag; consult invocations don't pass it (byte-equal). When passed, builds `CMDR_TO_CWD` map and threads `-c <cwd>` per pane in the split-window loop.
4. **Layer 4 — directive wiring** (commands/deploy.md Step 3a). One-line change to multi-repo preflight invocation: pass `--cwd-from "$ART_DIR/cmdr-cwd-map.txt"`. Single-repo path doesn't preflight, so it's untouched.

This order matters: Layers 1+2+4 require Layer 3 to be already in place if you want every Layer-1-fixed pane to also benefit from Layer-3's clean respawn. But Layer 3 is internally consistent on its own (works without Layers 1/2/4).

### Layer 3 — `lib/tmux.sh` `cw_pane_respawn`

Replace the entire function body:

```bash
cw_pane_respawn() {
  local pane="$1" commander="$2" model="$3" topic="$4" launch="$5" cwd="${6:-}"
  if [[ -n "$cwd" ]]; then
    tmux respawn-pane -k -c "$cwd" -t "$pane" "$launch"
  else
    tmux respawn-pane -k -t "$pane" "$launch"
  fi
  cw_pane_label_set "$pane" "$commander" "$model" "$topic"
  printf '%s\n' "$pane"
}
```

The signature, call sites, and return value are unchanged. tmux 3.4's `respawn-pane -c` accepts the start directory natively and is documented in `man tmux` SYNOPSIS for the command.

### Layer 1 — `bin/deploy-multi-init.sh` writes `cmdr-cwd-map.txt`

Append after the existing `mv "$TMP" "$ART_DIR/troopers.txt"` (around line 87) and before the closing `log_ok` / `exit 0`:

```bash
# v0.20.3: write a per-commander cwd map for preflight-layout.sh's --cwd-from
# flag. Separate file (not extending troopers.txt) so the existing Step 3b
# `read cmdr cwd provider` parser stays byte-equal.
```

### Layer 2 — `bin/preflight-layout.sh` `--cwd-from` flag

Two edits inside `bin/preflight-layout.sh`:

**Edit A — flag parsing (after the existing `--art-dir` block, before the positional-arg-count check):**

```bash
# v0.20.3: --cwd-from <abs-path> — TSV file <commander>\t<cwd>; per-pane cwd
# for split-window -c. Optional; consult invocations don't pass it (byte-equal).
CWD_FROM=""
if [[ "${1:-}" == "--cwd-from" ]]; then
  [[ -n "${2:-}" ]] || { echo "--cwd-from requires a value" >&2; exit 2; }
  CWD_FROM="$2"; shift 2
fi
```

This goes AFTER the `--art-dir` parse block so flag order is `--art-dir <X> --cwd-from <Y>` (the order Step 3a will use). If the user reverses, an "unknown flag" error fires from the existing positional check.

**Edit B — map load + per-pane `-c` threading:**

After the roster load (around line 60, before the `YODA_PANE` discovery), add:

```bash
declare -A CMDR_TO_CWD=()
if [[ -n "$CWD_FROM" ]]; then
  [[ -f "$CWD_FROM" ]] || { log_error "--cwd-from file not found: $CWD_FROM"; exit 1; }
  while IFS=$'\t' read -r _cmdr _cwd; do
    [[ -n "$_cmdr" && -n "$_cwd" ]] && CMDR_TO_CWD["$_cmdr"]="$_cwd"
  done < "$CWD_FROM"
fi
```

In the split-window loop (lines 101-116), insert the `-c` threading immediately before the `tmux split-window` call:

```bash
for i in "${!ROSTER[@]}"; do
  IFS=$'\t' read -r prov cmdr <<<"${ROSTER[$i]}"
  sentinel=$(build_sentinel "$cmdr" "$prov" "$TOPIC")
  SPLIT_C_FLAG=()
  if [[ -n "${CMDR_TO_CWD[$cmdr]:-}" ]]; then
    SPLIT_C_FLAG=( -c "${CMDR_TO_CWD[$cmdr]}" )
  fi
  PANE=$(tmux split-window -P -F '#{pane_id}' "$SPLIT_FLAG" -t "$PREV_PANE" "${SPLIT_C_FLAG[@]}" "$sentinel") || {
    log_error "split-window failed at index $i ($cmdr)"
    exit 1
  }
  ...
done
```

When `CWD_FROM` is empty (consult), `SPLIT_C_FLAG=()` empty array, and the bash array expansion `"${SPLIT_C_FLAG[@]}"` produces zero arguments — the split-window invocation is byte-equal to v0.20.2.

### NEW `tests/test_preflight_layout_cwd_from.sh`

tmux-required (skips when `$TMUX` unset). Covers Layer 1 + Layer 2 integration:

```bash
#!/usr/bin/env bash
# tests/test_preflight_layout_cwd_from.sh
# v0.20.3: preflight --cwd-from places each sentinel pane in the assigned cwd.
set -euo pipefail
cd "$(dirname "$0")"
source lib/assert.sh

[[ -n "${TMUX:-}" ]] || { echo "  SKIP: no tmux session ($TMUX unset)" >&2; exit 0; }
command -v tmux >/dev/null || { echo "  SKIP: tmux not on PATH" >&2; exit 0; }

PLUGIN_ROOT="$(cd .. && pwd)"
SANDBOX=$(mktemp -d)
trap 'rm -rf "$SANDBOX"' EXIT
export CLAUDE_PLUGIN_ROOT="$PLUGIN_ROOT"
export CLONE_WARS_HOME="$SANDBOX/.clone-wars"
mkdir -p "$CLONE_WARS_HOME"

source "$PLUGIN_ROOT/lib/log.sh"
source "$PLUGIN_ROOT/lib/state.sh"
source "$PLUGIN_ROOT/lib/consult.sh"

# Build two stub sub-repo dirs
DIR_A=$(mktemp -d)
DIR_B=$(mktemp -d)

TOPIC="cwdfrom-test-$$"
REPO_HASH=$(cw_repo_hash)
ART_DIR="$CLONE_WARS_HOME/state/$REPO_HASH/$TOPIC/_consult"
mkdir -p "$ART_DIR"

# Roster: 2 troopers
cat > "$ART_DIR/troopers.txt" <<EOF
codex	rex
claude	cody
EOF

# CMDR_TO_CWD map: rex → DIR_A, cody → DIR_B
cat > "$ART_DIR/cmdr-cwd-map.txt" <<EOF
rex	$DIR_A
cody	$DIR_B
EOF

# Run preflight in test window
TEST_WIN="cw-cwdfrom-$$-${RANDOM}"
tmux new-window -d -n "$TEST_WIN"
trap 'tmux kill-window -t "$TEST_WIN" 2>/dev/null || true; rm -rf "$SANDBOX" "$DIR_A" "$DIR_B"' EXIT
YODA_PANE=$(tmux list-panes -t "$TEST_WIN" -F '#{pane_id}' | head -1)
sleep 0.5

LOG_FILE="/tmp/cw-cwdfrom-rc-$$.log"
tmux send-keys -t "$YODA_PANE" \
  "CLAUDE_PLUGIN_ROOT='$PLUGIN_ROOT' CLONE_WARS_HOME='$CLONE_WARS_HOME' bash '$PLUGIN_ROOT/bin/preflight-layout.sh' --art-dir '$ART_DIR' --cwd-from '$ART_DIR/cmdr-cwd-map.txt' '$TOPIC' 2 > '$LOG_FILE' 2>&1; echo PFRC=\$? >> '$LOG_FILE'" Enter

got_pfrc=""
for _ in $(seq 1 30); do
  if [[ -f "$LOG_FILE" ]]; then
    line=$(grep '^PFRC=' "$LOG_FILE" 2>/dev/null | tail -1 || true)
    if [[ "$line" == "PFRC=0" ]]; then got_pfrc=0; break; fi
  fi
  sleep 0.5
done
[[ "$got_pfrc" == "0" ]] || { echo "FAIL: preflight rc=$got_pfrc"; cat "$LOG_FILE" >&2; exit 1; }

# Verify each pane's pane_current_path matches the assigned cwd
mapfile -t LINES < "$ART_DIR/preflight-panes.txt"
[[ ${#LINES[@]} -eq 2 ]] || { echo "FAIL: expected 2 lines"; exit 1; }

for line in "${LINES[@]}"; do
  cmdr="${line%%$'\t'*}"
  pane="${line#*$'\t'}"
  expected_cwd=""
  case "$cmdr" in
    rex)  expected_cwd="$DIR_A" ;;
    cody) expected_cwd="$DIR_B" ;;
    *)    echo "FAIL: unexpected cmdr '$cmdr'"; exit 1 ;;
  esac
  actual_cwd=$(tmux display-message -p -t "$pane" '#{pane_current_path}')
  [[ "$actual_cwd" == "$expected_cwd" ]] \
    || { echo "FAIL: pane $pane ($cmdr) cwd '$actual_cwd' != '$expected_cwd'"; exit 1; }
done

pass "preflight --cwd-from: each pane allocated in its assigned cwd"
```

## Risk assessment

- **Layer 3 (`cw_pane_respawn`)**: tmux 3.4+ supported empirically. CLAUDE.md's "tmux ≥ 3.0" requirement is sufficient. Failure mode if user is on tmux 2.x: `respawn-pane -c` returns "unknown flag" and the spawn fails fast with a clear error. Single caller is `bin/spawn.sh:181`; signature unchanged.
- **Layer 1 (`cmdr-cwd-map.txt`)**: pure file-write addition. Zero blast radius; no consumer of troopers.txt sees a change.
- **Layer 2 (`--cwd-from` flag)**: opt-in. consult invocations don't pass it; their `SPLIT_C_FLAG=()` empty array expands to zero args and the split-window call is character-by-character identical to v0.20.2. New invariant covered by both `tests/test_preflight_layout.sh` (existing, consult shape) and `tests/test_preflight_layout_cwd_from.sh` (new).
- **Layer 4 (directive)**: one-line addition to multi-repo Step 3a. Single-repo path doesn't preflight, so it's untouched.

Rollback is `git revert <merge-sha>`. Each layer is independently revertible.
