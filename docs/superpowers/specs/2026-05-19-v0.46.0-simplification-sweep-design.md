# v0.46.0 — Simplification Sweep Design

**Date:** 2026-05-19
**Status:** Approved (brainstormed 2026-05-19; ready for plan)
**Source:** Code-simplifier sweep against clone-wars v0.45.0 (10 findings; this spec covers 6 of them).

## Context

After v0.45.0 (inter-trooper visibility) landed, the codebase has accumulated cross-file duplication in the deep-research surface — particularly path construction and JSONL field extraction. A code-simplifier sweep surfaced 10 distinct findings. This release lands the six Low-risk consolidations; the remaining four are split across v0.47.0 (#2 + #8) and indefinite deferral (#7 + #10).

The thesis "smaller than oh-my-claudecode" makes simplification a first-class concern, not housekeeping. The added helpers also reduce drift surface for future `bin/deep-research-*.sh` scripts (saving ~6 boilerplate lines per new script).

## Goals

Land six Low-risk consolidations:

- **Path-layer** (#4, #5, #6) — clean up deep-research path construction across 6 bin scripts.
- **Helper extractions** (#3, #1, #9) — three independent shared helpers in `lib/`.

Net effect: ~70-90 lines saved + drop one dead variable (`OUTBOX_PATH` in `experiment-send.sh`) + ~6 lines saved per future deep-research bin script.

## Non-goals

- **Finding #2** — unified `cw_dr_json_field`. Deferred to v0.47.0. The existing call sites use subtly different fallback regexes (`[^,}]+` for numeric, `"[^"]*"` for string); merging needs careful regex design that deserves its own focused PR.
- **Finding #8** — hook helper extraction. Deferred to v0.47.0. Different surface area (`.claude/hooks/`, not plugin runtime); blast radius differs from in-plugin refactors.
- **Finding #7** — parallel `cw_<subsystem>_assert_topic` family consolidation. Deferred indefinitely. Memory observation 23638 records that the parallel family was deliberate. Reopening requires its own brainstorm.
- **Finding #10** — lift `_awk_esc` to lib. YAGNI watchlist. Helper has one caller today; lift when the second appears. The 13-line comment-heavy helper stays in `bin/deep-research-experiment-send.sh` until needed elsewhere.

## Design

### Surface 1: Path-layer helpers (Findings #4, #5, #6)

**New helper in `lib/deep-research.sh`:**

```bash
cw_deep_research_art_dir <topic>
  # Prints "<topic_state_dir>/_deep-research".
  # Mirrors cw_meditate_art_dir (lib/meditate.sh:12-14) and
  # cw_deploy_art_dir (lib/deploy.sh:14-16). Deep-research is the
  # missing third sibling.
```

**Existing helpers being adopted (no change to lib):**

- `cw_topic_state_dir` (lib/state.sh) — already exists and used by 3 of 6 sites; the other 3 reconstruct manually.
- `cw_outbox_path` / `cw_inbox_path` / `cw_pane_meta_path` (lib/ipc.sh:28-32) — already exist and used by consult / meditate sides; deep-research bypasses.
- `cw_pane_meta_read` (lib/ipc.sh:308-312) — already exists; deep-research rerolls grep+sed.

**Affected bin scripts (six files):**

- `bin/deep-research-abort.sh`
- `bin/deep-research-experiment-send.sh`
- `bin/deep-research-score.sh`
- `bin/deep-research-fresh-trooper.sh`
- `bin/deep-research-teardown.sh`
- `bin/deep-research-init.sh`

**Replacements:**

| Old pattern | New |
|---|---|
| `state_root + repo_hash + topic` concat | `cw_topic_state_dir "$topic"` |
| `$TOPIC_DIR/_deep-research` literal | `cw_deep_research_art_dir "$topic"` |
| `$TOPIC_DIR/$COMMANDER-codex/outbox.jsonl` | `cw_outbox_path "$COMMANDER" codex "$topic"` |
| `$TOPIC_DIR/$COMMANDER-codex/inbox.md` | `cw_inbox_path "$COMMANDER" codex "$topic"` |
| `$TOPIC_DIR/$COMMANDER-codex/pane.json` | `cw_pane_meta_path "$COMMANDER" codex "$topic"` |
| pane_id grep+sed from pane.json | `cw_pane_meta_read "$COMMANDER" codex "$topic"` |

**Dead-variable cleanup:** `bin/deep-research-experiment-send.sh` defines `OUTBOX` (L132) AND `OUTBOX_PATH` (L170) — same value, two variables. Collapse to one.

### Surface 2: Helper extractions (Findings #3, #1, #9)

**Three new helpers:**

```bash
# lib/deep-research.sh
cw_deep_research_metric_primary <metric_md_path>
  # awk-extracts the "**Primary metric:**" line value.
  # Returns empty on missing/malformed (no exit-fail).
  # Replaces 3 byte-equal awk blocks in:
  #   bin/deep-research-experiment-send.sh:143-149
  #   bin/deep-research-score.sh:43-48
  #   lib/deep-research.sh:445-449 (cw_deep_research_check_completion)

# lib/ipc.sh
cw_jsonl_string_field <jsonl_line> <key>
  # Generalization of existing cw_event_name_extract.
  # Extracts "key":"value" strings from a JSONL line.
  # Replaces 6 open-coded sed sites for "event"/"summary"/"ts"/etc.

# lib/deep-research.sh
cw_deep_research_trooper_event <art_dir> <commander> <event_verb> [k=v ...]
  # Thin wrapper over cw_deep_research_trooper_state_write that
  # stamps last_event_ts (UTC ISO-8601) + last_event=<verb>.
  # Passes through additional k=v args verbatim.
  # Replaces 3 byte-equal stamp blocks + lane_abandon helper internal.
```

**Call site updates:**

- `bin/deep-research-experiment-send.sh:143-149` → `metric_primary`
- `bin/deep-research-score.sh:43-48` → `metric_primary`
- `lib/deep-research.sh:445-449` → `metric_primary`
- `bin/deep-research-monitor.sh:76, 100, 140` → `cw_event_name_extract`
- `bin/deep-research-monitor.sh:103, 146` → `cw_jsonl_string_field … summary`
- `lib/deep-research.sh:635-636` → `cw_jsonl_string_field … ts/event`
- `bin/deploy-wave-wait.sh:65` → `cw_event_name_extract`
- `bin/deep-research-experiment-send.sh:279-284` → `trooper_event`
- `bin/deep-research-fresh-trooper.sh:56-62` → `trooper_event`
- `bin/deep-research-score.sh:154-158` → `trooper_event`
- `lib/deep-research.sh:858-863` (`cw_deep_research_lane_abandon`) → `trooper_event`

## Error handling

All four helpers fail soft on missing/malformed input — print empty + return 0 — matching the precedent set by `cw_event_name_extract` (lib/ipc.sh:167-169) and `_cw_dr_json_field` (lib/deep-research.sh:796-807). Callers already handle empty-string downstream; no exit-fail surface added.

The trooper-event helper requires `art_dir` + `commander` + `event_verb` (3 positional args). Missing args → `return 2` with stderr message, matching `cw_deep_research_format_peers_block`'s pattern.

## Testing

### Unit tests (T1 — RED initially, GREEN after helpers land)

- `tests/test_deep_research_art_dir.sh` — 3 cases: basic path, special chars in topic, integration with `cw_topic_state_dir`.
- `tests/test_deep_research_metric_primary.sh` — 4 cases: present, missing, malformed, multi-line value.
- `tests/test_deep_research_trooper_event.sh` — 3 cases: bare verb, verb+extra k=v, missing args (rc=2).
- `tests/test_jsonl_string_field.sh` — 4 cases: present, absent, escaped quotes, multi-field line.

### Static-wiring lock (T8 — skip-guarded at 0.46.0; activates at T9 version bump)

`tests/test_v0_46_0_static_wiring.sh` — seven invariants:

1. `cw_deep_research_art_dir` defined in `lib/deep-research.sh`.
2. `cw_deep_research_metric_primary` defined in `lib/deep-research.sh`.
3. `cw_deep_research_trooper_event` defined in `lib/deep-research.sh`.
4. `cw_jsonl_string_field` defined in `lib/ipc.sh`.
5. No `state_root.*repo_hash.*topic.*_deep-research` manual concat pattern in `bin/deep-research-*.sh`.
6. No raw `pane_id.*grep` pattern outside `lib/ipc.sh`.
7. No raw `"event":` sed pattern outside `lib/ipc.sh`.

### Behavioral coverage

Existing 277-test suite covers the rewired call sites. No new integration tests planned; the static-wiring lock + per-helper unit tests are the safety net.

## Lane structure

```
T0  baseline             branch + version 0.45.0 + suite green
T1  [test:]              RED tests for 4 new helpers
T2  [refactor(deep-research):]  #4 — cw_deep_research_art_dir + 6 bin migrations
T3  [refactor(ipc):]            #5 — outbox/inbox/pane path helpers + drop OUTBOX_PATH
T4  [refactor(deep-research):]  #6 — cw_pane_meta_read replaces pane_id regex
T5  [refactor(deep-research):]  #3 — cw_deep_research_metric_primary + 3 call sites
T6  [refactor(ipc):]            #1 — cw_event_name_extract + cw_jsonl_string_field
T7  [refactor(deep-research):]  #9 — cw_deep_research_trooper_event + 4 call sites
T8  [chore(release):]           tests/test_v0_46_0_static_wiring.sh (skip-guarded)
T9  [chore(release):]           version bump + CHANGELOG + CLAUDE.md current focus
T10                             push + PR
```

**Ordering rationale:** T2 (art_dir helper) lands before T3-T4 because the path-layer migrations use the new helper. T3 and T4 both touch `experiment-send.sh` + `fresh-trooper.sh` but stay separate commits for review clarity. T5, T6, T7 are independent of each other and the path layer.

**Suite-green checkpoint** runs at the end of every implementation task (T1-T8). If a test outside the v0.46.0 surface regresses, STOP and surface — do not paper over.

**Known flake:** `test_consult_targets_forces_escalation.sh` may flap. Retry once; if it fails twice, flag separately — do not block v0.46.0 on it.

## Risks

- All six in-scope findings rated **Low** by the sweep. The behavioral suite covers the rewired sites.
- Pre-existing flake `test_consult_targets_forces_escalation.sh` may flap once — retry policy applied; documented above.
- The static-wiring lock activates at T9 (version bump). If any T2-T7 commit drifted from the lock's invariants, T9's suite catches it — that's the intended forcing function.

The existing `cw_event_name_extract` becomes a thin wrapper over it OR stays for backward-compat — decided in implementation: keep `cw_event_name_extract` as a named alias to preserve the canonical name in existing callers.
