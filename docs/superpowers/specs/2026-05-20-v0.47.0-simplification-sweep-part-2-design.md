## Design

### Surface 1: Finding #2 — promote JSON field helper

**Current state** (lib/deep-research.sh:846-857):

```bash
_cw_dr_json_field() {
  local f="${1:-}" k="${2:-}"
  [[ -f "$f" && -n "$k" ]] || return 1
  if command -v jq >/dev/null 2>&1; then
    jq -r --arg k "$k" '.[$k] // empty' "$f"
  else
    grep -oE "\"$k\"[[:space:]]*:[[:space:]]*(\"[^\"]*\"|[0-9.eE+-]+|true|false|null)" "$f" \
      | head -1 \
      | sed -E "s/^\"$k\"[[:space:]]*:[[:space:]]*\"?([^\"]*)\"?$/\1/"
  fi
}
```

The helper already handles both jq and grep-fallback paths with a unified regex covering strings, numbers, booleans, and null. Currently only used by `cw_deep_research_render_status_brief`.

**Rename:** `_cw_dr_json_field` → `cw_deep_research_json_field` (drop underscore; promote to public). No body changes.

**Migrations:**

1. **`bin/deep-research-score.sh:69-89`** — Currently 19 lines of `if jq / else 5x grep|sed` branching. Collapse to 5 helper calls:

   ```bash
   metric=$(cw_deep_research_json_field "$result" metric_value)
   status=$(cw_deep_research_json_field "$result" status)
   runtime=$(cw_deep_research_json_field "$result" runtime_s)
   label=$(cw_deep_research_json_field "$result" approach_label)
   metric_name=$(cw_deep_research_json_field "$result" metric_name)
   ```

2. **`lib/deep-research.sh::cw_deep_research_format_peers_block`** (lines ~1058-1072) — 4 extractions (`approach_label`, `metric_value`, `status`, `notes`) → 4 helper calls. Surrounding empty-fallback logic (`[[ -z "$approach" ]] && approach="—"`) stays — that's caller-side defaulting, not extraction.

3. **`lib/deep-research.sh::cw_deep_research_validate_result_json_v033:284-286`** — 1 extraction (`metric_name`) → 1 helper call.

**Risk:** Medium. The renamed helper's behavior is identical to the existing private one (same body), but the migrated callers had subtly different fallback regexes — score.sh used `[^,}]+` for numeric, format_peers_block used `[^"]*"` for string. The unified helper's regex covers both; spot-check edge cases:
- `metric_value: 0.95` → number branch matches `0.95` → return `0.95` (no quotes)
- `status: "ok"` → string branch matches `"ok"` → return `ok` (sed strips quotes)
- `notes: "param-shrink to 96k worked"` → string branch matches the quoted span → returns the inside text

### Surface 2: Finding #5-partial — `cw_outbox_path_in`

**New helper in `lib/ipc.sh`:**

```bash
# cw_outbox_path_in <topic_dir> <commander> <model>
# Variant that takes the topic dir directly instead of reconstructing
# it via cw_topic_state_dir. Used when the caller already has the dir
# from a different anchor (e.g. monitor.sh has $ART_DIR, render_summary
# has $art_dir). Necessary for tests that seed synthetic state under
# mktemp -d — those fixtures don't have a canonical state-root.
cw_outbox_path_in() {
  printf '%s/%s-%s/outbox.jsonl\n' "$1" "$2" "$3"
}
```

Same physical output as `cw_outbox_path` for production paths; trusts the caller's dir.

**Migrations:**

1. **`bin/deep-research-monitor.sh:38`**:
   ```bash
   # Before:
   OUTBOX="$TOPIC_DIR/$COMMANDER-codex/outbox.jsonl"
   # After:
   OUTBOX="$(cw_outbox_path_in "$TOPIC_DIR" "$COMMANDER" codex)"
   ```

2. **`lib/deep-research.sh::cw_deep_research_render_summary`** (around line 631):
   ```bash
   # Before:
   outbox="$topic_dir/$cmdr-codex/outbox.jsonl"
   # After:
   outbox="$(cw_outbox_path_in "$topic_dir" "$cmdr" codex)"
   ```

**Risk:** Low. Drop-in replacement; test fixtures continue to work because the helper trusts the caller's dir.

### Surface 3: Finding #8 — `.claude/hooks/_lib.sh`

**New file** `.claude/hooks/_lib.sh`:

```bash
# .claude/hooks/_lib.sh — shared helpers for project-level PostToolUse hooks.
# Sourced by hook scripts in the same dir. Underscore prefix marks this as
# private to the hooks (not part of the plugin runtime).

# cw_hook_repo_root — absolute path to the clone-wars repo root.
# Assumes the caller is .claude/hooks/<name>.sh; walks up two dirs.
cw_hook_repo_root() {
  cd "$(dirname "${BASH_SOURCE[1]}")/../.." && pwd
}

# cw_hook_file_path_from_stdin — extract tool_input.file_path from
# Claude Code's hook JSON payload (no jq dep). Reads stdin; prints the
# value or empty if absent.
cw_hook_file_path_from_stdin() {
  local input; input=$(cat 2>/dev/null || true)
  printf '%s' "$input" \
    | grep -oE '"file_path"[[:space:]]*:[[:space:]]*"[^"]*"' \
    | head -1 \
    | sed -E 's/.*"file_path"[[:space:]]*:[[:space:]]*"([^"]*)".*/\1/'
}
```

**Migrations:**

1. **`.claude/hooks/post-edit-hardcoded-paths-lint.sh`** — replace lines 12-22 (HERE/REPO_ROOT + input/file_path block) with:
   ```bash
   source "$(dirname "${BASH_SOURCE[0]}")/_lib.sh"
   REPO_ROOT=$(cw_hook_repo_root)
   file_path=$(cw_hook_file_path_from_stdin)
   ```

2. **`.claude/hooks/post-version-bump-lock-check.sh`** — same shape replacement at lines 12-20.

Each hook drops ~7 lines net. Underscore-prefixed lib lives next to its callers (project-level), not in plugin's `lib/`.

**Risk:** Low. Hooks are advisory-only (never block); end-to-end verification is one Edit/Write to a tracked file with output observed.

## Error handling

All helpers fail soft on missing/empty input — empty string + return 0. Matches the existing pattern set by `cw_deep_research_metric_primary` and `cw_jsonl_string_field` (v0.46.0 precedent).

`cw_outbox_path_in` has no failure modes: it concatenates three positional args via printf. The caller is responsible for verifying the resulting path exists (production code already does via `[[ -f "$OUTBOX" ]]` guards).
