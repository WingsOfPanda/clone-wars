# v0.22.0 — Multi-repo deploy seam re-architecture

**Date:** 2026-05-10
**Spec status:** approved (Design B)
**Closes:** 5 layered bugs hit in v0.21.0 dogfood on `/home/liupan/ARS/docs/designs/2026-05-10-10t-checkpoint-deploy.md`

## Problem

`/clone-wars:deploy`'s multi-repo path (introduced in v0.20.0) reuses two
consult primitives — `bin/preflight-layout.sh` and `bin/spawn.sh
--target-pane` — and inlines a deploy-specific dispatch into
`commands/deploy.md` Step 3b. Neither the sharing nor the inline
dispatch was audited for fit with deploy's data shape. Five layered
bugs surfaced on the first end-to-end dogfood:

| # | Symptom | Root |
|---|---|---|
| 1 | spawn `--target-pane` always reads `_consult/<topic>/preflight-panes.txt`, even for deploy | `bin/spawn.sh:91-104` hardcodes `cw_consult_art_dir` — no deploy-aware branch, no override flag |
| 2 | preflight mis-parses deploy's 3-col `troopers.txt` as consult's 2-col | `bin/preflight-layout.sh:64,128` calls `cw_consult_load_troopers` + `IFS=$'\t' read -r prov cmdr` (consult's column order) |
| 3 | `preflight-panes.txt` commander column contains absolute paths instead of commander names | downstream of #2 — `cmdr` variable holds path |
| 4 | preflight panes allocated in Yoda's cwd, not their sub-repo's cwd (silently negates v0.20.3) | downstream of #2 — `${CMDR_TO_CWD[<path-as-cmdr>]}` always empty → `SPLIT_C_FLAG=()` → no `tmux split-window -c` |
| 5 | Troopers stay idle at "Ready event emitted" after dispatch — no nudge | `commands/deploy.md` Step 3b uses `cw_inbox_write` (low-level write-only) without the matching `cw_pane_send` nudge that every other dispatch path pairs |

Plus implicit Bug 0: zero integration tests across this seam — every
v0.20.x / v0.21.0 dogfood release gate was unticked because no one had
run multi-repo deploy end-to-end. Each component works in its consult
home; the deploy seam was directive-prose-only.

## Goal

After v0.22.0 lands:

- Each consumer of `troopers.txt` reads exactly the shape it expects;
  no polymorphic parsing, no auto-detect.
- `bin/spawn.sh --target-pane` works for both consult and deploy via
  an explicit `--preflight-art-dir <path>` flag (deploy passes it;
  consult omits it = byte-equal v0.21.0).
- Multi-repo deploy preflight panes are allocated already-rooted in
  each sub-repo's cwd (v0.20.3's intent restored).
- Multi-repo deploy dispatch uses the canonical write+nudge pattern
  via `bin/send.sh` — the same convention as
  `bin/consult-research-send.sh` and `bin/deploy-turn-send.sh`.
- A new tmux-dependent end-to-end test seals the seam — would have
  caught all 5 bugs.
- Single-repo deploy + flat-monorepo multi-repo deploy + `/consult` are
  byte-equal v0.21.0.

## Architecture

Two structural changes + one convention switch + one test seal.

### Change 1 — Sidecar `troopers-preflight.txt` (consult-shaped)

`bin/deploy-multi-init.sh` already writes `troopers.txt` (3-col
`<commander>\t<sub-repo-cwd>\t<provider>`) and `cmdr-cwd-map.txt`
(2-col `<commander>\t<cwd>`). v0.22.0 adds a third sidecar:

```
_deploy/<topic>/troopers-preflight.txt   — TSV: <provider>\t<commander>
```

Same format as consult's `_consult/<topic>/troopers.txt`. Written
in deterministic DAG order (matches the order in `troopers.txt`).
This is the file `preflight-layout.sh` will read when invoked from
the deploy path.

### Change 2 — `bin/preflight-layout.sh` gains explicit `--troopers-from <path>` flag

Currently preflight reads `$ART_DIR/troopers.txt` unconditionally
(via `cw_consult_load_troopers`, which expects consult's 2-col).
v0.22.0 adds a `--troopers-from <path>` flag:

- When omitted: read `$ART_DIR/troopers.txt` (byte-equal v0.21.0
  consult behavior).
- When passed: read the explicit path. Deploy passes
  `--troopers-from "$ART_DIR/troopers-preflight.txt"`.

The `cw_consult_load_troopers` call is unchanged — it's a passthrough
(`grep -v '^#'`) that works on any consult-shaped 2-col TSV regardless
of which file holds it. The downstream `IFS=$'\t' read -r prov cmdr`
parsing also stays correct because `troopers-preflight.txt` IS
consult-shaped. The `--cwd-from` flag (v0.20.3) keeps working
unchanged — its `<cmdr>\t<cwd>` map is keyed by real commander names,
which now match the `cmdr` variable correctly.

### Change 3 — `bin/spawn.sh` gains explicit `--preflight-art-dir <path>` flag

`bin/spawn.sh:91-104` currently hardcodes
`PFP="$(cw_consult_art_dir "$TOPIC")/preflight-panes.txt"`. v0.22.0
adds a `--preflight-art-dir <abs-path>` flag:

- When omitted: behavior unchanged (consult invocation = byte-equal
  v0.21.0).
- When passed: validate `--target-pane` against
  `<preflight-art-dir>/preflight-panes.txt`. Deploy passes
  `--preflight-art-dir "$ART_DIR"`.

The flag is **explicit-over-implicit**: callers that need a non-consult
preflight path declare it. No env-var sniffing, no topic-prefix magic.

### Change 4 — Step 3b dispatch uses `bin/send.sh @file` (canonical pattern)

Current Step 3b dispatch shape (BUG 5):

```bash
PROMPT=$(cw_deploy_build_dag_unit_prompt …)
cw_inbox_write "${REPO_TO_CMDR[$repo]}" "${REPO_TO_PROVIDER[$repo]}" "$TOPIC" "$PROMPT"
# STOPS — trooper TUI never receives a tmux signal
```

v0.22.0 dispatch shape (matches `consult-research-send.sh:48` +
`deploy-turn-send.sh:tail`):

```bash
PROMPT=$(cw_deploy_build_dag_unit_prompt …)
PROMPT_FILE="$ART_DIR/${REPO_TO_CMDR[$repo]}_dag_unit_prompt.md"
printf '%s' "$PROMPT" > "$PROMPT_FILE"
"$CLAUDE_PLUGIN_ROOT/bin/send.sh" "${REPO_TO_CMDR[$repo]}" "$TOPIC" "@$PROMPT_FILE"
```

`bin/send.sh` resolves the model from `pane.json` (already populated
by spawn), writes inbox.md, AND calls `cw_pane_send` to nudge — one
canonical entry point for all outbound dispatches. The
microoptimization rationale (skip model-resolution lookup) is
discarded; the consistency win is worth the ~20ms scan cost per
trooper.

### Change 5 — `tests/test_deploy_multi_repo_e2e.sh` (NEW, integration)

Tmux-dependent end-to-end test that asserts the full
preflight → spawn → dispatch → nudge seam works. Skips when
`$TMUX` is unset (mirrors existing tmux-dependent tests). The test:

1. Synthesizes 3 sub-repos under a fake hub cwd:
   `$tmp/hub/{flat-sib,inner/CapWords-A,inner/CapWords-B}/CLAUDE.md`.
2. Runs `bin/deploy-init.sh` against a hand-crafted multi-repo
   design doc (with `**Target Sub-Project(s):**` header + a parser-
   conforming `## Execution DAG` section).
3. Asserts: `troopers.txt` (3-col), `troopers-preflight.txt` (2-col,
   matches DAG order), `cmdr-cwd-map.txt` (2-col, commander-keyed).
4. Runs `bin/preflight-layout.sh --art-dir "$ART_DIR" --cwd-from
   "$ART_DIR/cmdr-cwd-map.txt" --troopers-from
   "$ART_DIR/troopers-preflight.txt" "$TOPIC" 3`.
5. Asserts `preflight-panes.txt` has commander names in column 1
   (NOT paths) and pane IDs in column 2; verifies each pane's
   `pane_current_path` matches its sub-repo via `tmux display-message`.
6. Mock-spawns 1 trooper (a sentinel script that emits
   `{"event":"ready"}` to its outbox then `sleep infinity`) into
   one preflight pane via `bin/spawn.sh --target-pane <id>
   --preflight-art-dir "$ART_DIR" --cwd "$sub-repo-cwd" "$cmdr"
   "$provider" "$TOPIC"`.
7. Dispatches a synthetic prompt via `bin/send.sh` (matches Step 3b's
   v0.22.0 shape).
8. Asserts: tmux `capture-pane` of the trooper pane shows
   `Read .../inbox.md` (i.e., the nudge arrived); inbox.md exists
   with the prompt content.
9. Tears down panes; cleans up.

This single test would have caught all 5 bugs. It's the seal that
prevents Bug 6.

## Backwards compatibility

- `bin/preflight-layout.sh`: when `--troopers-from` is omitted, reads
  `$ART_DIR/troopers.txt` exactly as before. Consult invocations are
  byte-equal v0.21.0.
- `bin/spawn.sh`: when `--preflight-art-dir` is omitted, looks up
  preflight-panes.txt under `cw_consult_art_dir` exactly as before.
  Consult invocations + single-repo deploy = byte-equal v0.21.0.
- `bin/deploy-multi-init.sh`: the new sidecar write is purely
  additive — `troopers.txt` and `cmdr-cwd-map.txt` formats unchanged.
- `commands/deploy.md` Step 3b dispatch: shape changes (now uses
  `bin/send.sh @file`), but the user-visible behavior is "trooper
  receives prompt and starts working" — which is the BROKEN expected
  behavior in v0.21.0. No legitimate caller depended on the bare
  `cw_inbox_write` shape.
- Single-repo deploy path: zero touch (Step 3a/3b/3c/3d are multi-repo
  only).
- /consult: zero touch (preflight + spawn flags are opt-in).

## Failure modes

| Where | Trigger | Behavior |
|---|---|---|
| `preflight-layout.sh --troopers-from` | path doesn't exist | rc=1 with `log_error "--troopers-from file not found: <path>"`; same shape as v0.20.3's `--cwd-from` guard |
| `spawn.sh --preflight-art-dir` | path doesn't contain `preflight-panes.txt` | rc=1 with `log_error "--target-pane requires preflight-panes.txt at: <path>/preflight-panes.txt"`; same shape as current consult-art-dir guard |
| `deploy-multi-init.sh` | sidecar write fails (full disk, etc.) | rc=1 with `log_error "mv troopers-preflight.txt failed"`; matches existing TMP+mv discipline |
| `bin/send.sh` (called from Step 3b) | model resolution finds no matching state dir | existing behavior — `log_error "no trooper '<cmdr>' on topic '<topic>'"` and rc=1; deploy directive surfaces and aborts the wave |
| Step 3b prompt-file write | `printf > $PROMPT_FILE` fails | bash `set -uo pipefail` propagates rc; deploy directive surfaces |

## Out of scope

- **Backwards-compat for v0.21.0 multi-repo deploy state.** Any
  in-flight v0.21.0 multi-repo deploy will fail under v0.22.0
  (preflight-panes.txt path mismatch). Acceptable: per CLAUDE.md
  status, NO v0.21.0 multi-repo deploy ever ran to completion (5 bugs).
  Users tear down via `bin/deploy-archive.sh` and re-invoke.
- **`bin/deploy-dispatch.sh` wrapper.** Inlining the prompt-file +
  send.sh pair into Step 3b directive prose is sufficient for now;
  if a 3rd dispatcher emerges (some future deploy variant) we extract
  then.
- **Auto-detect troopers.txt schema.** Considered (Design A) and
  rejected — polymorphic parsing puts the integration burden on a
  single helper and invites Bug N+1.
- **Full deploy/consult separation (Design C).** Considered and
  rejected for ~3× LoC + sync burden on duplicated split-window
  orchestration.
- **`bin/preflight-layout.sh` deprecation of `--art-dir` in favor of
  `--troopers-from + --panes-out`.** Could be cleaner (no implicit
  art-dir-derived path) but breaks consult byte-equal. Defer to v0.23+
  if the win materializes.
- **Hardening preflight-layout against consult's `troopers.txt`
  growing a 3rd column for some future feature.** Not on the roadmap.
